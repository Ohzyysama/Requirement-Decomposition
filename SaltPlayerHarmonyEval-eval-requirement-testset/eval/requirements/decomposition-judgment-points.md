# 需求拆分测试集 —— 77 个 (SR→AR) 判定点

> 标准答案来源：`SR-AR-mapping.md`（15 SR / 77 AR，重排编号版）。每个判定点是一条 (SR→AR) 断言：
> 「对 SR-X 做需求拆分时，应当拆出 AR-Y」。系统对每个 SR 的拆分结果与本表对照，逐点判定命中与否，
> 据此计算**拆分召回率 / 准确率 / 粒度合理性**。

## 如何使用

1. 让系统对仓库（或逐个 SR）做需求拆分，得到它自己的 `SR → {AR...}` 结果。
2. 用语义对齐把系统产出的每个细粒度需求映射到本表的 AR（标题/功能两者择一可判定等价）。
3. 逐个判定点记 命中（hit）/未命中（miss）；系统产出但不对应任何判定点的记为 过拆（spurious）。

## 评分口径

- **拆分召回率 Recall** = 命中判定点数 / 77。
- **拆分准确率 Precision** = 命中判定点数 / (命中 + 过拆)。
- **F1** = 2·P·R/(P+R)。
- **粒度合理性（按 SR）**：对每个 SR 比较 |系统拆出的 AR 数| 与 |标准 AR 数|，统计 偏粗(漏)/偏细(过拆)/吻合；可报告 SR 级平均绝对粒度误差 = Σ|预测AR数−标准AR数| / 15。
- **SR 覆盖率** = 系统识别出的 SR 数 / 15（识别即系统产出的总体需求能对齐到某个标准 SR）。
- 建议同时报告 micro（按 77 点汇总）与 macro（先算每 SR 的 P/R 再对 15 个 SR 平均）两种口径。
