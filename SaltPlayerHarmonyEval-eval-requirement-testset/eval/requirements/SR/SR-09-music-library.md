# SR-09 音乐库
- SR-ID: SR-09
- 包含 AR: AR-09-01 音质分布与容量统计、AR-09-02 音乐库搜索

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音乐库 |
| [需求描述] | 本需求提供音乐库的统计概览与全库检索能力，帮助用户从整体视角认识和检索本地曲库。音质分布与容量统计展示不同音质歌曲的分布、音乐库歌曲总大小及各音质歌曲数目（AR-09-01），让用户直观掌握曲库的音质构成与占用情况。音乐库搜索支持按关键词搜索歌曲，搜索结果以歌曲列表形式展示（AR-09-02），方便用户在全库范围内快速定位目标歌曲。两项能力共同构成音乐库的「概览 + 检索」入口。 |
| [需求价值] | 让用户一眼掌握本地曲库的音质构成与存储占用，并能在全库范围内快速检索定位歌曲，提升曲库认知与查找效率。 |
| [需求场景] | 用户进入音乐库查看不同音质歌曲分布、总大小与各音质数量；随后输入关键词搜索某首歌曲，从以歌曲列表展示的结果中点击播放。 |
| [目标用户] | 关注本地曲库音质构成与存储占用、并需要在全库快速查找歌曲的用户。 |
| [限制约束] | 统计与搜索范围均限定于已扫描入库的本地歌曲；音质分类、文件大小等依赖扫描元数据；搜索结果排序与匹配策略依实现。 |
| [外部依赖] | 本地媒体扫描入库数据、歌曲音质与文件大小元数据。 |
| [性能指标] | 评测基线目标：音乐库统计概览即时展示（<100ms 量级）；搜索输入响应及时，结果列表加载与滚动 ≥ 60fps。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 音乐库展示不同音质歌曲的分布及各音质歌曲数目。 2. 音乐库展示歌曲总大小（容量统计）。 3. 支持按关键词搜索歌曲，搜索结果以歌曲列表展示。 4. 统计与搜索均基于已扫描入库的本地歌曲，结果准确。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 音质分布与容量统计、音乐库搜索为通用能力，手机/平板一致展示；车机以搜索与播放为主，统计概览可简化展示或隐藏，避免驾驶场景信息过载。 |
| [视觉/生态/安全等扩展维度] | 视觉：统计以可视化图表/分组数据展示音质分布与容量，搜索结果沿用歌曲列表视觉规范；安全：统计与搜索仅基于本地已扫描数据，不涉及网络上传。 |
