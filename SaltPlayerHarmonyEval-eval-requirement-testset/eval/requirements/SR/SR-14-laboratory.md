# SR-14 实验功能（实验室）
- SR-ID: SR-14
- 包含 AR: AR-14-01 启动自动打开播放页、AR-14-02 立体歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 实验功能（实验室） |
| [需求描述] | 在「实验室」入口集中提供处于预览/试验阶段的能力，供用户尝鲜与反馈。聚合 AR-14-01 启动自动打开播放页（应用冷启动时直接进入播放页，缩短到达正在播放内容的路径）与 AR-14-02 立体歌词（3D 立体歌词视觉效果）两项原子能力。实验功能默认关闭，由用户主动开启，开启后即时生效并随应用配置持久化保存。其中立体歌词为预告/实验性能力，可能存在视觉或性能未稳定的情况。 |
| [需求价值] | 以低风险方式向用户开放前沿能力，收集真实反馈以指导后续正式化，提升尝鲜用户的参与感与产品差异化体验。 |
| [需求场景] | 用户在设置中进入「实验室」，打开「启动自动打开播放页」后重启应用直接进入播放页；打开「立体歌词」后在播放页查看 3D 立体歌词效果。 |
| [目标用户] | 愿意尝试新特性、对个性化与前沿体验有诉求的进阶用户。 |
| [限制约束] | 实验功能均为预览/试验性质，可能在后续版本调整或下线；立体歌词依赖设备渲染能力，低端设备上可能降级或表现不稳定；各开关仅影响对应实验能力，不改变其他既有功能行为。 |
| [外部依赖] | 启动自动打开播放页依赖应用启动流程与上次播放状态恢复；立体歌词依赖系统图形渲染能力与歌词数据来源；无强第三方服务依赖。 |
| [性能指标] | 基线目标：开启「启动自动打开播放页」后冷启动到达播放页时间不显著劣化；立体歌词渲染帧率保持流畅，开启后不引起播放页明显卡顿。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 实验室入口可正常进入，列出全部实验项且默认处于关闭状态；2. 开启「启动自动打开播放页」后重启应用能直接进入播放页，关闭后恢复默认启动页；3. 开启「立体歌词」后播放页能呈现 3D 立体歌词效果，关闭后恢复常规歌词；4. 各实验开关状态在应用重启后保持持久化；5. 实验功能开关互不干扰，不影响其他界面与播放行为。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 手机为主体验载体；平板可在更大屏幕下呈现立体歌词；车机出于行车安全与算力考虑，弱化或不开放立体歌词等高渲染开销能力，启动直达播放页则更契合车载即时收听场景。 |
| [视觉/生态/安全等扩展维度] | 立体歌词需兼顾动效可读性与无障碍，避免眩晕与过度刺激；实验功能应明确标识其预览性质，避免用户误解为稳定特性。 |
