# SR-07 文件夹与存储管理
- SR-ID: SR-07
- 包含 AR: AR-07-01 文件夹列表展示、AR-07-02 文件夹列表搜索、AR-07-03 隐藏文件夹、AR-07-04 文件夹详情-歌曲列表项展示、AR-07-05 文件夹详情-歌曲列表排序、AR-07-06 文件夹详情-ABC 快速索引、AR-07-07 文件夹详情-随机播放全部、AR-07-08 文件夹详情-多选操作、AR-07-09 文件夹详情-更多操作菜单、AR-07-10 文件夹详情-歌曲信息详情

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹与存储管理 |
| [需求描述] | 本需求从文件存储结构维度组织本地音乐，让用户按目录浏览与管理歌曲。文件夹列表页展示所有含本地歌曲的文件夹（AR-07-01），支持在列表内搜索（AR-07-02）以及隐藏指定文件夹（AR-07-03）。点击文件夹进入详情页，详情页复用整套歌曲列表能力：列表项展示封面/音质/歌名/艺术家（AR-07-04）、多维度排序（AR-07-05）、ABC 快速索引并定位当前歌曲（AR-07-06）、一键随机播放全部（AR-07-07）、多选后播放选中队列/加入歌单/从音乐库移除（AR-07-08）、单首更多操作菜单（AR-07-09）以及歌曲信息详情展示（AR-07-10）。整体让用户在「目录—歌曲」两级结构中完成浏览、播放与管理闭环。 |
| [需求价值] | 让用户从存储目录维度组织和查找音乐，便于定位某一来源或某一批次导入的歌曲，并在文件夹内完成与歌曲列表一致的播放与管理操作。 |
| [需求场景] | 用户进入文件夹页浏览所有含歌曲文件夹及歌曲数量，可搜索或隐藏文件夹；点击某文件夹进入详情页，对其中歌曲进行排序、索引定位、随机播放、多选批量操作、单首菜单操作及查看歌曲信息。 |
| [目标用户] | 习惯按文件夹/目录结构管理本地音乐的用户。 |
| [限制约束] | 仅展示已扫描入库且包含歌曲的文件夹；被隐藏/屏蔽的文件夹不在列表中展示；文件夹与歌曲信息均来源于本地扫描结果；移除、隐藏等操作仅作用于应用媒体库，不删除磁盘原始文件。 |
| [外部依赖] | 本地媒体扫描入库数据、文件夹与歌曲归类信息、歌单数据、系统分享能力。 |
| [性能指标] | 评测基线目标：文件夹列表与详情页歌曲列表加载、滚动 ≥ 60fps；进入页面即时展示（<100ms 量级）；ABC 索引定位与随机播放响应及时。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 文件夹页以列表展示所有含本地歌曲的文件夹，每项显示名称、路径与歌曲数量。 2. 支持在文件夹列表内搜索，并可隐藏指定文件夹，被隐藏的文件夹不再出现在列表中。 3. 点击文件夹进入详情页，列表项展示封面/音质/歌名/艺术家，支持多维度排序与 ABC 索引定位当前歌曲。 4. 详情页支持一键随机播放全部，以及多选后播放选中队列、添加到歌单、从音乐库移除。 5. 详情页单首歌曲提供更多操作菜单（加入歌单、下一首播放、分享、跳转艺术家/专辑、查看歌曲信息、从音乐库移除）并可查看完整歌曲信息详情。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 文件夹列表与详情浏览为通用能力，各形态一致；平板可利用更大屏展示更多列信息；车机以更大字号、简化交互为主，多选批量与歌曲信息详情等复杂操作在车机弱化或隐藏。 |
| [视觉/生态/安全等扩展维度] | 视觉：文件夹名为主标题、路径为副标题、右侧显示歌曲数量，详情页沿用歌曲列表视觉规范；安全：仅展示与操作本地已扫描歌曲，移除/隐藏不触及磁盘文件。 |
