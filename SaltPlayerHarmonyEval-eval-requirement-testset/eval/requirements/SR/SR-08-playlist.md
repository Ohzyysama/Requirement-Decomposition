# SR-08 歌单管理
- SR-ID: SR-08
- 包含 AR: AR-08-01 创建歌单、AR-08-02 重命名歌单、AR-08-03 删除歌单、AR-08-04 歌曲加入/移出歌单、AR-08-05 歌单导入/导出、AR-08-06 歌单排序

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌单管理 |
| [需求描述] | 本需求提供完整的歌单生命周期管理能力，让用户自由组织本地歌曲集合。支持新建歌单（AR-08-01）、重命名歌单（AR-08-02）与删除歌单（AR-08-03），完成歌单本身的增删改。支持将歌曲加入或移出指定歌单（AR-08-04），实现歌单内容维护。支持歌单的导入与导出（AR-08-05），便于在设备间迁移或备份歌单。支持调整歌单排列顺序（AR-08-06），让用户按个人偏好组织歌单展示。整体覆盖「歌单—歌曲—顺序—迁移」的全流程管理闭环。 |
| [需求价值] | 让用户按主题、心情或场景自定义组织歌曲集合，并可在设备间迁移备份，提升本地音乐的个性化管理与复用效率。 |
| [需求场景] | 用户创建一个新歌单并命名，将多首喜欢的歌曲加入其中，随后重命名、调整歌单排列顺序，或将歌单导出备份/导入到其他设备，不再需要的歌单则删除。 |
| [目标用户] | 希望按主题/场景自定义组织本地音乐并进行备份迁移的用户。 |
| [限制约束] | 歌单内歌曲来源于已扫描入库的本地歌曲；删除歌单仅移除歌单本身及其歌曲引用，不删除磁盘原始文件；导入歌单需符合应用支持的格式；歌单名称在应用内的唯一性约束依实现策略。 |
| [外部依赖] | 本地媒体扫描入库数据、本地歌单存储、文件系统读写（导入/导出）。 |
| [性能指标] | 评测基线目标：歌单创建/重命名/删除及加入移出歌曲操作即时响应（<100ms 量级）；歌单列表与排序拖动 ≥ 60fps；导入/导出在常规歌单规模下及时完成。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 支持新建歌单并命名，支持重命名与删除歌单。 2. 支持将歌曲加入指定歌单，以及从歌单移出歌曲。 3. 支持歌单的导入与导出。 4. 支持调整歌单排列顺序并持久保存。 5. 删除歌单仅移除歌单及其引用，不影响磁盘原始歌曲文件。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 歌单管理为通用能力，手机/平板完整支持创建、编辑、导入导出与排序；车机以浏览与播放歌单为主，创建、重命名、导入导出等编辑类操作在车机弱化或隐藏，避免驾驶场景复杂交互。 |
| [视觉/生态/安全等扩展维度] | 视觉：歌单以列表/卡片展示名称与歌曲数，编辑入口清晰；安全：导入导出限定本地文件读写，歌单操作不触及磁盘原始歌曲文件。 |
