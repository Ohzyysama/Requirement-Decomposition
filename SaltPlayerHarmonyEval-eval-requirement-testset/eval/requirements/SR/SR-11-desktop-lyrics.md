# SR-11 桌面与通知歌词
- SR-ID: SR-11
- 包含 AR: AR-11-01 桌面歌词开关、AR-11-02 锁定/解锁桌面歌词、AR-11-03 悬浮窗状态栏歌词、AR-11-04 通知栏歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 桌面与通知歌词 |
| [需求描述] | 本需求聚合应用在桌面、悬浮窗与通知栏等应用外区域展示实时歌词的能力。通过 AR-11-01 桌面歌词开关基于 AVSession 控制桌面歌词的开启与关闭；通过 AR-11-02 提供桌面歌词的锁定与解锁，锁定后歌词位置固定、避免误触；通过 AR-11-03 在状态栏悬浮窗展示当前播放歌词；通过 AR-11-04 在通知栏同步展示歌词。各入口歌词均随播放进度实时滚动，使用户在退出应用界面或锁屏场景下仍可查看当前歌词。 |
| [需求价值] | 让用户在不打开应用主界面的情况下随时查看实时歌词，提升后台与多任务场景下的歌词可见性与跟唱体验。 |
| [需求场景] | 用户开启桌面歌词后回到桌面或切换其他应用仍可见歌词；锁定桌面歌词后位置固定不被误触；在状态栏悬浮窗与通知栏查看随播放滚动的歌词。 |
| [目标用户] | 喜欢边做其他事边看歌词、跟唱以及后台听歌时查看歌词的用户。 |
| [限制约束] | 桌面歌词依赖系统悬浮窗/通知权限，未授权时相关入口不可用或需引导授权；桌面歌词锁定后不响应拖动等手势；多歌词入口应共享同一播放进度，避免不同步。 |
| [外部依赖] | 系统媒体会话能力（AVSession）、系统悬浮窗与通知栏展示能力、悬浮窗/通知相关权限、当前播放歌词数据源。 |
| [性能指标] | 评测基线目标：歌词随播放进度滚动延迟 < 300ms，开关桌面歌词响应 < 500ms。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 可通过开关开启/关闭桌面歌词，开启后退出应用界面仍可见实时滚动歌词；2. 可锁定/解锁桌面歌词，锁定后位置固定且不响应误触手势；3. 状态栏悬浮窗可展示当前播放歌词并随进度滚动；4. 通知栏可同步展示当前歌词；5. 缺少所需权限时给出引导且不影响主播放功能。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 手机/平板支持桌面歌词、悬浮窗与通知栏歌词；车机受系统悬浮窗/通知策略限制，桌面歌词与悬浮窗能力可能不适用，以通知/车机仪表歌词展示为主或降级。 |
| [视觉/生态/安全等扩展维度] | 桌面与通知歌词需遵循系统悬浮窗/通知生态规范，文字配色满足在桌面背景上的可读性与对比度，权限申请需符合最小必要与合规要求。 |
