# SR-06 艺术家管理
- SR-ID: SR-06
- 包含 AR: AR-06-01 艺术家列表项展示、AR-06-02 艺术家列表 ABC 快速索引、AR-06-03 艺术家详情头部展示、AR-06-04 艺术家详情歌曲列表、AR-06-05 艺术家详情专辑区块

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 艺术家管理 |
| [需求描述] | 提供从艺术家列表到艺术家详情的完整浏览与播放能力。艺术家列表项展示头像、艺术家名与歌曲数（AR-06-01），并支持 ABC 字母快速索引定位（AR-06-02）。进入艺术家详情后，头部展示头像与艺术家名（AR-06-03），歌曲列表展示曲目与歌曲数，支持随机播放、歌曲折叠及单首操作（点击播放、添加下一首、更多菜单）（AR-06-04），专辑区块则展示该艺术家相关专辑的封面/专辑名/专辑艺术家/歌曲数并支持跳转专辑详情页（AR-06-05）。 |
| [需求价值] | 以艺术家为维度组织本地曲库，帮助用户快速找到某位艺术家的全部歌曲与专辑并发起播放，与专辑管理形成互补的浏览路径，提升曲库可发现性。 |
| [需求场景] | 用户进入艺术家页浏览全部艺术家，通过 ABC 索引快速定位；点击某位艺术家进入详情，查看其歌曲与专辑，随机播放全部或对单首歌曲点击播放/加入下一首，并可从专辑区块跳转到对应专辑详情。 |
| [目标用户] | 习惯按艺术家检索与收听本地音乐的用户。 |
| [限制约束] | 艺术家信息来源于音频元数据，元数据缺失时以默认占位（如默认头像、未知艺术家）展示；ABC 索引依据艺术家名首字母/拼音归类；同名/多艺术家场景下的归并以元数据为准。 |
| [外部依赖] | 媒体库艺术家聚合数据、音频元数据与头像/封面读取能力、应用内路由跳转（专辑详情）。 |
| [性能指标] | 评测基线目标：艺术家列表滚动流畅（60fps 目标）；进入艺术家详情首屏渲染 < 300ms；头像与封面加载异步不阻塞滚动。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 艺术家列表项正确展示头像、艺术家名与歌曲数，头像缺失时显示默认占位；2. ABC 快速索引可按首字母定位到对应艺术家；3. 艺术家详情头部正确展示头像与艺术家名；4. 详情歌曲列表支持随机播放、歌曲折叠及单首点击播放/添加下一首/更多菜单操作；5. 专辑区块正确展示相关专辑信息并可跳转至专辑详情页。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 各端艺术家数据与详情逻辑一致；平板可在更大视图内并列展示歌曲与专辑区块，车机端简化交互、以大图与语音/按键播放为主，弱化折叠与精细菜单操作。 |
| [视觉/生态/安全等扩展维度] | 头像与封面加载需做内存与缓存优化；元数据异常需容错不崩溃；多区块布局在不同分辨率下保证视觉一致与对齐。 |
