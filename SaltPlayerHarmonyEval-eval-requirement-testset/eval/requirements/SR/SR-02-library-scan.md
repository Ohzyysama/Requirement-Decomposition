# SR-02 媒体库与本地扫描
- SR-ID: SR-02
- 包含 AR: AR-02-01 音频扫描、AR-02-02 重新扫描、AR-02-03 扫描屏蔽文件夹、AR-02-04 过滤 60 秒以下短音频

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 媒体库与本地扫描 |
| [需求描述] | 本需求负责本地音频的发现与入库，构成整个应用数据来源的基础。它聚合四项原子能力：扫描本地音频文件并解析元信息入库（AR-02-01）、用户主动触发重新扫描以刷新媒体库（AR-02-02）、在扫描时屏蔽（排除）指定文件夹（AR-02-03）、以及自动过滤时长低于 60 秒的短音频以减少铃声/提示音等噪声条目（AR-02-04）。扫描结果作为歌曲、专辑、艺术家、文件夹等上层视图的统一数据基底。 |
| [需求价值] | 准确、可控的本地扫描入库是离线音乐播放器的根基，决定了媒体库内容的完整度与纯净度，直接影响用户找歌与听歌体验。 |
| [需求场景] | 用户首次启动或新增本地音乐后触发扫描入库；在文件有变动时手动重新扫描刷新；为排除特定目录（如录音、缓存）而设置屏蔽文件夹；希望媒体库中不出现过短的非歌曲音频。 |
| [目标用户] | 拥有本地音乐文件、以离线收藏管理为主的用户。 |
| [限制约束] | 扫描依赖存储访问授权；仅纳入受支持的音频格式；屏蔽文件夹与短音频过滤规则在扫描阶段统一生效；重复扫描需正确处理增量、删除与去重，避免重复入库。 |
| [外部依赖] | 存储访问与媒体文件读取授权、系统媒体文件检索能力、音频元数据解析能力。 |
| [性能指标] | 扫描过程不阻塞主界面交互，可在后台进行并给出进度反馈；中等规模本地库扫描在合理时间内完成。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 首次或新增音乐后扫描能正确发现本地音频并解析元信息入库，上层歌曲/专辑/艺术家视图随之更新（覆盖 AR-02-01）；2. 手动触发重新扫描后媒体库正确刷新，新增条目入库、已删除文件移除、无重复条目（覆盖 AR-02-02）；3. 设置屏蔽文件夹后该目录及其内容不被纳入扫描结果（覆盖 AR-02-03）；4. 时长低于 60 秒的短音频在扫描时被自动过滤，不出现在媒体库中（覆盖 AR-02-04）。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 手机/平板具备完整的本地文件扫描与屏蔽配置能力；车机本地存储与目录结构受限，扫描与屏蔽能力按平台可访问范围裁剪。 |
| [视觉/生态/安全等扩展维度] | 扫描需在用户授权前提下访问存储，遵循系统隐私与权限规范；屏蔽与过滤策略对用户透明可配置。 |
