# SR-03 歌曲管理
- SR-ID: SR-03
- 包含 AR: AR-03-01 歌曲列表项展示、AR-03-02 歌曲列表搜索、AR-03-03 歌曲列表排序、AR-03-04 歌曲列表 ABC 快速索引、AR-03-05 歌曲列表随机播放全部、AR-03-06 歌曲列表多选操作、AR-03-07 歌曲更多操作菜单、AR-03-08 歌曲信息详情展示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲管理 |
| [需求描述] | 本需求围绕歌曲列表，聚合从浏览、检索到操作、查看详情的完整能力。它包含列表项展示封面/音质/歌名/艺术家（AR-03-01）、列表内搜索歌曲（AR-03-02）、多维度排序（AR-03-03）、ABC 字母快速索引并可定位到当前歌曲（AR-03-04）、一键随机播放整张列表（AR-03-05）、多选后批量操作（播放选中队列、添加到歌单、从音乐库移除，AR-03-06）、单曲更多操作菜单（添加到歌单、下一首播放、分享、跳转艺术家/专辑详情、查看歌曲信息、从音乐库移除，AR-03-07），以及歌曲信息详情展示标题/艺术家/专辑/专辑艺术家/时长/比特率/声道/采样率/位深/大小/格式/编码/版权/添加与修改时间/路径（AR-03-08）。该套能力同时作为文件夹详情等场景复用的歌曲列表基座。 |
| [需求价值] | 歌曲列表是用户管理与消费本地曲库的主入口，完善的展示、检索、排序、批量与单曲操作能力直接决定找歌效率与管理体验。 |
| [需求场景] | 用户浏览全部歌曲；按关键词搜索目标歌曲；按多维度排序整理列表；用 ABC 索引快速跳转并定位当前歌曲；一键随机播放整张列表；多选批量加入歌单/播放/移除；对单曲使用更多菜单进行播放、分享、跳转或移除；查看某首歌的完整技术与元信息。 |
| [目标用户] | 拥有较大本地曲库、需要高效检索与批量管理的听歌用户。 |
| [限制约束] | 操作对象需为已入库歌曲；从音乐库移除等操作需与多选/单选状态一致并防误触；多艺术家场景下跳转艺术家详情需先选择目标艺术家；菜单与详情字段以可解析到的元数据为准，缺失时合理留空或占位。 |
| [外部依赖] | 媒体库数据、音频元数据解析能力、歌单存储能力、系统分享能力、播放队列与跳转能力、封面图像资源。 |
| [性能指标] | 大列表滚动流畅、搜索与排序响应及时；ABC 索引定位即时无明显卡顿。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 歌曲列表项正确展示封面/音质/歌名/艺术家，列表内搜索可按关键词过滤出目标歌曲（覆盖 AR-03-01、AR-03-02）；2. 多维度排序生效，ABC 快速索引可跳转字母段并定位到当前播放歌曲（覆盖 AR-03-03、AR-03-04）；3. 一键随机播放可随机播放整张列表（覆盖 AR-03-05）；4. 多选后可批量播放选中队列、添加到歌单、从音乐库移除（覆盖 AR-03-06）；5. 单曲更多菜单可完成添加到歌单/下一首播放/分享/跳转艺术家与专辑详情/查看歌曲信息/从音乐库移除，且歌曲信息详情完整展示各项元数据字段（覆盖 AR-03-07、AR-03-08）。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 手机/平板提供完整的搜索、排序、多选与详情交互；车机受驾驶安全与交互规范限制，多选批量、详情长列表等复杂操作按车机场景简化或裁剪，保留点击播放、随机播放等核心能力。 |
| [视觉/生态/安全等扩展维度] | 分享遵循系统分享生态；从音乐库移除等操作需明确反馈防误删；列表与详情视觉遵循应用主题与各端设计规范。 |
