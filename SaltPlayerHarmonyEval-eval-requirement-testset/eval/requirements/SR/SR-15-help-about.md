# SR-15 帮助/关于/致谢
- SR-ID: SR-15
- 包含 AR: AR-15-01 帮助与 FAQ、AR-15-02 用户反馈、AR-15-03 关于与版本/检查更新、AR-15-04 开源致谢

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 帮助/关于/致谢 |
| [需求描述] | 为用户提供集中的帮助、反馈、版本与致谢信息域，降低使用门槛并增强产品透明度。聚合 AR-15-01 帮助与 FAQ（功能说明与常见问题解答）、AR-15-02 用户反馈（提交使用问题与建议）、AR-15-03 关于与版本/检查更新（展示版本信息并检查更新）、AR-15-04 开源致谢（列示所用开源组件及致谢）四项原子能力。用户可在此自助查阅用法、反馈问题、了解版本状态并查看开源贡献，形成完整的信息支持闭环。 |
| [需求价值] | 降低用户上手与求助成本，减少重复咨询；建立反馈渠道收集改进诉求；通过版本与开源致谢提升产品透明度、合规性与用户信任。 |
| [需求场景] | 用户在设置/关于页查阅帮助与 FAQ，展开问题查看解答；遇到问题时提交反馈；在关于页查看版本号并检查更新；浏览开源致谢了解所依赖的开源组件。 |
| [目标用户] | 初次使用或遇到使用疑问、希望反馈问题或关注版本与开源信息的全体用户。 |
| [限制约束] | 帮助内容以本地内置文案为主；检查更新与反馈提交在无网络时受限或不可用；致谢清单随版本静态维护；条目展开/收起仅影响当前视图，不改变应用状态。 |
| [外部依赖] | 检查更新依赖网络与更新服务可用性；用户反馈依赖反馈提交通道（如邮件或服务端接口）；如含外链跳转则依赖系统浏览器能力；帮助与开源致谢以内置文案为主，无强外部依赖。 |
| [性能指标] | 基线目标：帮助页与关于页打开、FAQ 条目展开响应 < 200ms，列表滚动流畅无明显卡顿；检查更新请求在正常网络下及时返回结果。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 帮助页能正确显示功能说明与 FAQ，条目可展开/收起，文案完整无截断或乱码；2. 用户反馈入口可正常进入并成功提交反馈，异常情况有明确提示；3. 关于页正确展示版本信息，检查更新能反馈「已是最新」或「有新版本」结果；4. 开源致谢页完整列出开源组件及致谢信息；5. 各页面返回操作正常，不影响其他界面状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 平板可利用更大屏幕分栏展示帮助与致谢内容；车机以精简静态文案为主，弱化长篇阅读与反馈输入场景，检查更新与开源致谢可保留只读展示。 |
| [视觉/生态/安全等扩展维度] | 文案遵循无障碍可读性要求；用户反馈与检查更新涉及网络传输，需符合系统隐私与安全规范；开源致谢需满足开源许可证的署名与合规要求。 |
