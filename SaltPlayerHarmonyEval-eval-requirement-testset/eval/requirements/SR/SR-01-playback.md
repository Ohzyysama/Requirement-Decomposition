# SR-01 播放控制与队列
- SR-ID: SR-01
- 包含 AR: AR-01-01 播放/暂停/上一曲/下一曲控制、AR-01-02 播放模式切换（循环/单曲/随机）、AR-01-03 播放队列查看/清空/移除/播放模式切换、AR-01-04 播放速度调节、AR-01-05 播放进度拖动与跳转、AR-01-06 睡眠定时、AR-01-07 播放页展示专辑封面、AR-01-08 播放页快捷操作入口

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放控制与队列 |
| [需求描述] | 本需求构成应用的核心播放域，聚合一整套围绕"当前曲目与播放队列"的原子能力。它包含最基础的播放/暂停/上一曲/下一曲控制（AR-01-01）、列表循环/单曲循环/随机三种播放模式切换（AR-01-02）、播放队列的查看/清空/单项移除及队列内模式切换（AR-01-03）、播放速度调节（AR-01-04）、进度条拖动定位与跳转（AR-01-05）、睡眠定时及"听完整首再停"开关（AR-01-06）。同时聚合播放页的视觉与快捷能力：主视觉区随切歌实时更新的专辑封面展示（AR-01-07），以及保持屏幕唤醒、沉浸模式、音频信息、跳转专辑/艺术家详情等快捷入口（AR-01-08）。各项控制在应用内播放界面与系统媒体卡片（锁屏/控制中心）保持一致体验。 |
| [需求价值] | 播放控制是音乐播放器不可或缺的核心交互；完整的队列、模式、进度、定时与快捷入口能力共同决定了用户日常听歌的基础体验与黏性。 |
| [需求场景] | 用户在播放界面进行播放/暂停、切歌、拖动进度、切换播放模式、调节速度、设置睡眠定时；查看与管理当前播放队列；从锁屏或控制中心系统媒体卡片下达播放控制命令；在播放页查看封面并使用快捷入口跳转详情或进入沉浸模式。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 播放控制仅对已加载或可恢复的曲目生效，暂停仅在播放中执行；控制操作需串行化以避免状态竞争；队列操作需与当前播放索引保持一致；睡眠定时与速度调节在合理区间内取值。 |
| [外部依赖] | 系统多媒体播放能力、系统媒体会话的媒体命令与锁屏/控制中心卡片、文件读取与授权能力、定时与后台运行能力、专辑封面图像资源。 |
| [性能指标] | 起播延迟基线目标小于 300 毫秒；切歌与进度跳转响应及时无明显卡顿。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 播放/暂停/上一曲/下一曲控制生效，恢复播放从此前进度继续，未加载时点击播放能自动恢复上次记忆曲目（覆盖 AR-01-01）；2. 可在列表循环/单曲循环/随机三种模式间切换且切歌行为符合所选模式（覆盖 AR-01-02、AR-01-03）；3. 播放队列可查看、清空、单项移除，并可在队列内切换播放模式（覆盖 AR-01-03）；4. 进度条拖动可定位并跳转到任意位置，播放速度调节即时生效，睡眠定时到点停止且"听完整首再停"开关有效（覆盖 AR-01-04、AR-01-05、AR-01-06）；5. 播放页封面随切歌实时更新、缺失时显示默认占位，快捷入口可触发保持唤醒/沉浸模式/音频信息/跳转专辑与艺术家详情（覆盖 AR-01-07、AR-01-08）。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 车机以系统媒体命令路径触发基础播放控制为主，沉浸模式、迷你快捷入口等富交互能力按车机交互规范裁剪；平板在大屏下可呈现更丰富的播放页与队列布局；各端入口形态差异由 UI 适配层处理。 |
| [视觉/生态/安全等扩展维度] | 后台持续播放需后台运行能力；系统媒体卡片命令回调遵循系统媒体会话生态；封面与快捷入口视觉遵循应用主题与各端设计规范。 |
