# SR-04 歌词显示
- SR-ID: SR-04
- 包含 AR: AR-04-01 加载内嵌/独立 LRC 歌词、AR-04-02 歌词同步滚动、AR-04-03 卡拉OK 逐字歌词、AR-04-04 歌词翻译显示切换、AR-04-05 播放页迷你歌词、AR-04-06 歌词设置

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌词显示 |
| [需求描述] | 围绕本地音乐播放构建完整的歌词显示能力。系统为当前曲目自动加载内嵌或同目录独立 LRC 歌词（AR-04-01），并随播放进度同步滚动与高亮当前行（AR-04-02），在歌词带逐字时间信息时呈现卡拉OK 逐字高亮效果（AR-04-03）。同时支持歌词翻译的显示/隐藏切换（AR-04-04）、播放页内嵌迷你歌词展示（AR-04-05），以及字体大小、居中对齐、字体粗细、视图模糊、隐藏控制面板等歌词外观设置（AR-04-06）。无可用歌词时给出「暂无歌词」友好占位，不影响正常播放。 |
| [需求价值] | 让用户边听歌边看歌词，覆盖内嵌与外挂两类来源、逐字与翻译等多种展示形态，并提供可个性化的歌词外观，显著提升本地听歌的沉浸感与体验完整度。 |
| [需求场景] | 用户进入播放页或歌词界面、切换曲目时，应用自动加载并展示歌词，歌词随进度滚动高亮；用户可开关翻译、调整歌词字体与对齐方式、进入迷你歌词或隐藏控制面板专注看词。 |
| [目标用户] | 希望边听歌边看歌词、并对歌词展示效果有个性化需求的本地音乐用户。 |
| [限制约束] | 仅支持标准 LRC 时间戳格式；内嵌歌词优先于独立文件；逐字效果依赖歌词含逐字时间信息；翻译展示依赖歌词自带翻译文本；无可用歌词时仅显示占位提示而不报错。 |
| [外部依赖] | 音频元数据读取能力、@ohos.file.fs（独立 LRC 文件读取）、文件访问授权、系统媒体播放进度回调。 |
| [性能指标] | 评测基线目标：切歌后歌词加载与首屏渲染 < 300ms；歌词滚动与逐字高亮跟随进度流畅（60fps 目标），解析与渲染不阻塞 UI 主线程。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 含内嵌或同名独立 .lrc 歌词的曲目播放时正确加载并展示歌词，两者均无时显示「暂无歌词」占位且不影响播放；2. 歌词随播放进度同步滚动并高亮当前行，进度拖动后歌词正确跟随定位；3. 歌词含逐字时间信息时呈现卡拉OK 逐字高亮效果；4. 翻译显示开关可正确切换歌词翻译的展示与隐藏；5. 播放页迷你歌词正常展示，歌词外观设置（字体大小/居中对齐/字体粗细/视图模糊/隐藏控制面板）即时生效。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 各端歌词加载与滚动逻辑一致；展示区域大小、迷你歌词与外观排版由各端 UI 适配；车机端以行车安全为前提，弱化交互密集的歌词设置项。 |
| [视觉/生态/安全等扩展维度] | 独立 LRC 文件读取需遵循系统文件访问授权；歌词解析异常需容错不崩溃；视觉上保证高亮与背景对比度，模糊/沉浸等效果不影响歌词可读性。 |
