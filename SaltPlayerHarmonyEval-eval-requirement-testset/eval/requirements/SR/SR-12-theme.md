# SR-12 主题与外观个性化
- SR-ID: SR-12
- 包含 AR: AR-12-01 主题模式、AR-12-02 列表显示歌曲封面开关、AR-12-03 主页/播放页壁纸、AR-12-04 圆形播放封面、AR-12-05 流光效果、AR-12-06 播放时保持屏幕常亮、AR-12-07 沉浸模式

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 主题与外观个性化 |
| [需求描述] | 本需求聚合应用界面的主题与外观个性化能力。通过 AR-12-01 提供"跟随系统/浅色/深色"三种主题模式；通过 AR-12-02 控制列表是否显示歌曲封面；通过 AR-12-03 设置主页与播放页壁纸；通过 AR-12-04 将播放封面以圆形展示；通过 AR-12-05 提供界面流光视觉效果；通过 AR-12-06 在播放期间保持屏幕常亮；通过 AR-12-07 进入沉浸模式以隐藏状态栏与控制面板、仅保留封面/歌词等核心视觉。各项设置即时生效并在重启后保持，让用户按个人偏好定制界面观感。 |
| [需求价值] | 让用户按个人审美与使用环境定制界面外观，提升视觉个性化与使用沉浸感、阅读舒适度。 |
| [需求场景] | 用户在外观/主题设置中切换深浅色主题、开关列表封面、设置主页/播放页壁纸、开启圆形封面与流光效果、开启播放常亮；在播放页进入/退出沉浸模式以专注于封面与歌词。 |
| [目标用户] | 注重界面个性化、夜间护眼与沉浸式听歌体验的用户。 |
| [限制约束] | 固定浅色/深色时不再跟随系统，主题切换需作用于全应用界面避免局部遗漏；沉浸模式（AR-12-07）与播放页快捷操作入口（AR-01-08）中的沉浸模式为同一能力的两处入口，应保持状态一致、互为联动，避免两处开关相互矛盾；播放常亮仅在播放期间生效，暂停或退出应释放，避免持续耗电。 |
| [外部依赖] | 系统深浅色配置能力（ConfigurationConstant/应用配色）、系统屏幕常亮/窗口能力、本地设置与壁纸资源持久化、当前歌曲封面数据源。 |
| [性能指标] | 评测基线目标：切换主题/壁纸/外观开关后界面在 300ms 内完成重绘，无明显闪烁。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 提供"跟随系统/浅色/深色"主题模式且单选生效、重启后保持；2. 列表显示歌曲封面开关、圆形播放封面、流光效果各开关即时生效；3. 可设置主页与播放页壁纸并正确展示；4. 开启播放常亮后播放期间屏幕不熄灭、暂停或退出后恢复；5. 可进入/退出沉浸模式，沉浸时隐藏状态栏与控制面板仅保留核心视觉，且与播放页快捷入口处的沉浸模式状态保持一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 各端均支持深浅色与基础外观设置，界面布局由各端适配；车机/平板深浅色以其系统策略为准，壁纸与流光等装饰能力在车机可能受系统规范限制而降级或不适用。 |
| [视觉/生态/安全等扩展维度] | 深浅色与壁纸配色需满足对比度与可读性，遵循系统主题生态；流光与圆形封面等视觉效果需保证性能与无障碍可读性，沉浸模式下仍需保留必要的退出与交互可达性。 |
