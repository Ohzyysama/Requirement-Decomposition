# SR-10 播放统计
- SR-ID: SR-10
- 包含 AR: AR-10-01 播放次数统计

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放统计 |
| [需求描述] | 本需求提供歌曲播放行为的统计能力，帮助用户回顾和发现自己的高频聆听内容。播放次数统计对歌曲的播放次数进行累计记录，并以歌曲列表形式展示统计结果（AR-10-01），用户可据此查看最常播放的歌曲。该能力将隐性的播放行为转化为可浏览的统计视图，为用户提供基于真实聆听习惯的歌曲入口。 |
| [需求价值] | 将用户的播放行为沉淀为可见的统计数据，帮助用户回顾高频歌曲、发现个人聆听偏好，提供基于使用习惯的快捷播放入口。 |
| [需求场景] | 用户进入播放统计查看按播放次数统计的歌曲列表，了解自己最常播放的歌曲，并可直接从列表中点击播放。 |
| [目标用户] | 关注个人聆听习惯、希望快速回到高频歌曲的用户。 |
| [限制约束] | 统计范围限定于已扫描入库的本地歌曲；播放次数累计规则（计入时机/去重策略）依实现；统计数据存储于本地。 |
| [外部依赖] | 本地媒体扫描入库数据、本地播放次数记录存储、播放控制域的播放事件。 |
| [性能指标] | 评测基线目标：播放统计列表即时展示（<100ms 量级），加载与滚动 ≥ 60fps；播放次数累计不阻塞播放主流程。 |
| [功耗指标] | 评测基线目标：息屏后台纯音频每小时电量 < 3%。 |
| [ROM&RAM] | 评测基线目标：安装包 < 100MB；运行内存 < 400MB。 |
| [验收标准] | 1. 应用对歌曲的播放次数进行累计统计。 2. 播放统计结果以歌曲列表形式展示。 3. 可从统计列表直接点击播放歌曲。 4. 统计数据基于本地播放行为，准确反映真实聆听次数。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机（部分能力）。 |
| [适用产品差异分析] | 播放次数统计为通用能力，手机/平板完整展示统计列表；车机以快捷播放高频歌曲为主，可简化统计展示，避免驾驶场景过多信息浏览。 |
| [视觉/生态/安全等扩展维度] | 视觉：统计结果沿用歌曲列表视觉规范，可附带播放次数标识；安全：播放次数记录仅存储于本地，不涉及网络上传。 |
