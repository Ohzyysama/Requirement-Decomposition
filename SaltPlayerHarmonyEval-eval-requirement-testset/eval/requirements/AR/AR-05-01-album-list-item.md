# AR-05-01 专辑列表项展示（封面/专辑名/歌曲数）
- AR-ID: AR-05-01
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑列表项展示（封面/专辑名/歌曲数） |
| [需求描述] | 专辑列表以网格或列表形式展示本地媒体库中的所有专辑，每个专辑项展示专辑封面、专辑名称以及该专辑包含的歌曲数量。封面取自专辑内歌曲的内嵌封面或缓存封面，缺失时显示占位图。点击专辑项进入对应的专辑详情页。 |
| [需求价值] | 让用户以专辑维度浏览本地音乐，通过封面与专辑名快速辨认并进入专辑，提升整张专辑的浏览与播放体验。 |
| [需求场景] | 用户进入专辑页，滚动浏览各专辑封面与名称，依据歌曲数判断专辑规模，点击某专辑进入其详情页。 |
| [目标用户] | 以专辑维度浏览本地音乐的普通用户。 |
| [限制约束] | 封面缺失时显示占位图；歌曲数随媒体库扫描结果动态统计；仅展示本地已入库专辑。 |
| [外部依赖] | 专辑封面缓存与封面提取、媒体库扫描入库数据（专辑聚合与歌曲计数）。 |
| [性能指标] | 评测基线目标：列表/网格滚动 ≥ 60fps；封面异步加载不阻塞滚动。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 专辑列表正确展示媒体库中所有专辑项。 2. 每个专辑项显示封面（缺失时显示占位图）、专辑名称与歌曲数量。 3. 歌曲数与该专辑实际入库歌曲数一致。 4. 点击专辑项进入对应专辑详情页。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 专辑项展示为通用能力，各形态一致；平板/车机可借更大屏幕展示更多列。 |
| [视觉/生态/安全等扩展维度] | 视觉：封面为主视觉，专辑名与歌曲数辅助信息；安全：仅展示本地已扫描专辑信息。 |
