# AR-12-02 列表显示歌曲封面开关
- AR-ID: AR-12-02
- 所属 SR: SR-12 主题与外观个性化

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 列表显示歌曲封面开关 |
| [需求描述] | 在外观设置中提供一个开关，控制歌曲列表项是否显示歌曲封面缩略图。开启时列表每一项左侧展示对应封面；关闭时隐藏封面，仅显示文字信息以获得更紧凑的列表。设置切换后立即作用于各歌曲列表，并在重启后保持。 |
| [需求价值] | 兼顾视觉丰富度与信息密度，让用户按喜好选择带封面或精简的列表样式。 |
| [需求场景] | 用户在外观设置打开/关闭"列表显示歌曲封面"开关；返回歌曲列表查看封面显示效果。 |
| [目标用户] | 偏好精简列表或偏好可视化封面的全体用户。 |
| [限制约束] | 开关作用于歌曲列表项封面；关闭时不应影响播放页等其他位置的封面展示。 |
| [外部依赖] | 封面图加载能力、本地设置持久化。 |
| [性能指标] | 基线目标：切换开关后列表刷新无明显卡顿；关闭后减少封面加载开销。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 外观设置中存在"列表显示歌曲封面"开关；2. 开启时歌曲列表项显示封面缩略图；3. 关闭时列表项隐藏封面、仅显示文字；4. 开关状态在应用重启后保持。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端列表项布局由对应分辨率适配，封面显隐逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 封面缺失时使用占位图，保证列表视觉一致性。 |
