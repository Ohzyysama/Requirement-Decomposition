# AR-01-07 播放页展示专辑封面
- AR-ID: AR-01-07
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放页展示专辑封面 |
| [需求描述] | 播放页主视觉区域展示当前播放歌曲所属专辑的封面图片。封面随切歌实时更新；当歌曲无内嵌封面或封面缺失时，显示统一的默认占位封面。封面作为播放页的核心视觉元素，可结合外观设置（如圆形封面、流光等）呈现。 |
| [需求价值] | 以专辑封面强化听歌时的视觉沉浸感，帮助用户直观确认当前播放曲目。 |
| [需求场景] | 用户进入播放页，看到大尺寸专辑封面；切换上一曲/下一曲时封面随之更新；无封面歌曲显示默认占位图。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 封面来源于歌曲内嵌或封面缓存，缺失时回退默认占位图；封面展示样式受外观个性化设置影响。 |
| [外部依赖] | 封面提取与封面缓存（封面图来源）、当前播放歌曲信息。 |
| [性能指标] | 评测基线目标：切歌后封面更新 < 200ms；封面异步加载不阻塞播放页交互。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 播放页展示当前歌曲所属专辑的封面图。 2. 切换歌曲时封面实时更新为新曲封面。 3. 歌曲无封面或封面缺失时显示统一默认占位封面。 4. 封面展示与外观设置（圆形封面等）协同生效。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 封面展示为通用能力；平板/车机因屏幕尺寸采用更大封面布局。 |
| [视觉/生态/安全等扩展维度] | 视觉：大尺寸封面为播放页核心视觉，可叠加圆形/流光效果；安全：仅展示本地歌曲封面，不联网拉取。 |
