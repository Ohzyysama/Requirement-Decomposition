# AR-05-03 专辑列表设置（排序: 标题/年份/数量；展示列数: 2/3/4）
- AR-ID: AR-05-03
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑列表设置（排序: 标题/年份/数量；展示列数: 2/3/4） |
| [需求描述] | 专辑列表提供展示设置入口，用户可选择排序维度，包括按标题、按年份、按歌曲数量排序；同时可选择网格展示列数，支持 2 列、3 列、4 列三档。设置变更后专辑列表立即按新规则重新排列与布局，所选设置在后续进入页面时保持生效。 |
| [需求价值] | 让用户按自己偏好组织与查看专辑，灵活调整排序维度与展示密度，兼顾辨识度与浏览效率。 |
| [需求场景] | 用户打开专辑列表设置，将排序改为「年份」并将展示列数设为 3 列，列表随即按年份排序并以三列网格重新布局。 |
| [目标用户] | 希望自定义专辑浏览方式的本地音乐用户。 |
| [限制约束] | 排序维度限定为标题/年份/数量三种；展示列数限定为 2/3/4 三档；年份缺失的专辑归入约定位置排序。 |
| [外部依赖] | 专辑列表数据与设置持久化存储（保存排序与列数偏好）。 |
| [性能指标] | 评测基线目标：切换排序/列数后重排与重布局响应及时，滚动 ≥ 60fps。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 提供按标题、年份、数量三种排序选项，选择后列表按对应维度重排。 2. 提供 2/3/4 三档展示列数，选择后网格按对应列数重新布局。 3. 设置变更立即生效。 4. 所选排序与列数在再次进入专辑列表时保持。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 排序与列数设置为通用能力；平板/车机大屏下可结合自适应布局展示更多列。 |
| [视觉/生态/安全等扩展维度] | 视觉：列数变化影响封面尺寸与网格密度；交互：设置即时反馈并持久保存。 |
