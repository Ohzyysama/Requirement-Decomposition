# AR-12-05 流光效果
- AR-ID: AR-12-05
- 所属 SR: SR-12 主题与外观个性化

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 流光效果 |
| [需求描述] | 提供开关启用界面流光视觉效果，通常基于当前封面取色在播放页等位置呈现流动的光晕/渐变动效。开启后界面随封面色彩展现动态流光氛围，关闭则恢复静态背景。设置切换即时生效并在重启后保持。 |
| [需求价值] | 通过动态光效增强播放氛围与视觉沉浸感，提升外观个性化体验。 |
| [需求场景] | 用户在外观设置中开启"流光效果"，进入播放页查看随封面色彩呈现的流动光效。 |
| [目标用户] | 追求动态视觉氛围与沉浸式体验的用户。 |
| [限制约束] | 流光为视觉氛围层，不应遮挡或影响前景信息可读性；需控制动效开销避免明显发热卡顿。 |
| [外部依赖] | 封面取色能力、动效/渲染能力、本地设置持久化。 |
| [性能指标] | 基线目标：开启流光后界面保持流畅，动效不引起明显掉帧。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 外观设置中存在"流光效果"开关；2. 开启时界面呈现基于封面色彩的流动光效；3. 关闭时恢复静态背景、无流光动效；4. 设置在应用重启后保持。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 低性能或车机场景可降级动效强度以保证流畅，效果开关逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 动效需兼顾观感与功耗，避免在前景文字区域造成可读性下降。 |
