# AR-11-04 通知栏歌词
- AR-ID: AR-11-04
- 所属 SR: SR-11 桌面与通知歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 通知栏歌词 |
| [需求描述] | 在系统媒体通知中展示当前播放曲目的实时歌词。下拉通知栏或在锁屏媒体通知上，除常规的曲目信息与播放控制外，还呈现随播放进度更新的歌词行。切歌时歌词与曲目信息一并刷新，关闭播放或停止后歌词随通知一并消失。 |
| [需求价值] | 复用用户高频查看的通知栏入口呈现歌词，无需打开应用即可边操作媒体控制边看歌词。 |
| [需求场景] | 用户下拉通知栏查看媒体通知，看到当前歌词随进度更新；切歌时通知歌词与封面/标题同步刷新；锁屏媒体通知同样展示歌词。 |
| [目标用户] | 习惯通过通知栏查看播放信息与歌词的用户。 |
| [限制约束] | 依赖系统媒体通知通道；通知区域空间有限，展示当前歌词行；无歌词数据时仅显示常规曲目信息。 |
| [外部依赖] | @ohos.multimedia.avsession（媒体会话与通知歌词数据）、系统通知服务。 |
| [性能指标] | 基线目标：通知歌词随进度刷新延迟<300ms；切歌后通知内容同步刷新<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 媒体通知中展示当前曲目歌词，并随播放进度逐行更新；2. 切歌时通知中的歌词与曲目信息一并刷新；3. 锁屏媒体通知同样展示实时歌词；4. 停止播放或关闭媒体会话后通知歌词随通知一并消失；5. 无歌词数据时通知正常显示曲目信息而不报错。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 通知歌词依赖各端媒体通知形态，平板按通知布局适配文本，车机经系统媒体通知/中控展示策略适配。 |
| [视觉/生态/安全等扩展维度] | 通知样式与媒体控件遵循系统媒体通知生态规范，歌词文本不破坏通知标准布局。 |
