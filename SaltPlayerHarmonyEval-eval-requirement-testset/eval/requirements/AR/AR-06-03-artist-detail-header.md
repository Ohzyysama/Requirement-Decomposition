# AR-06-03 艺术家详情头部展示（头像+艺术家名）
- AR-ID: AR-06-03
- 所属 SR: SR-06 艺术家管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 艺术家详情头部展示（头像+艺术家名） |
| [需求描述] | 进入某位艺术家的详情页后，页面头部展示该艺术家的头像与艺术家名。头像无图时使用默认占位图，艺术家名以醒目标题样式呈现，作为详情页的标识区域。 |
| [需求价值] | 让用户进入艺术家详情后清晰确认当前所在艺术家，提供页面身份标识与视觉锚点。 |
| [需求场景] | 用户从艺术家列表点击进入详情页，页面顶部即展示该艺术家头像与名称，确认浏览对象无误。 |
| [目标用户] | 查看某位艺术家详情的普通用户。 |
| [限制约束] | 头像缺失时显示默认占位图；头部仅展示头像与艺术家名，不承载操作逻辑。 |
| [外部依赖] | 头像缓存与头像提取（头像来源）、艺术家基础信息数据。 |
| [性能指标] | 评测基线目标：详情页打开头部即时渲染；头像异步加载不阻塞页面展示。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 详情页头部正确展示当前艺术家头像，无头像时显示默认占位图。 2. 头部以标题样式正确展示艺术家名。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 详情头部为通用展示能力，各形态一致；平板/车机可利用更大空间放大头部区域。 |
| [视觉/生态/安全等扩展维度] | 视觉：大尺寸头像与醒目艺术家名标题；安全：仅展示本地已扫描艺术家信息。 |
