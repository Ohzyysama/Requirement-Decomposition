# AR-07-01 文件夹列表展示（展示所有本地歌曲文件夹）
- AR-ID: AR-07-01
- 所属 SR: SR-07 文件夹与存储管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹列表展示（展示所有本地歌曲文件夹） |
| [需求描述] | 文件夹页以列表形式展示设备上所有包含本地歌曲的文件夹，每个文件夹项展示文件夹名称、所在路径以及该文件夹下的歌曲数量。列表按文件夹归类汇总扫描入库的歌曲，方便用户从存储结构维度浏览音乐。点击某个文件夹可进入对应的文件夹详情页查看该文件夹内的歌曲。 |
| [需求价值] | 让用户从文件存储结构的维度组织和查找音乐，便于按目录定位某一来源或某一批次导入的歌曲。 |
| [需求场景] | 用户进入文件夹页，浏览所有含歌曲的本地文件夹及其歌曲数量，点击某个文件夹进入其详情页。 |
| [目标用户] | 习惯按文件夹/目录结构管理本地音乐的用户。 |
| [限制约束] | 仅展示已扫描入库且包含歌曲的文件夹；被屏蔽或隐藏的文件夹不在列表中展示；文件夹信息来源于本地扫描结果。 |
| [外部依赖] | 本地媒体扫描入库数据、文件夹与歌曲归类信息。 |
| [性能指标] | 评测基线目标：文件夹列表加载与滚动 ≥ 60fps，进入页面即时展示（<100ms 量级）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 文件夹页以列表展示所有包含本地歌曲的文件夹。 2. 每个文件夹项显示文件夹名称、路径与歌曲数量。 3. 点击某文件夹可进入对应文件夹详情页。 4. 被隐藏/屏蔽的文件夹不出现在列表中。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 文件夹列表为通用能力，各形态一致；车机以更大字号与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：文件夹名为主标题、路径为副标题、右侧显示歌曲数量；安全：仅展示本地已扫描歌曲所在文件夹信息。 |
