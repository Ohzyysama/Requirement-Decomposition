# AR-01-06 睡眠定时（设置时长 + 自动延长到整首歌播放完毕开关）
- AR-ID: AR-01-06
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 睡眠定时（设置时长 + 自动延长到整首歌播放完毕开关） |
| [需求描述] | 提供睡眠定时功能，用户设置一段定时时长后，到时自动停止播放。同时提供「自动延长到整首歌播放完毕」开关：开启时，若定时到点处于歌曲播放中途，则不立即停止，而是等待当前这首歌自然播放完毕后再停止，避免在歌曲中途被打断。 |
| [需求价值] | 满足用户睡前听歌的自动停止需求，并以"听完整首"的体验优化避免突兀中断。 |
| [需求场景] | 用户打开睡眠定时，设置时长（并可开启自动延长到整首歌播放完毕开关）；到时后系统按设置自动停止播放。 |
| [目标用户] | 有睡前/定时收听需求的用户。 |
| [限制约束] | 定时需在后台亦能可靠触发；自动延长仅延后到当前曲目结束，不无限延长。 |
| [外部依赖] | 系统定时/计时能力、系统多媒体播放能力、本地偏好存储（定时设置）。 |
| [性能指标] | 定时触发时机准确，误差控制在可接受范围内。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 用户设置定时时长后，到点时自动停止播放；2. 开启「自动延长到整首歌播放完毕」开关且到点时正处于歌曲中途，则等待当前歌曲播放完毕后再停止；3. 关闭该开关时，到点即停止，不等待当前歌曲结束；4. 定时设置在界面有明确显示，并支持取消/修改；5. 应用退到后台时定时仍能可靠触发。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 睡眠定时以手机/平板睡前场景为主；车机场景使用较少但逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 后台到时停止需依赖后台运行能力，确保定时不被提前回收。 |
