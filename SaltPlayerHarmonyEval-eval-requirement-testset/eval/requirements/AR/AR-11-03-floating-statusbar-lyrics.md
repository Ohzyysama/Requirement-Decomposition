# AR-11-03 悬浮窗状态栏歌词
- AR-ID: AR-11-03
- 所属 SR: SR-11 桌面与通知歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 悬浮窗状态栏歌词 |
| [需求描述] | 在系统状态栏区域以悬浮窗形式展示当前播放曲目的实时歌词。开启后状态栏常驻显示一行随播放进度滚动的歌词文本，用户在使用其它应用或回到桌面时无需切回播放器即可看到歌词。关闭后悬浮窗歌词隐藏，状态栏恢复原状。 |
| [需求价值] | 在跨应用、息屏外的日常使用中持续呈现歌词，进一步降低用户查看歌词的成本。 |
| [需求场景] | 用户开启悬浮窗状态栏歌词后切换到其它应用，状态栏仍显示当前歌词并随播放推进更新；切歌时同步切换；关闭后悬浮窗消失。 |
| [目标用户] | 希望在任意界面状态栏处随时看到歌词的用户。 |
| [限制约束] | 依赖系统悬浮窗权限；状态栏空间有限，单次仅展示当前一行歌词；无歌词时不展示内容。 |
| [外部依赖] | 系统悬浮窗/窗口服务与悬浮窗权限、@ohos.multimedia.avsession（当前歌词数据）。 |
| [性能指标] | 基线目标：歌词随进度刷新延迟<200ms；悬浮窗显示/隐藏切换<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 开启后状态栏悬浮窗显示当前曲目歌词，并随播放进度逐行更新；2. 切换到其它应用或桌面时悬浮窗歌词仍持续展示与更新；3. 切歌时悬浮窗歌词同步切换为新曲目；4. 关闭后悬浮窗歌词隐藏，状态栏恢复原状；5. 未授予悬浮窗权限时给出引导且不强制展示。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 悬浮窗依赖各端状态栏与窗口能力，平板按可用宽度适配文本长度，车机按系统策略决定是否允许常驻悬浮歌词。 |
| [视觉/生态/安全等扩展维度] | 悬浮窗层级、避让与权限申请遵循系统悬浮窗生态规范，避免遮挡系统关键信息。 |
