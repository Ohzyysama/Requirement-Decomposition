# AR-08-01 创建歌单
- AR-ID: AR-08-01
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 创建歌单 |
| [需求描述] | 提供新建歌单的入口，用户可输入歌单名称创建一个空歌单。创建时对名称做基本校验，例如不允许空名称、对重名给出提示。创建成功后新歌单立即出现在歌单列表中，可用于后续添加歌曲。 |
| [需求价值] | 让用户按自己的喜好对歌曲进行分组归类，是歌单管理的入口与基础。 |
| [需求场景] | 用户在歌单页点击「新建歌单」，在弹窗中输入名称并确认，列表随即出现新歌单。 |
| [目标用户] | 有歌曲整理与分类需求的听歌用户。 |
| [限制约束] | 歌单名称不可为空；同名歌单需给出重名提示或做去重处理；名称长度受合理上限约束。 |
| [外部依赖] | 本地歌单持久化存储（应用首选项/数据库）。 |
| [性能指标] | 创建并刷新列表的响应时间<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 点击新建入口可弹出名称输入框；2. 输入合法名称并确认后，新空歌单立即出现在歌单列表中；3. 输入空名称时无法创建并有提示；4. 创建重名歌单时给出重名提示或合理去重处理。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端创建入口与弹窗形态随屏幕尺寸适配，核心创建逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 歌单数据存储于本地，遵循应用数据隔离与持久化规范。 |
