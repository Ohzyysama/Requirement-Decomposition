# AR-03-08 歌曲信息详情展示
- AR-ID: AR-03-08
- 所属 SR: SR-03 歌曲管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲信息详情展示（标题/艺术家/专辑/专辑艺术家/时长/比特率/声道/采样率/位深/大小/格式/编码/版权/添加与修改时间/路径） |
| [需求描述] | 歌曲信息详情页集中展示一首歌曲的完整元数据，包括标题、艺术家、专辑、专辑艺术家、时长、比特率、声道、采样率、位深、文件大小、格式、编码、版权、添加时间与修改时间、文件路径等字段。各字段按已扫描入库的元信息如实呈现，缺失字段以空缺或占位形式合理展示。 |
| [需求价值] | 让用户全面了解歌曲的音频规格与文件信息，便于辨别音质、来源与文件位置。 |
| [需求场景] | 用户在歌曲更多菜单点击「查看歌曲信息」，进入详情页查看该曲的比特率、采样率、位深、大小与文件路径等。 |
| [目标用户] | 关注音频规格、音质与文件来源的进阶用户。 |
| [限制约束] | 展示内容来自已扫描入库的歌曲元信息；部分字段可能因源文件缺失而为空；详情为只读展示。 |
| [外部依赖] | 媒体库扫描数据（元信息来源）、歌曲更多操作菜单入口（AR-03-07）。 |
| [性能指标] | 评测基线目标：打开详情页即时加载并完整呈现可用字段。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 详情页展示标题、艺术家、专辑、专辑艺术家、时长、比特率、声道、采样率、位深、大小、格式、编码、版权、添加与修改时间、路径等字段。 2. 各字段取值与扫描入库的元信息一致。 3. 缺失字段以空缺或占位形式合理展示，不报错。 4. 详情页为只读展示，可正常返回列表。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 信息详情为通用展示能力，各形态一致；车机以更大字号呈现关键字段、弱化次要信息。 |
| [视觉/生态/安全等扩展维度] | 视觉：字段以标签-取值成对清晰排布；安全：仅展示本地文件元信息，路径等敏感信息仅本地呈现不外传。 |
