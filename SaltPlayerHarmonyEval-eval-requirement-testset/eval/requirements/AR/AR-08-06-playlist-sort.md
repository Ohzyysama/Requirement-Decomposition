# AR-08-06 歌单排序
- AR-ID: AR-08-06
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌单排序 |
| [需求描述] | 支持调整歌单在列表中的排列顺序，便于用户把常用歌单置于显眼位置。可按名称、创建/修改时间等维度排序，或支持用户自定义拖动调整顺序。排序结果被持久化，再次进入页面时保持。 |
| [需求价值] | 让用户按使用习惯组织歌单顺序，提升查找与使用效率。 |
| [需求场景] | 用户在歌单页选择排序方式或拖动调整歌单顺序，列表按新顺序展示并保持。 |
| [目标用户] | 拥有多个歌单、有排序整理需求的听歌用户。 |
| [限制约束] | 排序方式与自定义顺序需持久化；自定义拖动顺序与按维度排序之间切换逻辑需清晰。 |
| [外部依赖] | 本地歌单持久化存储（应用首选项/数据库）。 |
| [性能指标] | 切换排序并刷新列表的响应时间<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 可选择按名称、时间等维度对歌单列表排序，列表即时按所选维度重排；2. 支持自定义拖动调整歌单顺序（如提供该方式）；3. 排序方式或自定义顺序被持久化，再次进入页面时保持；4. 切换排序时列表展示与底层数据顺序一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端排序入口与拖动交互随输入方式适配，核心排序与持久化逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 排序偏好存储于本地，遵循应用数据持久化规范。 |
