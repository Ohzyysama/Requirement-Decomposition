# AR-07-05 文件夹详情-歌曲列表排序
- AR-ID: AR-07-05
- 所属 SR: SR-07 文件夹与存储管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹详情-歌曲列表排序 |
| [需求描述] | 在文件夹详情页面内，用户可选择该文件夹歌曲列表的排序方式，支持按标题、文件名、专辑、艺术家、文件大小、年份、时长、修改时间、添加时间、播放次数等多个维度排序，部分维度提供升序与降序两种选择。默认按歌曲标题升序排列，中文歌曲按拼音顺序排序。用户所选排序方式会被保存，下次进入仍保持该排序。 |
| [需求价值] | 让用户在文件夹详情页内按自己的习惯组织该文件夹的歌曲列表，快速找到目标歌曲。 |
| [需求场景] | 用户在某文件夹详情页打开排序菜单，选择「按艺术家」或「按添加时间倒序」，该文件夹歌曲列表立即重排并在下次进入时保持。 |
| [目标用户] | 需要在文件夹内按特定维度组织与查找歌曲的用户。 |
| [限制约束] | 排序基于该文件夹下已扫描入库的歌曲数据；中文按拼音排序依赖系统转写能力；所选排序方式持久化。 |
| [外部依赖] | 本地已扫描歌曲数据、系统拼音转写与排序能力、设置持久化存储。 |
| [性能指标] | 评测基线目标：切换排序后列表重排即时生效 < 100ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 文件夹详情页内可按标题、文件名、专辑、艺术家、大小、年份、时长、修改时间、添加时间、播放次数等维度排序，部分维度可选升降序。 2. 默认按标题升序，中文按拼音排序。 3. 切换排序方式后列表立即按新方式重排。 4. 所选排序方式被保存，下次进入仍保持。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 排序为通用能力，各形态一致；车机以常用排序维度的简化选择为主。 |
| [视觉/生态/安全等扩展维度] | 生态：复用系统拼音排序与本地化能力；安全：排序仅作用于本地已扫描数据。 |
