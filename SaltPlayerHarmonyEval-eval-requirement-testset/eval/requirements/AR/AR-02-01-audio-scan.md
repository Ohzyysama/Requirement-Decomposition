# AR-02-01 音频扫描
- AR-ID: AR-02-01
- 所属 SR: SR-02 媒体库与本地扫描

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音频扫描 |
| [需求描述] | 应用扫描设备本地存储中的音频文件并将其入库到媒体库。扫描过程会读取每个文件的元数据（如标题、艺术家、专辑、时长等）并建立索引，供后续浏览与播放使用。首次启动或获得存储权限后即可触发扫描，扫描完成后媒体库呈现可浏览的歌曲、专辑、艺术家等数据。 |
| [需求价值] | 让用户的本地音乐自动汇入媒体库，是本地音乐播放体验的数据基础。 |
| [需求场景] | 用户首次进入应用或授予存储访问权限后，由系统发起本地音频文件的扫描与入库。 |
| [目标用户] | 拥有本地音乐文件的听歌用户。 |
| [限制约束] | 需先获得存储/媒体读取权限；仅扫描受支持的音频格式；扫描在后台进行避免阻塞主界面交互。 |
| [外部依赖] | 系统媒体库与文件访问能力（媒体元数据读取、文件系统遍历）、存储访问权限。 |
| [性能指标] | 基线目标：常规规模曲库（约千首）首次扫描入库可在可接受时间内完成且界面不卡顿。 |
| [功耗指标] | 继承所属 SR |
| [ROM&RAM] | 继承所属 SR |
| [验收标准] | 1. 在获得存储权限后触发扫描，本地受支持音频文件被识别并入库；2. 入库歌曲在歌曲、专辑、艺术家等列表中正确呈现且元数据完整；3. 扫描过程中主界面可正常操作，不发生明显卡顿；4. 扫描完成后媒体库数量与本地实际可识别音频文件一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线 |
| [适用产品] | 手机 / 平板 / 车机 |
| [适用产品差异分析] | 各端本地存储与媒体目录结构可能不同，由系统媒体能力屏蔽差异；车机侧扫描入口形态由各端 UI 适配。 |
| [视觉/生态/安全等扩展维度] | 访问本地媒体需遵循系统权限与隐私规范，仅在授权范围内读取文件。 |
