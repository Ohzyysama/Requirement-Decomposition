# AR-07-04 文件夹详情-歌曲列表项展示（封面/音质/歌名/艺术家）
- AR-ID: AR-07-04
- 所属 SR: SR-07 文件夹与存储管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹详情-歌曲列表项展示（封面/音质/歌名/艺术家） |
| [需求描述] | 在文件夹详情页面内，该文件夹下的每首歌以一行列表项展示：左侧显示歌曲封面（是否显示受「列表显示封面」开关控制，关闭时不占位）；歌名旁可显示音质徽标，分为高品质、超高品质、超高解析度三种标识，普通音质不显示徽标；主标题默认显示歌名，开启「显示文件名」开关时改为显示文件名；副标题显示艺术家，缺失时回退为文件名。正在播放的歌曲标题与副标题高亮显示，失效歌曲文字置灰。每行右侧默认有「更多」按钮，进入多选模式后改为勾选框。 |
| [需求价值] | 让用户在文件夹详情页内一眼获取每首歌的封面、名称、艺术家与音质信息，并识别当前播放与失效歌曲，提升浏览效率。 |
| [需求场景] | 用户进入某个文件夹详情页，滚动浏览该文件夹内歌曲，通过封面、歌名、音质徽标快速辨认，并看到正在播放的歌曲被高亮、失效歌曲被置灰。 |
| [目标用户] | 浏览某个文件夹内本地歌曲的普通用户。 |
| [限制约束] | 普通音质不显示徽标；封面与文件名显示受对应开关控制；多选模式下「更多」按钮替换为勾选框；失效歌曲仅置灰展示不阻断点击。 |
| [外部依赖] | 封面缓存与封面提取、列表显示偏好开关（封面/文件名）、本地已扫描歌曲数据。 |
| [性能指标] | 评测基线目标：列表滚动 ≥ 60fps；封面异步加载不阻塞滚动。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 文件夹详情页内每行正确显示封面（开关关闭时不显示）、歌名（开关开启时显示文件名）、艺术家副标题，必要时副标题回退为文件名。 2. 高品质/超高品质/超高解析度歌曲显示对应徽标，普通音质无徽标。 3. 正在播放的歌曲标题与副标题高亮显示，失效歌曲文字置灰。 4. 默认显示「更多」按钮，进入多选模式后该位置改为勾选框并反映选中状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 列表项展示为通用能力，各形态一致；车机以更大字号与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：高亮色标识当前播放、灰色标识失效、音质徽标小图标；安全：仅展示本地已扫描歌曲信息。 |
