# AR-02-04 过滤 60 秒以下短音频
- AR-ID: AR-02-04
- 所属 SR: SR-02 媒体库与本地扫描

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 过滤 60 秒以下短音频 |
| [需求描述] | 在扫描入库时，应用自动过滤时长低于 60 秒的短音频文件，不将其纳入媒体库。此举用于排除铃声、提示音、通知音等通常并非歌曲的短小音频。该过滤在扫描与重新扫描时均生效，使媒体库中呈现的内容以正常时长的歌曲为主。 |
| [需求价值] | 自动剔除铃声与提示音等短音频，减少噪声条目，让媒体库聚焦于真正的音乐内容。 |
| [需求场景] | 扫描/重新扫描过程中，对每个候选音频按时长判定，时长低于 60 秒的不入库。 |
| [目标用户] | 希望媒体库只包含正常歌曲、避免短音频干扰的用户。 |
| [限制约束] | 以 60 秒为时长阈值，低于该阈值的音频被排除；时长依赖元数据可正确读取。 |
| [外部依赖] | 系统媒体元数据读取能力（获取音频时长）。 |
| [性能指标] | 基线目标：时长判定不显著增加单文件扫描耗时。 |
| [功耗指标] | 继承所属 SR |
| [ROM&RAM] | 继承所属 SR |
| [验收标准] | 1. 时长低于 60 秒的音频在扫描后不出现在媒体库中；2. 时长达到或超过 60 秒的音频正常入库；3. 重新扫描时该过滤规则同样生效；4. 过滤后媒体库不再包含明显的铃声/提示音类短音频。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线 |
| [适用产品] | 手机 / 平板 / 车机 |
| [适用产品差异分析] | 各端时长元数据读取由系统媒体能力统一提供，过滤逻辑一致，无端侧差异。 |
| [视觉/生态/安全等扩展维度] | 过滤为入库阶段的自动行为，不涉及额外权限或隐私数据。 |
