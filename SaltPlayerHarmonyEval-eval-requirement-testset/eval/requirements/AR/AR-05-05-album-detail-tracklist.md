# AR-05-05 专辑详情歌曲列表（曲目项展示/点击播放/添加下一首/更多操作菜单）
- AR-ID: AR-05-05
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑详情歌曲列表（曲目项展示/点击播放/添加下一首/更多操作菜单） |
| [需求描述] | 专辑详情页展示该专辑下的歌曲列表，每个曲目项展示歌曲信息（如曲目号/歌名/艺术家等）。用户点击曲目项即播放该曲，曲目项提供「添加到下一首播放」入口，并提供「更多」操作菜单（添加到歌单、下一首播放、分享、跳转艺术家、跳转专辑、查看歌曲信息、从音乐库移除等）。正在播放的曲目以高亮标识。 |
| [需求价值] | 让用户在专辑维度下直接浏览并播放专辑内歌曲，并对单曲执行常用操作，形成完整的专辑收听与管理闭环。 |
| [需求场景] | 用户在专辑详情页点击某曲目开始播放，或点击「更多」打开操作菜单将该曲添加到歌单、设为下一首播放或查看歌曲信息。 |
| [目标用户] | 在专辑维度下收听与管理歌曲的本地音乐用户。 |
| [限制约束] | 更多菜单中跳转艺术家在多艺术家时需先选择艺术家；失效歌曲置灰展示；菜单项依赖对应能力可用性。 |
| [外部依赖] | 播放控制与队列（点击播放/下一首播放）、歌单管理（添加到歌单）、系统分享、歌曲信息详情、媒体库移除能力。 |
| [性能指标] | 评测基线目标：曲目列表滚动 ≥ 60fps；点击播放响应及时。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 专辑详情页正确展示该专辑下所有曲目项。 2. 点击曲目项即播放该曲，正在播放曲目高亮标识。 3. 曲目项提供「添加到下一首播放」入口并生效。 4. 「更多」菜单提供添加到歌单、下一首播放、分享、跳转艺术家、跳转专辑、查看歌曲信息、从音乐库移除等操作。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 曲目列表与操作为通用能力，各形态一致；车机以更大字号与简化菜单交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：高亮标识当前播放曲目、置灰标识失效曲目；生态：分享依赖系统分享能力；安全：仅操作本地已扫描歌曲。 |
