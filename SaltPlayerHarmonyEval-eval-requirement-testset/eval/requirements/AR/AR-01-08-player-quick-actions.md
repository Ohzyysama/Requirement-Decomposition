# AR-01-08 播放页快捷操作入口
- AR-ID: AR-01-08
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放页快捷操作入口（保持屏幕唤醒/沉浸模式/查看音频信息/跳转专辑详情/跳转艺术家详情） |
| [需求描述] | 播放页提供一组针对当前歌曲的快捷操作入口：①播放时保持屏幕唤醒（开关，开启后播放期间屏幕不自动熄灭）；②进入沉浸模式（隐藏多余界面元素，专注播放与歌词）；③查看当前歌曲音频信息（标题/艺术家/专辑/格式/比特率/采样率等）；④展示当前歌曲所属专辑并点击跳转专辑详情页；⑤展示当前歌曲艺术家并点击跳转艺术家详情页（多艺术家时先选择具体艺术家）。 |
| [需求价值] | 让用户在播放页内一站式发起常用操作，无需返回列表即可保持屏幕、沉浸欣赏、查看信息并快速跳转专辑/艺术家。 |
| [需求场景] | 用户在播放页点击「保持屏幕唤醒」开关防止熄屏；点击进入沉浸模式专注听歌；点击查看音频信息；点击专辑/艺术家信息跳转对应详情页。 |
| [目标用户] | 在播放页深度操作当前歌曲的用户。 |
| [限制约束] | 跳转艺术家详情时若该曲有多个艺术家需先选择；保持屏幕唤醒仅在播放期间生效；沉浸模式相关行为与外观设置中的沉浸模式一致。 |
| [外部依赖] | 屏幕常亮能力、沉浸模式设置、歌曲音频信息面板、专辑详情页、艺术家详情页、媒体库数据。 |
| [性能指标] | 评测基线目标：各快捷入口触发即时响应 < 100ms。 |
| [功耗指标] | 继承所属 SR（保持屏幕唤醒会增加屏幕功耗，仅播放期间生效）。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 播放页提供「播放时保持屏幕唤醒」开关，开启后播放期间屏幕不自动熄灭。 2. 提供进入沉浸模式入口，进入后隐藏多余界面元素。 3. 可查看当前歌曲音频信息。 4. 可展示并点击跳转当前歌曲专辑详情页。 5. 可展示并点击跳转艺术家详情页，多艺术家时先弹出选择。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 快捷入口为通用能力；车机因驾驶安全通常精简入口与交互。 |
| [视觉/生态/安全等扩展维度] | 视觉：快捷入口以图标按钮形式聚合于播放页；生态：屏幕常亮、沉浸模式遵循系统窗口能力；安全：跳转与信息查看仅作用于本地媒体库数据。 |
