# AR-08-03 删除歌单
- AR-ID: AR-08-03
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 删除歌单 |
| [需求描述] | 提供删除歌单的能力，用户可移除不再需要的歌单。删除前给出二次确认，避免误删。删除仅移除歌单本身及其内部曲目引用，不影响音乐库中的原始歌曲文件，删除后歌单从列表中即时消失。 |
| [需求价值] | 让用户清理无用歌单，保持歌单列表整洁有序。 |
| [需求场景] | 用户在歌单更多菜单中选择「删除」，确认弹窗后该歌单从列表移除。 |
| [目标用户] | 有歌曲整理与分类需求的听歌用户。 |
| [限制约束] | 删除前需二次确认；仅删除歌单与曲目引用，不删除音乐库原始歌曲文件。 |
| [外部依赖] | 本地歌单持久化存储（应用首选项/数据库）。 |
| [性能指标] | 删除并刷新列表的响应时间<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 删除操作前弹出二次确认提示；2. 确认后该歌单立即从歌单列表中消失；3. 删除歌单不影响音乐库中对应歌曲的原始文件与入库状态；4. 取消确认时歌单保持不变。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端删除入口与确认弹窗形态随屏幕尺寸适配，核心删除逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 删除仅操作本地歌单数据，不涉及对原始媒体文件的删除权限。 |
