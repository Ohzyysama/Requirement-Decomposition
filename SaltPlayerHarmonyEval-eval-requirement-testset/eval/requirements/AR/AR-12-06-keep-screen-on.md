# AR-12-06 播放时保持屏幕常亮
- AR-ID: AR-12-06
- 所属 SR: SR-12 主题与外观个性化

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放时保持屏幕常亮 |
| [需求描述] | 提供开关让应用在播放音乐期间保持屏幕常亮，不进入自动息屏。开启后只要正在播放，屏幕便持续点亮；暂停或停止播放后解除常亮、恢复系统正常息屏策略。设置在重启后保持。 |
| [需求价值] | 满足看歌词、看封面或展示场景下不希望屏幕自动熄灭的使用诉求。 |
| [需求场景] | 用户开启"播放时保持屏幕常亮"，播放歌曲期间屏幕保持点亮；暂停后恢复正常息屏。 |
| [目标用户] | 需要边播放边查看屏幕（歌词/封面/展示）的用户。 |
| [限制约束] | 仅在播放进行时保持常亮；非播放状态应释放常亮以免不必要耗电。 |
| [外部依赖] | 屏幕常亮/保活窗口能力、播放状态、本地设置持久化。 |
| [性能指标] | 基线目标：播放开始/结束时常亮状态及时生效与释放。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 外观/播放设置中存在"播放时保持屏幕常亮"开关；2. 开启且正在播放时屏幕不自动息屏；3. 暂停或停止播放后解除常亮、恢复系统息屏策略；4. 关闭开关后播放期间不再保持常亮；5. 设置在应用重启后保持。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机常处常亮场景，该开关在车机端意义弱化；手机/平板按系统息屏策略生效。 |
| [视觉/生态/安全等扩展维度] | 常亮会增加屏幕功耗，仅在播放期间生效以平衡续航。 |
