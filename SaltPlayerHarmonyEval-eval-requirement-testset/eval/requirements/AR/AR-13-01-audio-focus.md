# AR-13-01 音频焦点
- AR-ID: AR-13-01
- 所属 SR: SR-13 音频输出

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音频焦点 |
| [需求描述] | 应用在播放时向系统申请音频焦点，并响应系统的焦点变化事件，与其他音频应用协同共享发声权。当来电、导航播报、其他媒体抢占焦点时自动暂停或降低音量，焦点恢复后再恢复播放。临时失去焦点（如短促提示音）时可暂时压低音量，避免与系统声音冲突。 |
| [需求价值] | 保证多应用同时使用音频时的良好共存体验，避免声音叠加混乱，符合系统音频生态规范。 |
| [需求场景] | 用户播放音乐时来电、收到通知提示音、切换到其他媒体应用、导航语音播报抢占焦点；焦点释放后回到本应用继续播放。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 焦点抢占/恢复行为遵循系统音频框架策略；恢复播放仅在此前为播放态且焦点重新获得时执行。 |
| [外部依赖] | @ohos.multimedia.audio(音频焦点/中断事件)、@ohos.multimedia.media(AVPlayer)、@ohos.multimedia.avsession。 |
| [性能指标] | 基线目标：焦点中断到暂停响应<200ms；焦点恢复到恢复播放<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 播放过程中来电或其他应用抢占音频焦点时，本应用自动暂停并停止发声；2. 抢占结束、焦点恢复后，能自动恢复到此前的播放状态继续播放；3. 遇到短促的临时中断（如提示音）时可暂时压低音量而非完全停止，提示结束后音量恢复；4. 焦点状态变化不会导致界面播放/暂停状态显示与实际发声不一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机环境中导航播报抢占焦点更频繁，焦点恢复策略需与车载语音协同；交互入口差异由各端适配。 |
| [视觉/生态/安全等扩展维度] | 音频焦点遵循系统音频框架与 AVSession 生态规范，确保多应用音频共存合规。 |
