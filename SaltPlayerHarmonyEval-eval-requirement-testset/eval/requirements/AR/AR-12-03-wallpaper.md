# AR-12-03 主页/播放页壁纸
- AR-ID: AR-12-03
- 所属 SR: SR-12 主题与外观个性化

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 主页/播放页壁纸 |
| [需求描述] | 允许用户为主页与播放页设置自定义壁纸作为背景。用户可选择图片作为壁纸，设置后主页/播放页以该图为背景显示，并可恢复为默认背景。壁纸设置在重启后保持，且需保证前景文字与控件在壁纸上仍清晰可读。 |
| [需求价值] | 提供个性化背景表达，增强沉浸感与界面美观度。 |
| [需求场景] | 用户在外观设置中选择壁纸图片应用于主页/播放页；或清除壁纸恢复默认背景。 |
| [目标用户] | 追求个性化外观与沉浸式播放体验的用户。 |
| [限制约束] | 壁纸作用于主页/播放页背景层；需保证前景内容可读，避免遮挡关键信息。 |
| [外部依赖] | 图片选择/读取能力、图片资源访问授权、本地设置持久化。 |
| [性能指标] | 基线目标：壁纸应用后页面渲染流畅，大图加载不阻塞主交互。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 用户可选择图片设置为主页/播放页壁纸；2. 设置后对应页面以该壁纸为背景显示；3. 可清除壁纸恢复默认背景；4. 壁纸设置在应用重启后保持；5. 壁纸背景下前景文字与控件仍清晰可读。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 不同屏幕比例下壁纸按各端适配缩放裁剪，避免变形或留白。 |
| [视觉/生态/安全等扩展维度] | 读取本地图片需相应授权；壁纸明暗需与前景内容保持足够对比度。 |
