# AR-11-02 锁定/解锁桌面歌词
- AR-ID: AR-11-02
- 所属 SR: SR-11 桌面与通知歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 锁定/解锁桌面歌词 |
| [需求描述] | 为已开启的桌面歌词提供锁定与解锁能力。锁定后桌面歌词固定在当前位置，不再响应拖动、缩放等手势操作，避免误触移动；解锁后用户可重新调整歌词位置与样式。锁定状态在切歌与应用重启后保持。 |
| [需求价值] | 防止用户在日常使用桌面时误操作移动桌面歌词，保证歌词稳定停留在期望位置。 |
| [需求场景] | 用户点击锁定按钮固定桌面歌词；锁定状态下尝试拖动歌词不生效；点击解锁后可再次拖动调整位置。 |
| [目标用户] | 已开启桌面歌词并希望固定其位置的用户。 |
| [限制约束] | 仅在桌面歌词已开启时可用；锁定状态下不响应位置/样式调整类手势。 |
| [外部依赖] | @ohos.multimedia.avsession（桌面歌词会话）、系统桌面/窗口服务。 |
| [性能指标] | 基线目标：锁定/解锁切换响应<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 点击锁定后桌面歌词固定在当前位置，拖动手势不再移动歌词；2. 锁定状态下歌词仍随播放进度正常逐行更新；3. 点击解锁后桌面歌词恢复可拖动/可调整状态；4. 锁定状态在切歌及应用重启后保持不变。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 锁定交互依赖各端手势能力，车机以触控为主且默认锁定以减少行车干扰；不支持桌面歌词的产品上该功能不可见。 |
| [视觉/生态/安全等扩展维度] | 锁定/解锁状态指示遵循系统桌面规范，避免遮挡其它桌面元素。 |
