# AR-15-04 开源致谢
- AR-ID: AR-15-04
- 所属 SR: SR-15 帮助/关于/致谢

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 开源致谢 |
| [需求描述] | 在关于相关页面集中展示应用所使用的开源组件及其许可信息，向开源社区致谢。用户可浏览开源项目列表，查看每个组件的名称与许可证类型，并可进一步查看许可证全文或项目主页链接，满足开源合规与透明披露要求。 |
| [需求价值] | 履行开源许可的署名与披露义务，保障合规，同时体现对开源生态的尊重。 |
| [需求场景] | 用户在关于页点击"开源致谢/开放源代码许可"，浏览组件列表，点击某项查看其许可证或主页。 |
| [限制约束] | 致谢内容以内置静态清单为主；查看许可证全文或主页可能需要外链跳转。 |
| [目标用户] | 关注开源合规的用户、开发者及审计相关方。 |
| [外部依赖] | 如提供主页/许可证外链则依赖系统浏览器；其余内容为本地内置。 |
| [性能指标] | 基线目标：致谢列表打开<200ms；条目点击与详情展示响应<200ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 能进入开源致谢页并显示完整的开源组件列表；2. 每个组件展示名称与许可证类型信息；3. 点击组件可查看其许可证全文或跳转项目主页；4. 列表内容完整、无缺漏或乱码，返回操作正常。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 平板可分栏展示列表与许可证详情；车机以静态只读列表为主，弱化外链跳转。 |
| [视觉/生态/安全等扩展维度] | 许可信息需准确完整以满足合规审计；外链跳转遵循系统隐私与安全规范。 |
