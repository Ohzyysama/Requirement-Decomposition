# AR-11-01 桌面歌词开关（AVSession）
- AR-ID: AR-11-01
- 所属 SR: SR-11 桌面与通知歌词

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 桌面歌词开关（AVSession） |
| [需求描述] | 提供桌面歌词的总开关，允许用户一键开启或关闭桌面歌词的展示。桌面歌词通过系统 AVSession 通道与媒体会话联动，开启后随当前播放曲目实时同步显示对应歌词行，关闭后则不再占用桌面层。开关状态在退出应用与重启后保持记忆。 |
| [需求价值] | 让用户在不打开应用界面的情况下也能在桌面看到实时歌词，提升听歌的沉浸感与便捷性。 |
| [需求场景] | 用户在设置或播放页点击桌面歌词开关；开启后切歌或进度推进时桌面歌词随之滚动更新；关闭后桌面歌词消失。 |
| [目标用户] | 希望在桌面常驻查看歌词的听歌用户。 |
| [限制约束] | 依赖系统对 AVSession 桌面歌词通道的支持；仅在有当前播放会话且歌词可用时展示有效内容。 |
| [外部依赖] | @ohos.multimedia.avsession（媒体会话与桌面歌词能力）、系统桌面/窗口服务。 |
| [性能指标] | 基线目标：开关切换响应<300ms；歌词行随播放进度同步刷新延迟<200ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 开启开关后桌面出现实时歌词，并随当前曲目播放进度逐行更新；2. 切换歌曲时桌面歌词同步切换为新曲目的歌词；3. 关闭开关后桌面歌词立即消失，不再占用桌面层；4. 应用退出并重启后，桌面歌词开关保持上次设置的开/关状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 桌面歌词依赖各端桌面/窗口形态，平板与车机按系统能力适配展示位置；不支持桌面层的产品上该开关自动降级隐藏。 |
| [视觉/生态/安全等扩展维度] | 遵循系统 AVSession 媒体生态；桌面歌词层级与样式遵从系统桌面规范。 |
