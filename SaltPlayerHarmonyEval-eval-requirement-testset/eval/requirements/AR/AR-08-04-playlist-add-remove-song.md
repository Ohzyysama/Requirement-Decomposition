# AR-08-04 歌曲加入/移出歌单
- AR-ID: AR-08-04
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲加入/移出歌单 |
| [需求描述] | 支持把歌曲加入指定歌单，也支持从歌单中移出歌曲。用户可从歌曲菜单或多选操作选择「添加到歌单」，挑选目标歌单完成加入；在歌单详情中可将某首歌移出。加入时对已存在曲目做去重，移出仅解除歌单引用，不影响音乐库原始歌曲。 |
| [需求价值] | 让用户灵活维护每个歌单的曲目内容，是歌单实际可用性的核心。 |
| [需求场景] | 用户在歌曲菜单选「添加到歌单」并选择目标歌单；或在歌单详情中将某首歌移出歌单。 |
| [目标用户] | 有歌曲整理与分类需求的听歌用户。 |
| [限制约束] | 同一首歌在同一歌单内不重复加入；移出仅解除引用，不删除音乐库原始歌曲文件。 |
| [外部依赖] | 本地歌单持久化存储（应用首选项/数据库）。 |
| [性能指标] | 加入/移出并刷新歌单的响应时间<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 从歌曲菜单或多选选择「添加到歌单」可选定目标歌单完成加入；2. 加入后歌曲出现在该歌单详情中，且重复加入同一首歌时做去重处理；3. 在歌单详情中可将某首歌移出，移出后该歌从歌单消失；4. 移出歌曲不影响其在音乐库中的原始文件与入库状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端加入/移出入口形态随屏幕尺寸适配，核心曲目维护逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 仅维护歌单与歌曲的引用关系，不涉及对原始媒体文件的修改。 |
