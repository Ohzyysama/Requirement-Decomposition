# AR-02-02 重新扫描
- AR-ID: AR-02-02
- 所属 SR: SR-02 媒体库与本地扫描

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 重新扫描 |
| [需求描述] | 用户可手动触发对本地音频的重新扫描，以刷新媒体库内容。重新扫描会重新遍历本地音频文件，纳入新增文件、移除已删除文件并更新已变化文件的元数据，使媒体库与设备上的实际音乐保持一致。完成后列表数据即时刷新呈现最新结果。 |
| [需求价值] | 让用户在新增或删除本地音乐后能够主动同步媒体库，保持库内容的准确与最新。 |
| [需求场景] | 用户在设置或媒体库入口主动触发重新扫描，以刷新增删或变更后的本地音频。 |
| [目标用户] | 经常增删本地音乐文件、需要保持媒体库最新的用户。 |
| [限制约束] | 需具备存储/媒体读取权限；重新扫描期间避免重复并发触发；仅处理受支持的音频格式。 |
| [外部依赖] | 系统媒体库与文件访问能力（媒体元数据读取、文件系统遍历）、存储访问权限。 |
| [性能指标] | 基线目标：重新扫描可在可接受时间内完成，扫描期间界面保持响应。 |
| [功耗指标] | 继承所属 SR |
| [ROM&RAM] | 继承所属 SR |
| [验收标准] | 1. 用户触发重新扫描后，本地新增的音频文件被纳入媒体库；2. 已从本地删除的音频在重新扫描后从媒体库中移除；3. 文件元数据发生变化时，库内对应信息得到更新；4. 重新扫描完成后相关列表即时刷新呈现最新结果。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线 |
| [适用产品] | 手机 / 平板 / 车机 |
| [适用产品差异分析] | 各端触发入口形态可能不同，由各端 UI 适配；底层扫描逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 重新扫描读取本地媒体需遵循系统权限与隐私规范。 |
