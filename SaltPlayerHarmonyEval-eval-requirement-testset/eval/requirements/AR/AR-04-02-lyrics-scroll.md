# AR-04-02 歌词同步滚动
- AR-ID: AR-04-02
- 所属 SR: SR-04 歌词显示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌词同步滚动 |
| [需求描述] | 歌词随播放进度自动同步滚动。当前演唱行高亮并保持在视图聚焦区域，随播放时间推进自动切换到下一行并平滑滚动。拖动进度或切换播放位置时，歌词即时跳转到对应行；用户手动滑动浏览歌词后，短暂停顿即恢复跟随当前进度自动滚动。 |
| [需求价值] | 让歌词与音乐精准对齐，用户无需手动跟随即可随时看到正在演唱的内容。 |
| [需求场景] | 用户在播放/歌词界面观看歌词时，歌词随播放进度自动滚动并高亮当前行；拖动进度条后歌词同步跳转。 |
| [目标用户] | 边听边看歌词的本地音乐用户。 |
| [限制约束] | 依赖歌词带有有效时间戳；手动浏览状态下暂停自动滚动，停顿后恢复跟随。 |
| [外部依赖] | 播放进度回调、AR-04-01 已加载的带时间戳歌词数据。 |
| [性能指标] | 基线目标：当前行切换与滚动动画流畅（60fps 目标），进度跳转后歌词定位延迟<200ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 播放过程中歌词随进度自动滚动，当前演唱行高亮并居于聚焦区域；2. 进入下一行时滚动过渡平滑无跳变；3. 拖动进度条后歌词即时跳转到对应行；4. 用户手动滑动浏览歌词后，停顿片刻自动恢复跟随当前进度滚动。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 滚动逻辑一致，聚焦区域位置与可视行数由各端屏幕尺寸适配。 |
| [视觉/生态/安全等扩展维度] | 滚动动画需保证流畅度与视觉平滑，避免卡顿或闪烁。 |
