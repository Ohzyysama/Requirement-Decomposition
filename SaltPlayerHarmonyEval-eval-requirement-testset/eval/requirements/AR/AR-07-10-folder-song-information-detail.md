# AR-07-10 文件夹详情-歌曲信息详情展示（标题/艺术家/专辑/专辑艺术家/时长/比特率/声道/采样率/位深/大小/格式/编码/版权/添加与修改时间/路径）
- AR-ID: AR-07-10
- 所属 SR: SR-07 文件夹与存储管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹详情-歌曲信息详情展示（标题/艺术家/专辑/专辑艺术家/时长/比特率/声道/采样率/位深/大小/格式/编码/版权/添加与修改时间/路径） |
| [需求描述] | 在文件夹详情页面内，用户对某首歌选择「查看歌曲信息」后，信息面板分组展示该曲的详细元数据：标题、艺术家、专辑、专辑艺术家、时长（分:秒）、比特率（kbps）、声道数（仅在大于 0 时显示）、采样率（Hz，仅在大于 0 时显示）、位深（仅在大于 0 时显示）、文件大小（自动选择合适单位）、格式、编码、版权、添加时间、修改时间、文件路径（以可读形式展示）。某项缺失或未知时以「未知」占位，信息区域可上下滚动查看完整内容。 |
| [需求价值] | 让用户在文件夹详情页内全面了解一首歌的音质参数、格式、来源与文件信息，便于辨别音质、核对来源与定位文件位置。 |
| [需求场景] | 用户在某文件夹详情页对一首歌选择「查看歌曲信息」，查看标题、专辑、时长、比特率、采样率、位深、文件大小、格式、路径等详细信息。 |
| [目标用户] | 关注歌曲音质参数与文件信息的进阶用户。 |
| [限制约束] | 声道数/采样率/位深仅在数值大于 0 时显示；缺失项以「未知」占位；展示内容来源于本地已扫描歌曲元数据。 |
| [外部依赖] | 本地已扫描歌曲的元数据；无网络依赖。 |
| [性能指标] | 评测基线目标：信息面板展示与滚动流畅，打开即时（<100ms 量级）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 文件夹详情页内信息面板分组展示标题、艺术家、专辑、专辑艺术家、时长（分:秒）、比特率（kbps）、文件大小（自动单位）、格式、编码、版权、添加时间、修改时间、文件路径（可读形式）。 2. 声道数、采样率（Hz）、位深仅在数值大于 0 时展示，否则不显示该项。 3. 缺失或未知的信息项以「未知」占位展示。 4. 信息区域可上下滚动查看完整内容。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 歌曲信息展示为通用能力，手机/平板一致；车机以简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：信息按基础信息与媒体技术信息分组卡片展示，标签在上、取值在下；安全：仅本地展示元数据，路径以解码后可读形式呈现，不涉及网络上报。 |
