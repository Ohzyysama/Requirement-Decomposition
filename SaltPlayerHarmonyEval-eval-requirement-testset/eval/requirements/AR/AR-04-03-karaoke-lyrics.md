# AR-04-03 卡拉OK 逐字歌词
- AR-ID: AR-04-03
- 所属 SR: SR-04 歌词显示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 卡拉OK 逐字歌词 |
| [需求描述] | 在歌词带有逐字时间信息时，提供卡拉OK 式的逐字高亮效果。当前演唱行内，文字随演唱进度从左到右逐字（逐音节）渐进着色，已唱部分与未唱部分形成颜色区分，营造卡拉OK 跟唱体验。当歌词仅含行级时间戳时，自动回退为整行高亮。 |
| [需求价值] | 提供更精细的逐字跟唱体验，提升歌词显示的沉浸感与趣味性。 |
| [需求场景] | 用户播放带逐字时间信息的歌曲时，当前行文字随演唱进度逐字点亮。 |
| [目标用户] | 喜爱跟唱、追求沉浸歌词体验的用户。 |
| [限制约束] | 逐字高亮依赖歌词含逐字时间数据；缺失逐字信息时回退为整行高亮。 |
| [外部依赖] | 播放进度回调、含逐字时间戳的歌词数据。 |
| [性能指标] | 基线目标：逐字着色随进度平滑推进，渲染保持流畅（60fps 目标）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 歌词含逐字时间信息时，当前行文字随演唱进度由前向后逐字渐进高亮；2. 已唱与未唱部分颜色明显区分；3. 逐字着色推进与音乐演唱进度对齐；4. 歌词仅含行级时间戳时自动回退为整行高亮且不报错。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 逐字高亮逻辑一致，字号与着色范围由各端排版适配。 |
| [视觉/生态/安全等扩展维度] | 着色过渡需平滑自然，配色需在浅色/深色主题下均清晰可辨。 |
