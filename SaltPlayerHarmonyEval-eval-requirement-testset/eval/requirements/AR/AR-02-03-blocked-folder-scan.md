# AR-02-03 扫描屏蔽文件夹
- AR-ID: AR-02-03
- 所属 SR: SR-02 媒体库与本地扫描

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 扫描屏蔽文件夹 |
| [需求描述] | 用户可设置在扫描时屏蔽（排除）指定的文件夹，使这些文件夹内的音频不被纳入媒体库。被屏蔽文件夹中的文件在扫描与重新扫描时均被跳过，因而不会出现在歌曲、专辑、艺术家等列表中。用户可管理屏蔽列表，新增或移除被屏蔽的文件夹，调整后在下次扫描时生效。 |
| [需求价值] | 让用户排除录音、播客、铃声等非音乐目录，避免无关音频污染媒体库，提升库的纯净度。 |
| [需求场景] | 用户在设置中将某个文件夹加入屏蔽列表，后续扫描/重新扫描将跳过该文件夹下的音频。 |
| [目标用户] | 希望排除特定目录、保持媒体库整洁的用户。 |
| [限制约束] | 屏蔽以文件夹为单位生效；调整屏蔽列表后通常需在下次扫描时才反映到库内容；需存储访问权限以读取目录结构。 |
| [外部依赖] | 系统文件访问能力（目录遍历与路径识别）、本地配置存储。 |
| [性能指标] | 基线目标：屏蔽判定不显著增加扫描耗时。 |
| [功耗指标] | 继承所属 SR |
| [ROM&RAM] | 继承所属 SR |
| [验收标准] | 1. 用户可将指定文件夹加入屏蔽列表；2. 被屏蔽文件夹内的音频在扫描/重新扫描后不出现在媒体库中；3. 将文件夹从屏蔽列表移除后，其内音频在下次扫描时被重新纳入；4. 屏蔽设置在多次扫描间持久保留。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线 |
| [适用产品] | 手机 / 平板 / 车机 |
| [适用产品差异分析] | 各端目录结构与路径表示可能不同，由系统文件能力屏蔽差异；设置入口形态由各端 UI 适配。 |
| [视觉/生态/安全等扩展维度] | 屏蔽配置属本地用户偏好，遵循本地存储与隐私规范。 |
