# AR-15-01 帮助与 FAQ
- AR-ID: AR-15-01
- 所属 SR: SR-15 帮助/关于/致谢

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 帮助与 FAQ |
| [需求描述] | 为用户提供集中的帮助文档与常见问题（FAQ）入口。用户可在帮助页浏览功能说明与高频问题解答，快速了解播放、歌单、扫描等核心用法。问题条目支持展开/收起，便于在不离开页面的情况下查阅细节，帮助用户自助解决使用疑问。 |
| [需求价值] | 降低用户上手门槛与求助成本，减少重复咨询，提升使用体验与留存。 |
| [需求场景] | 用户在设置或关于页点击"帮助/常见问题"，浏览功能说明与 FAQ，展开某条问题查看详细解答。 |
| [目标用户] | 初次使用或遇到使用疑问的全体用户。 |
| [限制约束] | 帮助内容以本地内置文案为主；条目展开/收起仅影响当前视图，不改变应用状态。 |
| [外部依赖] | 无强外部依赖；如含跳转外链则依赖系统浏览器能力。 |
| [性能指标] | 基线目标：帮助页打开与条目展开响应<200ms，列表滚动流畅无明显卡顿。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 进入帮助页能正确显示功能说明与 FAQ 列表；2. 点击某条问题可展开查看解答，再次点击可收起；3. 列表内容完整、文案无截断或乱码；4. 返回操作能正常退出帮助页且不影响其他界面状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 平板可利用更大屏幕分栏展示；车机以精简静态文案为主，弱化长篇阅读场景。 |
| [视觉/生态/安全等扩展维度] | 文案遵循无障碍可读性要求；外链跳转需符合系统隐私与安全规范。 |
