# AR-14-01 启动软件自动打开播放页
- AR-ID: AR-14-01
- 所属 SR: SR-14 实验功能（实验室）

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 启动软件自动打开播放页 |
| [需求描述] | 在实验室中提供一个开关，开启后每次启动应用会自动跳转到播放页。用户打开应用即直接进入正在播放/可播放的播放界面，而无需手动从主页再点进播放页。该开关默认关闭，关闭时启动后停留在常规首页。 |
| [需求价值] | 让以"听歌"为主的用户开机即达播放界面，减少进入播放的操作步骤，提升常用场景的直达效率。 |
| [需求场景] | 用户在实验室开启"启动自动打开播放页"开关后，下次冷启动或重新打开应用时自动进入播放页。 |
| [目标用户] | 以播放为核心使用场景、希望快速进入播放界面的用户。 |
| [限制约束] | 实验性预览功能，仅在实验室开关开启时生效；默认关闭；自动跳转仅在应用启动阶段触发一次，不影响后续正常导航。 |
| [外部依赖] | 应用首页路由/导航能力、播放页页面与本地设置项持久化存储。 |
| [性能指标] | 基线目标：开关开启时启动到进入播放页的额外跳转耗时<200ms，不引入可感知的启动卡顿。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 在实验室开启该开关后，重新启动应用自动进入播放页；2. 关闭该开关后，重新启动应用停留在常规首页，不自动跳转；3. 开关状态在应用重启后保持不变；4. 自动跳转后播放页功能与手动进入时一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端启动入口与首页形态不同，自动跳转的落地页由各端播放页适配，交互效果保持一致。 |
| [视觉/生态/安全等扩展维度] | 作为实验室预览开关呈现，需明确"实验性功能"标识；启动跳转应遵循系统导航规范，避免遮挡或打断系统级提示。 |
