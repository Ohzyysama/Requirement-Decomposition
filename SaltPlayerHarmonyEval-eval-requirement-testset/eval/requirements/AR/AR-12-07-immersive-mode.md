# AR-12-07 沉浸模式
- AR-ID: AR-12-07
- 所属 SR: SR-12 主题与外观个性化

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 沉浸模式 |
| [需求描述] | 提供沉浸模式开关与体验：进入沉浸模式后，界面隐藏状态栏、控制面板等多余元素，仅保留核心视觉（如专辑封面、歌词），让用户专注于听歌与歌词浏览。沉浸模式可在播放页/歌词页触发，退出后恢复常规界面布局。 |
| [需求价值] | 通过精简界面元素营造专注、沉浸的听歌氛围，提升视觉与情绪体验。 |
| [需求场景] | 用户在播放页进入沉浸模式，状态栏与控制面板隐去，屏幕仅显示封面与歌词；再次操作或退出后界面恢复正常。 |
| [目标用户] | 偏好沉浸专注听歌体验的用户。 |
| [限制约束] | 沉浸模式仅隐藏界面元素、不改变播放行为；与播放页快捷操作入口中的「沉浸模式」为同一能力；退出后恢复常规布局。 |
| [外部依赖] | 系统窗口/状态栏显隐能力、播放页与歌词页界面、外观个性化设置。 |
| [性能指标] | 评测基线目标：进入/退出沉浸模式切换响应 < 150ms，过渡流畅无明显闪烁。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 提供沉浸模式开关/入口。 2. 进入沉浸模式后隐藏状态栏与控制面板等多余元素，保留核心视觉。 3. 退出沉浸模式后界面恢复常规布局。 4. 沉浸模式不影响正在进行的播放。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 沉浸模式为通用外观能力；平板/车机依屏幕形态调整保留元素。 |
| [视觉/生态/安全等扩展维度] | 视觉：以隐藏冗余元素聚焦封面与歌词营造沉浸感；生态：状态栏显隐遵循系统窗口能力。 |
