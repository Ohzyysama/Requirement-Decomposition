# AR-07-02 文件夹列表页搜索歌曲
- AR-ID: AR-07-02
- 所属 SR: SR-07 文件夹与存储管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 文件夹列表页搜索歌曲 |
| [需求描述] | 文件夹列表页提供搜索入口，用户输入关键词后可在文件夹相关的本地歌曲范围内检索匹配的歌曲，搜索结果以歌曲列表形式展示。搜索随输入实时过滤，关键词可匹配歌名、艺术家等信息。清空关键词后恢复原文件夹列表展示。 |
| [需求价值] | 让用户在文件夹维度下也能快速按关键词找到目标歌曲，免去逐个文件夹翻找。 |
| [需求场景] | 用户在文件夹页点击搜索框输入关键词，页面实时展示匹配的歌曲列表，清空后回到文件夹列表。 |
| [目标用户] | 需要在文件夹页快速检索本地歌曲的用户。 |
| [限制约束] | 搜索范围限于本地已扫描入库的歌曲；按关键词匹配歌名/艺术家等字段；隐藏/屏蔽文件夹下的歌曲不纳入结果。 |
| [外部依赖] | 本地已扫描歌曲数据；无网络依赖。 |
| [性能指标] | 评测基线目标：输入关键词到结果刷新即时响应 < 100ms；结果列表滚动 ≥ 60fps。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 文件夹列表页提供搜索入口，可输入关键词检索本地歌曲。 2. 搜索随输入实时过滤，结果以歌曲列表展示。 3. 关键词可匹配歌名、艺术家等信息。 4. 清空关键词后恢复原文件夹列表。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 搜索为通用能力，各形态一致；车机因驾驶安全通常弱化文本输入搜索。 |
| [视觉/生态/安全等扩展维度] | 视觉：顶部搜索框与实时结果列表；安全：仅检索本地已扫描歌曲。 |
