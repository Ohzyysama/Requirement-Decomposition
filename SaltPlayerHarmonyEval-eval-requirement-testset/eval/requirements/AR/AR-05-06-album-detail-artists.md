# AR-05-06 专辑详情参与创作艺术家区块（头像/名/歌曲数，点击跳转艺术家详情页）
- AR-ID: AR-05-06
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑详情参与创作艺术家区块（头像/名/歌曲数，点击跳转艺术家详情页） |
| [需求描述] | 专辑详情页提供「参与创作艺术家」区块，聚合展示该专辑中所有参与创作的艺术家，每位艺术家展示头像、艺术家名以及其在本专辑（或媒体库）中的歌曲数。用户点击某位艺术家，即跳转到该艺术家的详情页查看其完整作品。 |
| [需求价值] | 让用户从专辑维度发现并跳转到相关艺术家，扩展浏览路径，建立专辑与艺术家之间的导航连接。 |
| [需求场景] | 用户在专辑详情页向下浏览到参与创作艺术家区块，点击其中一位艺术家头像，跳转进入该艺术家详情页。 |
| [目标用户] | 希望按艺术家维度延展浏览的本地音乐用户。 |
| [限制约束] | 仅展示该专辑实际参与创作的艺术家；头像缺失时显示占位图；歌曲数依据媒体库聚合统计。 |
| [外部依赖] | 媒体库艺术家聚合数据（头像/歌曲数）、艺术家详情页（跳转目标）。 |
| [性能指标] | 评测基线目标：区块加载与跳转响应及时，头像异步加载不阻塞页面。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 专辑详情页展示参与创作艺术家区块。 2. 每位艺术家显示头像（缺失时占位图）、艺术家名与歌曲数。 3. 区块仅包含该专辑实际参与创作的艺术家。 4. 点击艺术家跳转到对应艺术家详情页。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 艺术家区块与跳转为通用能力，各形态一致；平板/车机大屏下可横向展示更多艺术家。 |
| [视觉/生态/安全等扩展维度] | 视觉：圆形头像与艺术家名横向排布；交互：点击艺术家跳转艺术家详情页；安全：仅展示本地已扫描艺术家信息。 |
