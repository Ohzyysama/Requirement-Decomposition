# AR-09-01 音质分布与容量统计
- AR-ID: AR-09-01
- 所属 SR: SR-09 音乐库

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音质分布与容量统计 |
| [需求描述] | 在音乐库概览中以统计视图呈现全库歌曲的音质构成。展示不同音质（如标准、高品质、无损等）歌曲的分布占比，给出音乐库所有歌曲所占的总存储大小，并分别列出各音质档位下的歌曲数目。用户无需逐首查看即可快速掌握自己曲库的音质结构与容量规模。 |
| [需求价值] | 帮助用户直观了解曲库的音质构成与存储占用，辅助判断是否需要补充无损资源或清理空间。 |
| [需求场景] | 用户进入音乐库统计概览页面，查看音质分布图表、歌曲总大小及各音质对应的歌曲数量。 |
| [目标用户] | 关注音质与存储管理的本地曲库用户。 |
| [限制约束] | 统计范围限于已纳入音乐库的本地歌曲；音质识别依赖文件元数据，缺失或异常元数据可能归入未知档位。 |
| [外部依赖] | 本地媒体扫描与元数据解析、文件系统容量信息。 |
| [性能指标] | 基线目标：进入统计页后概览数据加载与渲染<500ms（大库可分批计算）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 统计页正确展示各音质档位的歌曲分布及其占比；2. 展示音乐库全部歌曲的总存储大小，数值单位清晰；3. 分别给出每个音质档位的歌曲数目，各档位数目之和与全库歌曲总数一致；4. 元数据缺失的歌曲被归入明确的未知/其他档位而非丢失；5. 曲库内容增删后，重新进入或刷新统计页数据随之更新。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 统计逻辑各端一致；平板可利用更大屏幅展示更丰富的图表布局，车机以简化只读概览呈现。 |
| [视觉/生态/安全等扩展维度] | 容量与音质统计仅基于本地文件，不涉及上传；图表配色遵循应用整体视觉规范。 |
