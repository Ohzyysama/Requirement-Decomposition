# AR-08-05 歌单导入/导出
- AR-ID: AR-08-05
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌单导入/导出 |
| [需求描述] | 提供歌单的导入与导出能力。导出将歌单及其曲目信息写出为歌单文件，便于备份或迁移；导入则读取歌单文件并在本地重建歌单，按曲目信息匹配音乐库中的本地歌曲。对无法匹配到本地歌曲的条目给出合理提示或跳过。 |
| [需求价值] | 支持歌单的备份、迁移与跨设备复用，降低用户重新整理歌单的成本。 |
| [需求场景] | 用户导出某个歌单为文件保存；或选择歌单文件导入，应用据此重建歌单并匹配本地歌曲。 |
| [外部依赖] | @ohos.file.fs/文件选择与读写、本地歌单持久化存储。 |
| [目标用户] | 有歌单备份与迁移需求的听歌用户。 |
| [限制约束] | 导入按曲目信息匹配本地音乐库；无法匹配的条目给出提示或跳过；文件读写需用户授权。 |
| [性能指标] | 常规规模歌单（数百首内）导入/导出在数秒内完成。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 可将选定歌单导出为歌单文件并保存到指定位置；2. 可选择歌单文件导入并在本地重建歌单；3. 导入时按曲目信息匹配音乐库中的本地歌曲；4. 对无法匹配到本地歌曲的条目给出提示或合理跳过；5. 文件读写在用户授权下进行。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端文件选择与保存入口依系统文件能力适配，核心导入导出逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 文件访问遵循系统文件授权与沙箱规范，导入仅在本地匹配歌曲不联网。 |
