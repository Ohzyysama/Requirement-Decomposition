# AR-03-05 歌曲列表随机播放全部
- AR-ID: AR-03-05
- 所属 SR: SR-03 歌曲管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲列表随机播放全部 |
| [需求描述] | 歌曲列表提供「随机播放全部」入口，用户点击后将当前列表的全部歌曲以随机顺序加入播放队列并开始播放。该操作会以当前列表（含搜索/筛选后的结果）为范围构建随机队列，使用户无需手动选择即可随机欣赏整张列表。 |
| [需求价值] | 一键随机畅听整张歌曲列表，降低选择成本，提升随机收听体验。 |
| [需求场景] | 用户在歌曲列表点击「随机播放」，应用将全部歌曲打乱并开始播放第一首。 |
| [限制约束] | 随机范围为当前列表（含搜索/筛选结果）内的歌曲；列表为空时无可播放内容。 |
| [目标用户] | 希望随机收听本地歌曲、不愿手动挑选的用户。 |
| [外部依赖] | 播放控制与队列能力（SR-01，构建并播放队列）、当前歌曲列表数据。 |
| [性能指标] | 评测基线目标：点击后即时构建随机队列并开始播放，无可感知延迟。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 点击「随机播放全部」后以当前列表全部歌曲构建随机队列并开始播放。 2. 随机范围正确覆盖搜索/筛选后的当前列表。 3. 列表为空时操作不产生异常且有合理反馈。 4. 进入随机播放后播放队列与播放模式状态正确。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 随机播放为通用能力，各形态一致；车机以更大入口按钮便于行车点击。 |
| [视觉/生态/安全等扩展维度] | 视觉：随机播放入口图标清晰可辨；安全：仅播放本地歌曲。 |
