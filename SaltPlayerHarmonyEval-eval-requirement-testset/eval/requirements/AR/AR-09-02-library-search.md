# AR-09-02 音乐库搜索
- AR-ID: AR-09-02
- 所属 SR: SR-09 音乐库

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音乐库搜索 |
| [需求描述] | 在音乐库内提供搜索能力，用户输入关键词即可在全库范围内检索匹配的歌曲。检索结果以歌曲列表的形式展示，便于用户从结果中直接点击播放或进行后续操作。搜索应实时或在确认后给出匹配项，关键词为空或无匹配时给出相应的空态反馈。 |
| [需求价值] | 让用户在庞大曲库中快速定位目标歌曲，免去逐屏翻找，提升查找效率。 |
| [需求场景] | 用户在音乐库搜索入口输入歌曲名等关键词，系统在曲库中匹配并以歌曲列表展示结果。 |
| [目标用户] | 拥有较大本地曲库、需要快速定位歌曲的用户。 |
| [限制约束] | 搜索范围限于已纳入音乐库的歌曲；匹配依据歌曲名等可检索字段，匹配规则以应用既有策略为准。 |
| [外部依赖] | 本地曲库数据与索引、歌曲列表展示组件。 |
| [性能指标] | 基线目标：输入关键词后结果返回与列表渲染<300ms（常规库规模）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 输入关键词后能检索出曲库中匹配的歌曲；2. 搜索结果以歌曲列表形式展示，列表项信息与库内歌曲一致；3. 结果列表中的歌曲可直接点击播放或触发后续操作；4. 关键词无匹配或为空时给出明确的空态/提示而非报错；5. 修改关键词后结果相应更新。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 搜索与结果展示逻辑各端一致；车机侧建议以更大字号与简化交互呈现结果列表，降低驾驶场景操作负担。 |
| [视觉/生态/安全等扩展维度] | 搜索仅在本地曲库内进行，不上传关键词；结果列表样式遵循应用歌曲列表统一视觉规范。 |
