# AR-06-01 艺术家列表项展示（头像/艺术家名/歌曲数）
- AR-ID: AR-06-01
- 所属 SR: SR-06 艺术家管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 艺术家列表项展示（头像/艺术家名/歌曲数） |
| [需求描述] | 艺术家列表中每一行展示一位艺术家：左侧显示艺术家头像（无头像时使用默认占位图），右侧主标题显示艺术家名，副标题或附属信息显示该艺术家名下的歌曲数。整行可点击进入该艺术家的详情页。 |
| [需求价值] | 让用户一眼获取每位艺术家的头像、名称与作品数量，便于快速辨认并进入对应艺术家详情。 |
| [需求场景] | 用户在艺术家列表滚动浏览，通过头像与艺术家名快速辨认目标艺术家，并据歌曲数了解其作品规模，点击进入艺术家详情。 |
| [目标用户] | 浏览本地艺术家列表的普通用户。 |
| [限制约束] | 头像缺失时显示默认占位图；歌曲数为该艺术家名下本地已扫描歌曲的数量；整行点击进入详情。 |
| [外部依赖] | 头像缓存与封面/头像提取（头像来源）、本地歌曲库统计（歌曲数来源）。 |
| [性能指标] | 评测基线目标：列表滚动 ≥ 60fps；头像异步加载不阻塞滚动。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 每行正确显示艺术家头像，无头像时显示默认占位图。 2. 主标题正确显示艺术家名。 3. 正确显示该艺术家名下的歌曲数。 4. 点击列表项进入对应艺术家详情页。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 列表项展示为通用能力，各形态一致；车机以更大字号与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：圆形头像与默认占位图、歌曲数附属文字；安全：仅展示本地已扫描艺术家信息。 |
