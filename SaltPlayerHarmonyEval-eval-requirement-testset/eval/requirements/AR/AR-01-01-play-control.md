# AR-01-01 播放/暂停/上一曲/下一曲控制
- AR-ID: AR-01-01
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放/暂停/上一曲/下一曲控制 |
| [需求描述] | 为当前曲目提供最基础的播放控制：播放、暂停、上一曲、下一曲。用户点击播放即开始播放所选歌曲，点击暂停立即停止发声；在尚未加载歌曲时点击播放，应能从上次记忆的曲目自动恢复并起播。点击上一曲/下一曲在当前播放队列中切换到相邻曲目。除应用内的播放界面外，锁屏与控制中心的系统媒体卡片也能触发相同的播放、暂停、上一曲、下一曲控制，效果保持一致。 |
| [需求价值] | 满足用户对当前曲目最基本的开始/停止/切换控制，是播放器不可或缺的核心交互。 |
| [需求场景] | 用户在播放界面点击播放/暂停、上一曲/下一曲按钮；锁屏或控制中心的系统媒体卡片下达播放、暂停、上一曲、下一曲命令。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 仅对已加载或可恢复的曲目生效；暂停仅在正在播放时执行；播放控制需串行化以避免状态竞争。 |
| [外部依赖] | 系统多媒体播放能力、系统媒体会话（AVSession）媒体命令、文件读取与授权能力。 |
| [性能指标] | 起播延迟基线目标小于 300 毫秒。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 点击播放后歌曲开始发声，界面进入"播放中"状态，且不残留上一次的播放错误提示；2. 点击暂停后界面立即切换为"已暂停"状态并停止发声；3. 恢复播放时从此前暂停的进度位置继续播放；4. 在未加载任何歌曲时点击播放，能自动恢复上次记忆的曲目并起播；5. 锁屏/控制中心系统媒体卡片上的播放、暂停、上一曲、下一曲操作与应用内对应控制效果一致。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机主要经系统媒体命令路径触发；交互入口形态差异由各端 UI 适配。 |
| [视觉/生态/安全等扩展维度] | 后台继续播放需后台运行能力；媒体命令回调遵循系统媒体会话生态。 |
