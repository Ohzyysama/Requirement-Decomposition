# AR-04-05 播放页迷你歌词
- AR-ID: AR-04-05
- 所属 SR: SR-04 歌词显示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放页迷你歌词 |
| [需求描述] | 在播放页封面区域内嵌一个迷你歌词展示区，无需进入全屏歌词界面即可看到当前演唱的一两行歌词，并随播放进度同步更新与高亮。点击迷你歌词区域可展开进入完整歌词界面；当前曲目无歌词时，迷你区显示「暂无歌词」占位。 |
| [需求价值] | 让用户在播放主界面即可顺带浏览歌词，减少跳转操作，提升一屏信息密度。 |
| [需求场景] | 用户停留在播放页时，封面区内嵌的迷你歌词随进度滚动展示当前行；点击后进入完整歌词界面。 |
| [目标用户] | 希望在播放主界面顺带看歌词的用户。 |
| [限制约束] | 迷你区展示行数有限（通常 1～2 行）；同步逻辑复用歌词滚动能力；无歌词时显示占位。 |
| [外部依赖] | AR-04-01 歌词加载、AR-04-02 同步滚动、播放进度回调。 |
| [性能指标] | 基线目标：迷你歌词随进度更新流畅，点击展开到完整歌词界面响应<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 播放页封面区内嵌展示迷你歌词，随播放进度同步更新当前行；2. 迷你歌词当前行高亮且与播放进度对齐；3. 点击迷你歌词区域展开进入完整歌词界面；4. 当前曲目无歌词时迷你区显示「暂无歌词」占位。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 迷你歌词位置与可见行数由各端播放页布局适配，交互逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 迷你歌词需与封面背景保持足够对比度，确保可读性。 |
