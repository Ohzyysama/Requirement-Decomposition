# AR-08-02 重命名歌单
- AR-ID: AR-08-02
- 所属 SR: SR-08 歌单管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 重命名歌单 |
| [需求描述] | 允许用户修改已有歌单的名称。进入重命名入口后，输入框预填当前名称，用户编辑后确认即更新。重命名同样遵循非空与重名校验，更新成功后列表与详情页中的歌单名称同步刷新。 |
| [需求价值] | 让用户在使用过程中灵活调整歌单命名，保持歌单组织清晰可辨。 |
| [需求场景] | 用户在歌单更多菜单中选择「重命名」，修改名称并确认，名称在列表与详情中即时更新。 |
| [目标用户] | 有歌曲整理与分类需求的听歌用户。 |
| [限制约束] | 新名称不可为空；与其他歌单重名时需给出提示；名称长度受合理上限约束。 |
| [外部依赖] | 本地歌单持久化存储（应用首选项/数据库）。 |
| [性能指标] | 重命名并刷新名称的响应时间<300ms。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 重命名入口弹出的输入框预填当前歌单名称；2. 修改为合法名称并确认后，列表与详情页名称同步更新；3. 名称改为空时无法保存并有提示；4. 改为与其他歌单重名时给出提示或合理处理。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端重命名入口形态随屏幕尺寸适配，核心更新逻辑一致。 |
| [视觉/生态/安全等扩展维度] | 名称更新写入本地存储，遵循应用数据持久化规范。 |
