# AR-01-02 播放模式切换（循环/单曲/随机）
- AR-ID: AR-01-02
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放模式切换（循环/单曲/随机） |
| [需求描述] | 提供列表循环、单曲循环、随机播放三种播放模式，用户可在三者之间循环切换。列表循环按队列顺序依次播放并在末尾回到队首；单曲循环始终重复当前曲目；随机播放以打乱的顺序在队列中选曲。切换后的模式状态需在界面上明确反映，并影响后续上一曲/下一曲及自动续播的选曲逻辑。 |
| [需求价值] | 让用户根据收听场景自由选择曲目播放顺序，提升收听体验的灵活性。 |
| [需求场景] | 用户在播放界面或播放队列中点击播放模式按钮，在三种模式间循环切换。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 模式切换需在当前播放队列上下文中生效；模式状态需持久保留，应用重启后恢复。 |
| [外部依赖] | 系统多媒体播放能力、本地偏好存储（持久化当前播放模式）。 |
| [性能指标] | 模式切换即时生效，界面状态更新无可感知延迟。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 反复点击播放模式按钮可在列表循环、单曲循环、随机播放三种模式间循环切换，界面图标/文案同步反映当前模式；2. 列表循环模式下播放到队尾后回到队首继续播放；3. 单曲循环模式下当前曲目结束后重复播放同一曲目；4. 随机播放模式下下一曲为队列中随机选取的曲目；5. 切换后的播放模式在应用重启后仍被正确恢复。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端共用同一套模式切换逻辑，仅入口控件形态与布局随端适配。 |
| [视觉/生态/安全等扩展维度] | 模式状态图标需在浅色/深色主题下均清晰可辨。 |
