# AR-15-03 关于与版本/检查更新
- AR-ID: AR-15-03
- 所属 SR: SR-15 帮助/关于/致谢

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 关于与版本/检查更新 |
| [需求描述] | 在关于页集中展示应用名称、图标、当前版本号等基本信息，并提供检查更新入口。用户点击检查更新后，应用对比当前版本与最新可用版本：若已是最新则提示已为最新版本，若存在新版本则提示版本信息并引导用户前往升级。整个过程给出清晰的状态反馈。 |
| [需求价值] | 让用户清楚了解所用版本并便捷获取更新，保障功能与安全修复及时触达。 |
| [需求场景] | 用户在设置页进入"关于"，查看版本信息并点击"检查更新"，根据结果选择是否升级。 |
| [目标用户] | 关注版本信息与希望保持应用最新的全体用户。 |
| [限制约束] | 检查更新依赖网络连通；无网络或服务不可用时应给出失败提示而非静默失败。 |
| [外部依赖] | 网络请求能力与更新检查服务/渠道；升级跳转依赖应用市场或下载页可用性。 |
| [性能指标] | 基线目标：关于页打开<200ms；检查更新结果返回（含网络）目标<3s 并有加载态提示。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 关于页正确显示应用名称、图标与当前版本号；2. 点击检查更新有明确加载/进行中反馈；3. 已是最新版本时提示"已是最新版本"；4. 存在新版本时提示版本信息并提供前往升级的引导；5. 网络异常或检查失败时给出可理解的错误提示。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机更新通常随系统/车厂渠道分发，可隐藏或弱化自助检查更新入口；手机/平板支持完整检查与跳转。 |
| [视觉/生态/安全等扩展维度] | 版本检查与下载遵循系统安全与签名校验规范；升级跳转遵循应用市场生态。 |
