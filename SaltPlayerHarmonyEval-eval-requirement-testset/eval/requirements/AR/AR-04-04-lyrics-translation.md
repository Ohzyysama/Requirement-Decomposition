# AR-04-04 歌词翻译显示切换
- AR-ID: AR-04-04
- 所属 SR: SR-04 歌词显示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌词翻译显示切换 |
| [需求描述] | 当歌词包含翻译内容时，提供翻译显示的开关切换。开启后在每行原文歌词下方对应展示翻译文本，关闭后仅显示原文。切换状态被记忆，下次查看歌词时沿用上次选择；当歌词不含翻译时，隐藏该切换入口或置灰不可用。 |
| [需求价值] | 帮助用户理解外语歌曲含义，并可按需收起翻译以保持界面简洁。 |
| [需求场景] | 用户播放含翻译的歌曲，在歌词界面切换翻译显示开关以展示或隐藏译文。 |
| [目标用户] | 收听外语歌曲、需要理解歌词含义的用户。 |
| [限制约束] | 仅在歌词含翻译数据时可用；无翻译时入口不可用。 |
| [外部依赖] | 含翻译文本的歌词数据、翻译显示偏好的本地持久化。 |
| [性能指标] | 基线目标：切换翻译显示后界面即时刷新（<150ms），不影响滚动同步。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 歌词含翻译时开启开关，每行原文下方展示对应译文；2. 关闭开关后仅显示原文歌词；3. 翻译显示状态被记忆，再次查看歌词沿用上次选择；4. 歌词不含翻译时切换入口隐藏或置灰不可用。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 翻译切换逻辑一致，原文与译文的排版间距由各端适配。 |
| [视觉/生态/安全等扩展维度] | 译文与原文需在视觉层级上区分清晰，避免混淆。 |
