# AR-05-02 专辑列表 ABC 快速索引
- AR-ID: AR-05-02
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑列表 ABC 快速索引 |
| [需求描述] | 专辑列表侧边提供 ABC 字母快速索引条，按专辑名首字母分组。用户拖动或点击索引条上的字母，列表即快速定位并滚动到以该字母开头的专辑分组。索引条随排序方式与专辑分布动态生成。 |
| [需求价值] | 在专辑数量较多时，让用户通过字母索引快速跳转到目标专辑，避免长距离滚动，提升查找效率。 |
| [需求场景] | 用户在含大量专辑的列表中，点击或拖动右侧字母索引条上的某个字母，列表立即滚动到对应字母开头的专辑位置。 |
| [目标用户] | 拥有较多专辑、需要快速定位的本地音乐用户。 |
| [限制约束] | 索引依赖专辑名首字母分组，需与当前排序方式匹配；非字母开头专辑归入特定分组（如 #）。 |
| [外部依赖] | 专辑列表数据与排序结果（提供首字母分组依据）。 |
| [性能指标] | 评测基线目标：索引拖动定位响应及时，滚动 ≥ 60fps。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 专辑列表侧边显示 ABC 字母索引条。 2. 点击或拖动字母可定位到以该字母开头的专辑分组。 3. 索引分组与当前专辑列表的首字母分布一致。 4. 非字母开头的专辑归入约定分组。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 索引交互为通用能力，各形态一致；车机以更大触控热区与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：侧边索引条与当前定位字母高亮反馈；交互：支持点击与拖动两种方式。 |
