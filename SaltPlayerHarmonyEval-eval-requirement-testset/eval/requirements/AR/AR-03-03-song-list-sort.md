# AR-03-03 歌曲列表排序
- AR-ID: AR-03-03
- 所属 SR: SR-03 歌曲管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲列表排序 |
| [需求描述] | 歌曲列表支持按多个维度排序，例如歌名、艺术家、专辑、添加时间、修改时间等，并可在升序与降序之间切换。用户选择排序方式后，列表立即按所选维度与方向重排，所选排序偏好被记忆并在再次进入时沿用。 |
| [需求价值] | 让用户按自己习惯的维度组织歌曲列表，更快找到目标歌曲，提升浏览与管理效率。 |
| [需求场景] | 用户打开排序菜单，选择「按添加时间倒序」，列表立即按最近添加在前重排；下次进入仍保持该排序。 |
| [目标用户] | 需要按特定顺序浏览或整理本地歌曲的用户。 |
| [限制约束] | 排序维度限于已扫描入库的歌曲元信息字段；同一时间仅生效一种排序维度与一个方向。 |
| [外部依赖] | 媒体库扫描数据（排序所依据的元信息）、排序偏好持久化存储。 |
| [性能指标] | 评测基线目标：切换排序后列表重排即时完成，无可感知卡顿。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 支持按歌名、艺术家、专辑、添加时间、修改时间等多维度排序。 2. 支持升序/降序切换。 3. 选择排序后列表立即按所选维度与方向重排。 4. 排序偏好被记忆，再次进入列表时沿用。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 排序为通用能力，各形态一致；车机以更大点选目标与简化菜单为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：当前排序维度与方向在菜单中明确标识；安全：仅对本地数据排序。 |
