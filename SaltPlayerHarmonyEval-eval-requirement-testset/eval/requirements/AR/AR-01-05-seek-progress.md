# AR-01-05 播放进度拖动与跳转
- AR-ID: AR-01-05
- 所属 SR: SR-01 播放控制与队列

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放进度拖动与跳转 |
| [需求描述] | 通过进度条让用户定位到曲目的任意播放位置。用户拖动进度滑块时，界面实时显示对应的时间位置；松手后播放从该位置跳转并继续。进度条同时随播放实时推进，准确反映当前播放进度。 |
| [需求价值] | 让用户快速跳到想听的片段或重听某段内容，是基础且高频的播放控制能力。 |
| [需求场景] | 用户在播放界面拖动进度条滑块到目标位置后松手，播放从该位置继续。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 跳转位置需限定在曲目时长范围内；拖动过程中应避免频繁跳转造成的卡顿。 |
| [外部依赖] | 系统多媒体播放能力（定位/seek）。 |
| [性能指标] | 松手后跳转响应迅速，目标位置定位准确。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 拖动进度滑块时界面实时显示对应的时间位置；2. 松手后播放从所选位置继续，定位准确；3. 未拖动时进度条随播放实时向前推进，反映真实进度；4. 跳转位置不会超出曲目时长范围。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机考虑驾驶安全，进度拖动交互可做简化；核心跳转能力一致。 |
| [视觉/生态/安全等扩展维度] | 拖动时建议提供清晰的时间气泡或刻度反馈，提升定位精度。 |
