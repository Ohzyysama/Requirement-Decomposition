# AR-03-02 歌曲列表搜索
- AR-ID: AR-03-02
- 所属 SR: SR-03 歌曲管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲列表搜索 |
| [需求描述] | 在歌曲列表内提供搜索能力，用户输入关键词后按歌名、艺术家等字段实时过滤列表，仅展示匹配的歌曲。清空关键词或退出搜索后恢复完整列表，搜索结果保持与原列表一致的列表项展示与交互能力。 |
| [需求价值] | 在大量本地歌曲中快速定位目标歌曲，免去逐条滚动查找，提升查找效率。 |
| [需求场景] | 用户在歌曲列表点击搜索框，输入歌名或艺术家关键词，列表实时收敛为匹配结果，点击即可播放或操作。 |
| [目标用户] | 本地歌曲数量较多、需要快速查找歌曲的用户。 |
| [限制约束] | 搜索范围限于当前歌曲列表内的歌曲；匹配以歌名、艺术家等文本字段为主；空关键词恢复全量列表。 |
| [外部依赖] | 媒体库扫描数据（被搜索的歌曲集合）、歌曲列表项展示能力（AR-03-01）。 |
| [性能指标] | 评测基线目标：输入关键词后过滤结果即时刷新，无可感知卡顿。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 输入关键词后列表实时过滤，仅显示匹配歌曲。 2. 匹配覆盖歌名与艺术家等主要文本字段。 3. 清空关键词或退出搜索后恢复完整列表。 4. 搜索结果项保持与原列表一致的展示与点击/菜单交互。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 搜索为通用能力，各形态一致；车机场景建议以更大输入控件与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：搜索框与结果列表一致风格；安全：仅在本地数据内搜索，不上传关键词。 |
