# AR-13-03 播放暂停淡入淡出
- AR-ID: AR-13-03
- 所属 SR: SR-13 音频输出

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放暂停淡入淡出 |
| [需求描述] | 在播放与暂停切换时对音量包络做淡入淡出处理，避免声音骤起骤停造成的爆音与突兀感。点击暂停时声音在短时间内平滑淡出后静止，恢复播放时从静音平滑渐入到正常音量。切换曲目、起播等场景同样应用一致的渐变过渡。 |
| [需求价值] | 消除播放/暂停瞬间的爆音与突兀感，让起停过渡更自然柔和，提升整体听感质量。 |
| [需求场景] | 用户点击暂停/播放、切换上一曲下一曲、起播或从中断中恢复播放时触发音量渐变过渡。 |
| [目标用户] | 全体听歌用户。 |
| [限制约束] | 淡入淡出时长固定（约 500 毫秒包络）；与音频焦点中断暂停共用同一渐变逻辑，避免叠加冲突。 |
| [外部依赖] | @ohos.multimedia.media(AVPlayer 音量渐变)。 |
| [性能指标] | 基线目标：淡出/淡入包络固定约 500ms（FADE_DURATION_MS），过渡平滑无可闻爆音。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 点击暂停时声音在约 500 毫秒内平滑淡出后停止，无突兀截断与爆音；2. 恢复播放时声音从静音平滑渐入至正常音量；3. 切换曲目或起播时同样应用一致的渐变过渡；4. 快速连续点击播放/暂停时渐变不产生音量跳变或异常残响。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端渐变逻辑一致，仅触发入口（按钮/系统媒体卡片/车载控制）形态不同。 |
| [视觉/生态/安全等扩展维度] | 渐变过渡与界面播放/暂停状态切换动效协调，保持听觉与视觉一致。 |
