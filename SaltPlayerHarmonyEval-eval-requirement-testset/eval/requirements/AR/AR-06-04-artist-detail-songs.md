# AR-06-04 艺术家详情歌曲列表（曲目项/歌曲数显示，支持随机播放、歌曲折叠，单首操作: 点击播放/添加下一首/更多菜单）
- AR-ID: AR-06-04
- 所属 SR: SR-06 艺术家管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 艺术家详情歌曲列表（曲目项/歌曲数显示，支持随机播放、歌曲折叠，单首操作: 点击播放/添加下一首/更多菜单） |
| [需求描述] | 艺术家详情页展示该艺术家名下的歌曲列表，每个曲目项展示歌曲信息并显示总歌曲数，提供「随机播放」一键随机播放全部歌曲，并支持歌曲区块的折叠/展开。每首歌支持单首操作：点击该曲直接播放、添加为下一首播放、以及打开「更多」菜单进行扩展操作。 |
| [需求价值] | 让用户在艺术家详情页集中浏览并播放其全部歌曲，支持随机播放与折叠整理，并对单首歌曲快速执行常用操作，提升收听效率。 |
| [需求场景] | 用户进入艺术家详情，查看其歌曲数与曲目列表，点击「随机播放」收听全部，或折叠歌曲区块查看专辑区块；对某首歌点击播放、添加为下一首或打开更多菜单。 |
| [目标用户] | 查看并播放某位艺术家作品的普通用户。 |
| [限制约束] | 歌曲数为该艺术家名下本地已扫描歌曲数量；随机播放作用于该艺术家全部歌曲；单首操作复用歌曲列表项交互能力。 |
| [外部依赖] | 本地歌曲库（歌曲数据与歌曲数）、播放队列与播放控制、歌曲更多菜单能力。 |
| [性能指标] | 评测基线目标：列表滚动 ≥ 60fps；点击播放/随机播放响应即时。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 正确展示该艺术家歌曲列表及总歌曲数。 2. 点击「随机播放」随机播放该艺术家全部歌曲。 3. 支持歌曲区块折叠/展开。 4. 单首歌曲支持点击播放、添加下一首播放。 5. 单首歌曲可打开「更多」菜单执行扩展操作。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 歌曲列表与单首操作为通用能力，各形态一致；车机以更大字号与简化菜单交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：歌曲数标识、折叠/展开指示、随机播放入口；交互：点击播放、添加下一首、更多菜单。 |
