# AR-05-04 专辑详情头部展示（封面/专辑名/专辑艺术家/歌曲数/总时长/年份）
- AR-ID: AR-05-04
- 所属 SR: SR-05 专辑管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 专辑详情头部展示（封面/专辑名/专辑艺术家/歌曲数/总时长/年份） |
| [需求描述] | 进入专辑详情页后，页面头部集中展示该专辑的概览信息：专辑封面、专辑名称、专辑艺术家、专辑包含的歌曲数量、所有歌曲累计的总时长以及专辑发行年份。各项信息从专辑聚合数据中读取，缺失项（如年份或专辑艺术家）按约定方式省略或回退展示。 |
| [需求价值] | 让用户在进入专辑后一眼掌握专辑的整体信息，快速判断专辑归属、规模与时长，提升专辑浏览体验。 |
| [需求场景] | 用户从专辑列表点击某专辑进入详情页，头部立即展示封面、专辑名、专辑艺术家、歌曲数、总时长与年份。 |
| [目标用户] | 查看专辑整体信息的本地音乐用户。 |
| [限制约束] | 年份、专辑艺术家等信息缺失时省略或回退展示；歌曲数与总时长由专辑内歌曲实时聚合得出。 |
| [外部依赖] | 专辑封面缓存与封面提取、媒体库专辑聚合数据（歌曲数/总时长/年份/专辑艺术家）。 |
| [性能指标] | 评测基线目标：进入详情页头部信息加载及时，封面异步加载不阻塞页面渲染。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 详情页头部显示专辑封面（缺失时显示占位图）。 2. 显示专辑名称与专辑艺术家。 3. 显示歌曲数量与所有歌曲累计总时长，且与实际数据一致。 4. 显示专辑年份，缺失时按约定省略或回退。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 头部信息展示为通用能力，各形态一致；平板/车机大屏下可采用更宽松的头部布局。 |
| [视觉/生态/安全等扩展维度] | 视觉：封面为头部主视觉，专辑名/艺术家/统计信息分层排布；安全：仅展示本地已扫描专辑信息。 |
