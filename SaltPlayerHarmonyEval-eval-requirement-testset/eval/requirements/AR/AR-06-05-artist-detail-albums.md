# AR-06-05 艺术家详情专辑区块（封面/专辑名/专辑艺术家/歌曲数，点击跳转专辑详情页）
- AR-ID: AR-06-05
- 所属 SR: SR-06 艺术家管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 艺术家详情专辑区块（封面/专辑名/专辑艺术家/歌曲数，点击跳转专辑详情页） |
| [需求描述] | 艺术家详情页展示该艺术家名下的专辑区块，每个专辑项展示专辑封面、专辑名、专辑艺术家与该专辑的歌曲数。点击某个专辑项即跳转到对应专辑的详情页，便于按专辑维度浏览该艺术家作品。 |
| [需求价值] | 让用户在艺术家详情页按专辑维度浏览其作品，快速识别各专辑并进入专辑详情，丰富浏览路径。 |
| [需求场景] | 用户在艺术家详情页浏览专辑区块，通过封面与专辑名辨认目标专辑，点击进入该专辑详情页查看完整曲目。 |
| [目标用户] | 按专辑浏览某位艺术家作品的普通用户。 |
| [限制约束] | 专辑封面缺失时显示默认占位图；歌曲数为该专辑本地已扫描歌曲数量；点击专辑项跳转专辑详情页。 |
| [外部依赖] | 封面缓存与封面提取（专辑封面来源）、本地专辑数据与歌曲统计、专辑详情页路由。 |
| [性能指标] | 评测基线目标：专辑区块滚动 ≥ 60fps；封面异步加载不阻塞滚动。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 专辑区块正确展示各专辑封面、专辑名、专辑艺术家与歌曲数。 2. 专辑封面缺失时显示默认占位图。 3. 点击专辑项跳转到对应专辑详情页。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线；平板/车机为扩展。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 专辑区块展示与跳转为通用能力，各形态一致；平板/车机可利用更大空间增加每行专辑列数。 |
| [视觉/生态/安全等扩展维度] | 视觉：专辑封面、专辑名/专辑艺术家/歌曲数文字、默认占位图；交互：点击跳转专辑详情。 |
