# AR-13-02 音量平衡
- AR-ID: AR-13-02
- 所属 SR: SR-13 音频输出

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 音量平衡 |
| [需求描述] | 提供左右声道音量平衡调节，让用户在听感偏移或单侧耳机/外放音量不均时自行校正。用户通过平衡滑杆在左右之间连续调节，居中为左右等响，向一侧偏移则相应增强该侧、减弱另一侧。设置即时生效并在下次播放时保持。 |
| [需求价值] | 满足听力差异、设备声道不均或个人偏好用户对左右声道平衡的个性化校正需求，提升听感舒适度。 |
| [需求场景] | 用户在音频输出设置中拖动左右声道平衡滑杆，调整后立即试听效果。 |
| [目标用户] | 对声道平衡有校正需求或个性化偏好的听歌用户。 |
| [限制约束] | 平衡调节作用于当前输出链路，受输出设备（耳机/外放）本身能力限制；极端偏移时单侧可能完全静音。 |
| [外部依赖] | @ohos.multimedia.media(AVPlayer 音量/声道增益)、@ohos.multimedia.audio。 |
| [性能指标] | 基线目标：拖动调节到听感变化延迟<100ms；设置持久化即时生效。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 拖动左右声道平衡滑杆时，对应声道的相对音量随之实时变化；2. 滑杆居中时左右声道响度一致；3. 向左或向右调节后，听感明显偏向对应一侧；4. 调节后的平衡值被保存，下次播放仍保持上次设置。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 车机多扬声器布局下平衡含义与双声道耳机不同，需结合车载声场适配；交互入口差异由各端 UI 适配。 |
| [视觉/生态/安全等扩展维度] | 平衡滑杆视觉与系统设置控件风格保持一致，避免误操作导致单侧静音。 |
