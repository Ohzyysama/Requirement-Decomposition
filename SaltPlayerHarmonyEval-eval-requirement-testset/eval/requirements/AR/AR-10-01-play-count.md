# AR-10-01 播放次数统计
- AR-ID: AR-10-01
- 所属 SR: SR-10 播放统计

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 播放次数统计 |
| [需求描述] | 记录并统计每首歌曲的累计播放次数，并以歌曲列表的形式展示统计结果。用户可据此查看哪些歌曲被播放得最多，列表中每一项呈现歌曲信息及其对应的播放次数。统计随用户实际播放行为持续累加，列表可作为了解个人听歌偏好的入口。 |
| [需求价值] | 量化用户的听歌偏好，帮助用户回顾常听曲目，并为发现高频歌曲提供直观入口。 |
| [需求场景] | 用户进入播放统计页面，查看以播放次数统计的歌曲列表；播放歌曲后对应次数随之累加。 |
| [目标用户] | 关注个人听歌习惯与高频曲目的用户。 |
| [限制约束] | 统计依据应用内实际播放行为累加；非应用内播放或异常中断的计次以既有播放计数策略为准。 |
| [外部依赖] | 本地播放记录/计数持久化、歌曲列表展示组件。 |
| [性能指标] | 基线目标：进入统计页后列表数据加载与渲染<300ms（常规库规模）。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 每首歌曲的累计播放次数被正确记录并展示；2. 统计结果以歌曲列表形式呈现，列表项含歌曲信息与对应播放次数；3. 播放某歌曲后其播放次数相应增加并在重新进入统计页时反映；4. 列表中的歌曲可点击播放或触发后续操作；5. 无任何播放记录时给出明确的空态而非报错。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 计数与列表展示逻辑各端一致；平板可借更大屏幅展示更多列信息，车机以简化只读列表呈现。 |
| [视觉/生态/安全等扩展维度] | 播放次数仅本地统计与存储，不上传用户听歌数据；列表样式遵循应用歌曲列表统一视觉规范。 |
