# AR-03-01 歌曲列表项展示（封面/音质/歌名/艺术家）
- AR-ID: AR-03-01
- 所属 SR: SR-03 歌曲管理

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 歌曲列表项展示（封面/音质/歌名/艺术家） |
| [需求描述] | 歌曲列表中每一行展示一首歌曲的核心信息：左侧显示歌曲封面（是否显示受「列表显示封面」开关控制，关闭时不占位），歌名旁可显示音质徽标（高品质/超高品质/超高解析度，普通音质不显示），主标题显示歌名或在「显示文件名」开关开启时显示文件名，副标题显示艺术家并在信息缺失时回退为文件名。正在播放的歌曲标题与副标题以高亮色显示，失效或不可用歌曲文字置灰；每行右侧默认显示「更多」按钮，进入多选模式后替换为勾选框。 |
| [需求价值] | 让用户一眼获取每首歌的封面、名称、艺术家与音质信息，并直观识别当前播放与失效歌曲，提升浏览效率。 |
| [需求场景] | 用户在歌曲列表滚动浏览，通过封面、歌名、音质徽标快速辨认歌曲，并看到正在播放的歌曲被高亮、失效歌曲被置灰。 |
| [目标用户] | 浏览本地歌曲列表的普通用户。 |
| [限制约束] | 普通音质不显示徽标；封面与文件名显示受对应开关控制；多选模式下「更多」按钮替换为勾选框；失效歌曲仅置灰展示不阻断点击。 |
| [外部依赖] | 封面缓存与封面提取（封面图来源）、列表显示偏好开关（封面/文件名）、媒体库扫描数据（歌曲元信息）。 |
| [性能指标] | 评测基线目标：列表滚动 ≥ 60fps；封面异步加载不阻塞滚动。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 每行正确显示封面（开关关闭时不显示）、歌名（开关开启时显示文件名）、艺术家副标题，必要时副标题回退为文件名。 2. 高品质/超高品质/超高解析度歌曲显示对应徽标，普通音质无徽标。 3. 正在播放的歌曲标题与副标题高亮显示。 4. 失效歌曲文字置灰但仍可点击。 5. 默认显示「更多」按钮，进入多选模式后该位置改为勾选框并反映选中状态。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 列表项展示为通用能力，各形态一致；车机以更大字号与简化交互为主。 |
| [视觉/生态/安全等扩展维度] | 视觉：高亮色标识当前播放、灰色标识失效、音质徽标小图标；安全：仅展示本地已扫描歌曲信息。 |
