# AR-04-01 加载内嵌/独立 LRC 歌词
- AR-ID: AR-04-01
- 所属 SR: SR-04 歌词显示

| 字段 | 填写内容 |
|------|----------|
| [需求标题] | 加载内嵌/独立 LRC 歌词 |
| [需求描述] | 为当前曲目自动加载并解析歌词。优先读取音频文件内嵌的歌词标签，若无内嵌歌词则按同名规则在歌曲所在目录查找并加载独立的 .lrc 文件。解析后将带时间戳的歌词行供后续滚动与高亮使用；当两类歌词均不存在时，给出「暂无歌词」的友好占位提示。 |
| [需求价值] | 让用户在播放本地音乐时也能看到歌词，覆盖内嵌与外挂两种常见歌词来源，提升听歌体验完整度。 |
| [需求场景] | 用户切歌或进入播放/歌词界面时，应用自动为当前曲目加载内嵌歌词或同目录下的独立 LRC 文件并展示。 |
| [目标用户] | 希望边听歌边看歌词的本地音乐用户。 |
| [限制约束] | 仅支持标准 LRC 时间戳格式；内嵌歌词优先于独立文件；无可用歌词时仅显示占位提示而不报错。 |
| [外部依赖] | 音频元数据读取能力、@ohos.file.fs（独立 LRC 文件读取）、文件访问授权。 |
| [性能指标] | 基线目标：切歌后歌词加载与首屏渲染<300ms；解析不阻塞 UI 主线程。 |
| [功耗指标] | 继承所属 SR。 |
| [ROM&RAM] | 继承所属 SR。 |
| [验收标准] | 1. 含内嵌歌词的曲目播放时正确加载并展示内嵌歌词；2. 无内嵌歌词但存在同名 .lrc 文件时，加载并展示该独立歌词；3. 内嵌与独立歌词同时存在时优先使用内嵌歌词；4. 两类歌词均不存在时显示「暂无歌词」占位且不影响正常播放。 |
| [验收设备] | HarmonyOS NEXT（API 12+）手机为主基线。 |
| [适用产品] | 手机 / 平板 / 车机。 |
| [适用产品差异分析] | 各端歌词加载逻辑一致，展示区域大小与排版由各端 UI 适配。 |
| [视觉/生态/安全等扩展维度] | 独立 LRC 文件读取需遵循系统文件访问授权；解析异常需容错不崩溃。 |
