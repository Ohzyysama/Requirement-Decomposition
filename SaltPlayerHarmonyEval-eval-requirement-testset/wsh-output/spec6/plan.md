# 隐藏状态栏SPEC

设置-用户界面中的隐藏状态栏开关，默认关闭，打开后隐藏手机上部状态栏。该功能与沉浸模式存在联动关系。

## 场景一

### 场景概述

用户打开"隐藏状态栏"开关，状态栏立即隐藏

### 场景逻辑步骤

- 1. 用户进入"设置"页面，找到"用户界面"分类
- 2. 用户点击"隐藏状态栏"开关，将其从关闭状态切换为开启状态
- 3. 开关状态持久化存储，记录为开启
- 4. 手机上部状态栏立即隐藏

## 场景二

### 场景概述

用户关闭"隐藏状态栏"开关，且当前沉浸模式未开启，状态栏立即恢复显示

### 场景逻辑步骤

- 1. 用户进入"设置"页面，找到"用户界面"分类
- 2. 用户点击"隐藏状态栏"开关，将其从开启状态切换为关闭状态
- 3. 开关状态持久化存储，记录为关闭
- 4. 当前沉浸模式未开启，手机上部状态栏立即恢复显示

## 场景三

### 场景概述

用户关闭"隐藏状态栏"开关，但当前沉浸模式已开启，状态栏保持隐藏

### 场景逻辑步骤

- 1. 用户进入"设置"页面，找到"用户界面"分类
- 2. 用户点击"隐藏状态栏"开关，将其从开启状态切换为关闭状态
- 3. 开关状态持久化存储，记录为关闭
- 4. 由于当前沉浸模式已开启，手机上部状态栏保持隐藏状态不变

## 场景四

### 场景概述

应用重新启动时，"隐藏状态栏"开关为开启状态，状态栏在启动后自动隐藏

### 场景逻辑步骤

- 1. 用户此前已将"隐藏状态栏"开关设置为开启状态
- 2. 用户关闭应用后重新启动应用
- 3. 应用启动时读取持久化存储中的开关状态，识别为开启
- 4. 应用主界面加载完成后，手机上部状态栏自动隐藏

## 场景五

### 场景概述

"隐藏状态栏"开关为开启状态时，弹窗打开后状态栏保持隐藏

### 场景逻辑步骤

- 1. 用户此前已将"隐藏状态栏"开关设置为开启状态，当前状态栏处于隐藏状态
- 2. 用户触发任意底部弹窗（如更多操作菜单、歌单选择弹窗等）
- 3. 弹窗打开时读取"隐藏状态栏"开关状态，识别为开启
- 4. 状态栏保持隐藏，不会因弹窗打开而闪现

## 场景六

### 场景概述

"隐藏状态栏"开关为关闭状态且沉浸模式未开启时，弹窗打开后状态栏保持显示

### 场景逻辑步骤

- 1. 用户当前"隐藏状态栏"开关为关闭状态，且沉浸模式未开启，状态栏正常显示
- 2. 用户触发任意底部弹窗（如更多操作菜单、歌单选择弹窗等）
- 3. 弹窗打开时读取"隐藏状态栏"开关状态及沉浸模式状态，均为关闭
- 4. 状态栏保持正常显示
