# 主界面壁纸SPEC

设置-用户界面-主界面壁纸-选择图片，将图片资源拷贝到应用文件夹下面，为应用设置背景图（不包含播放页以及播放队列页）

## 场景一

### 场景概述

用户首次设置主界面壁纸，选择图片成功并应用为主界面背景

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"选项
- 2. 在"用户界面"页面中点击"主界面壁纸"选项，进入主界面壁纸设置页面
- 3. 用户点击选择图片按钮，系统弹出图片选择器
- 4. 用户在图片选择器中选择一张图片并确认
- 5. 应用将所选图片资源拷贝到应用文件夹下的壁纸存储目录中
- 6. 应用持久化保存壁纸图片的文件路径
- 7. 主界面立即显示所选图片作为背景壁纸，壁纸以裁剪方式填充整个主界面区域
- 8. 播放页及播放队列页不受影响，不显示该壁纸背景

## 场景二

### 场景概述

用户已设置过主界面壁纸，重新选择新图片替换现有壁纸

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"选项
- 2. 在"用户界面"页面中点击"主界面壁纸"选项，进入主界面壁纸设置页面
- 3. 用户点击选择图片按钮，系统弹出图片选择器
- 4. 用户在图片选择器中选择一张新图片并确认
- 5. 应用删除应用文件夹中已有的旧壁纸文件
- 6. 应用将新选择的图片资源拷贝到应用文件夹下的壁纸存储目录中
- 7. 应用更新持久化保存的壁纸图片文件路径为新文件路径
- 8. 主界面立即刷新显示新选择的图片作为背景壁纸，壁纸以裁剪方式填充整个主界面区域
- 9. 播放页及播放队列页不受影响，不显示该壁纸背景

## 场景三

### 场景概述

用户打开图片选择器后取消选择，不进行任何壁纸变更

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"选项
- 2. 在"用户界面"页面中点击"主界面壁纸"选项，进入主界面壁纸设置页面
- 3. 用户点击选择图片按钮，系统弹出图片选择器
- 4. 用户未选择任何图片，通过返回或取消操作关闭图片选择器
- 5. 应用不执行任何壁纸相关操作，主界面壁纸保持当前状态不变（若之前有壁纸则继续显示，若之前无壁纸则继续无壁纸）

## 场景四

### 场景概述

用户删除已设置的主界面壁纸，主界面恢复为无壁纸状态

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"选项
- 2. 在"用户界面"页面中点击"主界面壁纸"选项，进入主界面壁纸设置页面
- 3. 用户点击删除壁纸按钮
- 4. 应用删除应用文件夹中已保存的壁纸文件
- 5. 应用清除持久化保存的壁纸文件路径（置为空）
- 6. 主界面立即移除壁纸背景显示，恢复为默认背景状态

## 场景五

### 场景概述

壁纸仅在主界面显示，播放页及播放队列页不显示壁纸背景

### 场景逻辑步骤

- 1. 用户已成功设置主界面壁纸
- 2. 用户在主界面浏览各页面（如歌曲列表、歌单、专辑、艺术家、文件夹等页面），这些页面均显示壁纸作为背景
- 3. 用户进入播放页时，播放页不显示主界面壁纸，播放页使用自身独立的背景样式
- 4. 用户进入播放队列页时，播放队列页不显示主界面壁纸，播放队列页使用自身独立的背景样式

## 场景六

### 场景概述

应用重启后，已设置的主界面壁纸仍然生效显示

### 场景逻辑步骤

- 1. 用户已成功设置主界面壁纸
- 2. 用户关闭应用或应用被系统回收
- 3. 用户重新启动应用
- 4. 应用启动时读取持久化保存的壁纸文件路径
- 5. 应用根据文件路径加载壁纸图片，主界面继续显示之前设置的壁纸背景
