# 播放界面删除图片SPEC

在设置-用户界面-播放界面壁纸中，用户可以删除已设置的播放界面背景图片，删除后播放页及播放队列页恢复默认背景（动态流光效果），删除的是拷贝到应用文件夹下的图片资源副本。

## 场景一

### 场景概述

用户已设置播放界面背景图片，点击删除并确认，图片被删除，播放页及播放队列页恢复默认背景

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，在"播放界面背景图片"区域中，当前已设置了自定义背景图片，此时显示"删除当前图片"选项
- 2. 用户点击"删除当前图片"选项
- 3. 弹出确认对话框，询问用户是否确认删除
- 4. 用户点击确认按钮
- 5. 删除应用文件夹下存储的背景图片资源副本文件
- 6. 清除已保存的背景图片路径配置
- 7. 应用重新启动
- 8. 重启后，播放页背景恢复为默认的动态流光效果（基于当前播放歌曲封面生成的动态色彩渐变）
- 9. 播放队列页背景同步恢复为默认的动态流光效果，因为播放队列页与播放页共享同一背景

## 场景二

### 场景概述

用户已设置播放界面背景图片，点击删除但在确认对话框中取消操作

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，在"播放界面背景图片"区域中，当前已设置了自定义背景图片，此时显示"删除当前图片"选项
- 2. 用户点击"删除当前图片"选项
- 3. 弹出确认对话框，询问用户是否确认删除
- 4. 用户点击取消按钮
- 5. 确认对话框关闭，背景图片保持不变，播放页及播放队列页继续显示自定义背景图片

## 场景三

### 场景概述

用户未设置播放界面背景图片时，删除选项不可见

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面-播放界面壁纸"页面
- 2. 当前未设置自定义背景图片（背景图片路径为空）
- 3. "删除当前图片"选项不显示，用户无法执行删除操作
- 4. 此时播放页及播放队列页显示默认的动态流光效果背景
