# 播放界面选择图片SPEC

设置-用户界面-播放界面壁纸页面选择图片，将图片资源拷贝到应用文件夹下面，为播放页以及播放队列页设置背景图。

## 场景一

### 场景概述

用户在设置中选择图片并成功设置为播放界面背景图

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"，点击"播放界面壁纸"按钮进入页面
- 2. 用户点击"选择图片"按钮
- 3. 系统弹出文件选择器，仅展示图片类型文件供用户选择
- 4. 用户选中一张图片并确认
- 5. 读取用户选中的图片数据
- 6. 如果当前已存在播放界面背景图文件，则先删除该文件
- 7. 将图片数据写入应用文件夹下的指定路径（covers/player_screen_cover）
- 8. 将该图片文件的路径持久化存储
- 9. 播放页和播放队列页的背景图实时更新为用户选择的图片

## 场景二

### 场景概述

用户打开图片选择器后取消选择，不做任何变更

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"，点击"播放界面壁纸"按钮进入页面
- 2. 用户点击"选择图片"按钮
- 3. 系统弹出文件选择器
- 4. 用户未选择任何图片，通过返回或取消操作关闭文件选择器
- 5. 播放界面背景图保持不变，不执行任何写入或删除操作

## 场景三

### 场景概述

用户删除当前已设置的播放界面背景图，播放页恢复默认效果

### 场景逻辑步骤

- 1. 用户进入"设置"页面，点击"用户界面"，点击"播放界面壁纸"按钮进入页面
- 2. 用户点击"删除当前图片"按钮
- 3. 删除应用文件夹下已存储的播放界面背景图文件
- 4. 清除持久化存储中的背景图路径记录
- 5. 应用重新启动以刷新界面
- 6. 播放页和播放队列页不再显示自定义背景图，恢复为默认的动态流光效果

## 场景四

### 场景概述

进入播放页时，已设置自定义背景图的展示效果

### 场景逻辑步骤

- 1. 用户已通过设置选择并保存了播放界面背景图
- 2. 用户进入播放页
- 3. 播放页读取持久化存储中的背景图路径
- 4. 当背景图路径不为空时，以该图片作为播放页的全屏背景，图片以裁剪方式填充整个屏幕
- 5. 播放队列页作为播放页的子页面，共享同一背景图，展示效果与播放页一致
- 6. 设置自定义背景图后，默认的动态流光效果被禁用

## 场景五

### 场景概述

进入播放页时，未设置自定义背景图的展示效果

### 场景逻辑步骤

- 1. 用户未设置播放界面背景图，或已删除之前设置的背景图
- 2. 用户进入播放页
- 3. 播放页读取持久化存储中的背景图路径
- 4. 当背景图路径为空时，播放页显示默认的动态流光效果作为背景
- 5. 播放队列页同样显示默认的动态流光效果
