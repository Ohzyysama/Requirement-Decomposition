# 显示关闭按钮SPEC

设置-通知中提供"显示关闭按钮"开关，默认开启。开启后，通知栏、状态栏、实况窗对应的播放器卡片在切换到下一曲按钮右侧增加关闭按钮，点击后可以关闭当前应用。

## 场景一

### 场景概述

"显示关闭按钮"开关默认开启，播放器卡片上显示关闭按钮

### 场景逻辑步骤

- 1. 用户首次安装应用或未修改过该设置，"显示关闭按钮"开关默认为开启状态
- 2. 当有音乐播放时，通知栏、状态栏、实况窗对应的播放器卡片上，在"切换到下一曲"按钮的右侧显示一个关闭按钮
- 3. 关闭按钮始终位于播放控制按钮组的最右侧位置

## 场景二

### 场景概述

用户点击播放器卡片上的关闭按钮，应用被关闭

### 场景逻辑步骤

- 1. 用户在通知栏、状态栏或实况窗的播放器卡片上点击关闭按钮
- 2. 当前播放的音乐暂停
- 3. 移除通知栏中的播放器通知
- 4. 停止音乐播放服务
- 5. 终止应用进程，应用完全退出

## 场景三

### 场景概述

用户手动关闭"显示关闭按钮"开关，播放器卡片上不再显示关闭按钮

### 场景逻辑步骤

- 1. 用户进入"设置-通知"页面
- 2. 用户将"显示关闭按钮"开关从开启切换为关闭
- 3. 通知栏、状态栏、实况窗对应的播放器卡片立即刷新
- 4. 刷新后的播放器卡片上不再显示关闭按钮，"切换到下一曲"按钮成为播放控制按钮组的最右侧按钮

## 场景四

### 场景概述

用户手动开启"显示关闭按钮"开关，播放器卡片上重新显示关闭按钮

### 场景逻辑步骤

- 1. 用户进入"设置-通知"页面
- 2. 用户将"显示关闭按钮"开关从关闭切换为开启
- 3. 通知栏、状态栏、实况窗对应的播放器卡片立即刷新
- 4. 刷新后的播放器卡片上在"切换到下一曲"按钮右侧重新显示关闭按钮

## 场景五

### 场景概述

"显示关闭按钮"开关关闭时，用户无法通过播放器卡片关闭应用

### 场景逻辑步骤

- 1. 用户已将"显示关闭按钮"开关设置为关闭
- 2. 通知栏、状态栏、实况窗对应的播放器卡片上不存在关闭按钮
- 3. 用户无法通过播放器卡片上的操作来关闭应用，音乐播放服务持续运行
