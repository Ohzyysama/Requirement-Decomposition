# 删除主界面壁纸SPEC

删除应用主界面的背景壁纸图片，还原应用默认背景。删除的是拷贝到应用文件夹下面的图片资源副本。

## 场景一

### 场景概述

用户确认删除当前已设置的主界面壁纸，壁纸文件存在，成功删除并还原默认背景

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，点击"主界面壁纸"选项进入壁纸设置页面
- 2. 当前已设置主界面壁纸时，页面显示"删除当前图片"按钮
- 3. 用户点击"删除当前图片"按钮
- 4. 弹出确认对话框，询问用户是否确认删除
- 5. 用户点击确认按钮
- 6. 删除应用文件夹中存储的壁纸图片文件副本
- 7. 清除壁纸路径配置，将壁纸配置置为空
- 8. 主界面壁纸立即消失，还原为应用默认背景

## 场景二

### 场景概述

用户取消删除主界面壁纸操作，壁纸保持不变

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，点击"主界面壁纸"选项进入壁纸设置页面
- 2. 当前已设置主界面壁纸时，页面显示"删除当前图片"按钮
- 3. 用户点击"删除当前图片"按钮
- 4. 弹出确认对话框，询问用户是否确认删除
- 5. 用户点击取消按钮
- 6. 关闭确认对话框，壁纸配置和壁纸文件均保持不变，主界面壁纸继续显示

## 场景三

### 场景概述

用户确认删除主界面壁纸，但壁纸文件已不存在（被外部删除），仍然清除配置并还原默认背景

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，点击"主界面壁纸"选项进入壁纸设置页面
- 2. 当前壁纸配置不为空（壁纸路径仍有记录），页面显示"删除当前图片"按钮
- 3. 用户点击"删除当前图片"按钮
- 4. 弹出确认对话框，询问用户是否确认删除
- 5. 用户点击确认按钮
- 6. 尝试删除壁纸文件，但文件已不存在，跳过文件删除步骤
- 7. 清除壁纸路径配置，将壁纸配置置为空
- 8. 主界面还原为应用默认背景

## 场景四

### 场景概述

当前未设置主界面壁纸时，不显示删除按钮，用户无法执行删除操作

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，点击"主界面壁纸"选项进入壁纸设置页面
- 2. 当前未设置主界面壁纸（壁纸配置为空）
- 3. 页面不显示"删除当前图片"按钮，用户无法触发删除操作
