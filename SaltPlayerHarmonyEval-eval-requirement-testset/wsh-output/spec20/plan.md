# 卡拉OK歌词动画兼容策略SPEC

设置-歌词-歌词界面中提供卡拉OK歌词动画兼容策略选项，控制歌词逐字特效的显示范围。播放页歌词设置同步提供该选项，两处设置保持一致。默认值为"当前行"。

## 场景一

### 场景概述

用户选择"当前行"策略，当前歌曲中部分歌词行带有单字时间戳，部分没有

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"当前行"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，逐行判断每一行歌词是否带有单字时间戳
- 5. 对于带有单字时间戳的歌词行，显示逐字卡拉OK特效动画
- 6. 对于没有单字时间戳的歌词行，不显示逐字卡拉OK特效动画

## 场景二

### 场景概述

用户选择"当前行"策略，当前歌曲中所有歌词行均没有单字时间戳

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"当前行"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，逐行判断每一行歌词是否带有单字时间戳
- 5. 由于所有歌词行均没有单字时间戳，所有歌词行都不显示逐字卡拉OK特效动画

## 场景三

### 场景概述

用户选择"扩展全部"策略，当前歌曲中至少有一行歌词带有单字时间戳

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"扩展全部"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，检测当前歌曲的歌词中是否存在至少一行带有单字时间戳的歌词行
- 5. 由于当前歌曲中存在至少一行带有单字时间戳的歌词行，所有歌词行都显示逐字卡拉OK特效动画（包括没有单字时间戳的歌词行）

## 场景四

### 场景概述

用户选择"扩展全部"策略，当前歌曲中所有歌词行均没有单字时间戳

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"扩展全部"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，检测当前歌曲的歌词中是否存在至少一行带有单字时间戳的歌词行
- 5. 由于当前歌曲中没有任何歌词行带有单字时间戳，所有歌词行都不显示逐字卡拉OK特效动画

## 场景五

### 场景概述

用户选择"总是"策略，当前歌曲的歌词中有单字时间戳的歌词行

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"总是"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，无需检测歌词行是否带有单字时间戳
- 5. 所有歌词行都显示逐字卡拉OK特效动画

## 场景六

### 场景概述

用户选择"总是"策略，当前歌曲的歌词中没有任何单字时间戳

### 场景逻辑步骤

- 1. 用户进入"设置-歌词-歌词界面"页面，或在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择"总是"
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词显示时，无需检测歌词行是否带有单字时间戳
- 5. 所有歌词行都显示逐字卡拉OK特效动画（即使没有任何单字时间戳数据）

## 场景七

### 场景概述

用户在播放页歌词设置中修改策略，设置页同步更新

### 场景逻辑步骤

- 1. 用户在播放页打开歌词设置面板
- 2. 用户在"卡拉OK歌词动画兼容策略"选项中选择一个新的策略
- 3. 策略设置立即生效并持久化保存
- 4. 播放页歌词立即按照新策略刷新逐字特效显示
- 5. 用户进入"设置-歌词-歌词界面"页面时，"卡拉OK歌词动画兼容策略"选项显示的值与播放页歌词设置中选择的值一致

## 场景八

### 场景概述

用户在设置页修改策略，播放页歌词实时同步更新

### 场景逻辑步骤

- 1. 用户正在播放页查看歌词
- 2. 用户进入"设置-歌词-歌词界面"页面
- 3. 用户在"卡拉OK歌词动画兼容策略"选项中选择一个新的策略
- 4. 策略设置立即生效并持久化保存
- 5. 用户返回播放页时，歌词已按照新策略刷新逐字特效显示
- 6. 播放页歌词设置面板中"卡拉OK歌词动画兼容策略"选项显示的值与设置页中选择的值一致

## 场景九

### 场景概述

用户从未修改过该策略设置，使用默认值

### 场景逻辑步骤

- 1. 用户首次使用应用或从未修改过"卡拉OK歌词动画兼容策略"设置
- 2. 该策略的默认值为"当前行"
- 3. 播放页歌词显示时，按照"当前行"策略的逻辑执行：只为当前歌曲的歌词中有单字时间戳的歌词行显示逐字卡拉OK特效动画

## 场景十

### 场景概述

用户切换歌曲后，策略基于新歌曲的歌词数据重新生效

### 场景逻辑步骤

- 1. 用户已设置某一策略（当前行、扩展全部或总是）
- 2. 用户通过上一首、下一首、播放队列切歌或自动切歌等方式切换到新歌曲
- 3. 新歌曲的歌词加载完成后，基于新歌曲的歌词数据重新按照当前策略判断每行是否显示逐字卡拉OK特效动画
- 4. 如果策略为"当前行"，则根据新歌曲中每行歌词是否带有单字时间戳分别判断
- 5. 如果策略为"扩展全部"，则根据新歌曲中是否存在至少一行带有单字时间戳的歌词行来决定所有行是否显示
- 6. 如果策略为"总是"，则新歌曲所有歌词行都显示逐字卡拉OK特效动画
