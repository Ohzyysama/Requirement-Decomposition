# 桌面歌词SPEC

设置-歌词-桌面歌词，开关默认关闭，打开后在设备桌面显示歌词，支持手势拖动歌词调整歌词位置。

## 场景一

### 场景概述

用户在歌词设置页面开启桌面歌词开关，且已授予悬浮窗权限

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面，找到"桌面歌词"区域
- 2. 桌面歌词开关默认为关闭状态
- 3. 用户点击桌面歌词开关，请求开启
- 4. 检测到已授予悬浮窗权限
- 5. 在设备桌面上显示桌面歌词悬浮窗，悬浮窗显示当前播放歌曲的歌词文本；若当前无歌词则显示"暂无歌词"
- 6. 弹出toast提示"桌面歌词已开启"
- 7. 开关状态更新为开启，并持久化保存开关状态

## 场景二

### 场景概述

用户在歌词设置页面开启桌面歌词开关，但未授予悬浮窗权限

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面，找到"桌面歌词"区域
- 2. 用户点击桌面歌词开关，请求开启
- 3. 检测到未授予悬浮窗权限
- 4. 弹出toast提示"没有悬浮窗权限"
- 5. 桌面歌词悬浮窗不会显示，开关状态保持关闭

## 场景三

### 场景概述

用户在歌词设置页面关闭桌面歌词开关

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面，桌面歌词开关当前为开启状态
- 2. 用户点击桌面歌词开关，请求关闭
- 3. 桌面歌词悬浮窗从设备桌面上消失
- 4. 弹出toast提示"桌面歌词已关闭"
- 5. 开关状态更新为关闭，并持久化保存开关状态

## 场景四

### 场景概述

桌面歌词已开启，用户通过手势拖动歌词调整位置

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示在设备桌面上，且未处于锁定状态
- 2. 用户手指按住歌词文本区域
- 3. 用户手指在屏幕上移动，桌面歌词悬浮窗跟随手指实时移动，支持水平和垂直方向的自由拖动
- 4. 用户手指抬起，歌词停留在最终拖动位置
- 5. 歌词的垂直位置被持久化保存，下次显示桌面歌词时恢复到该位置

## 场景五

### 场景概述

桌面歌词已开启且未锁定，用户点击歌词文本（非拖动），切换控制面板显示

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示在设备桌面上，且未处于锁定状态
- 2. 用户点击歌词文本区域（手指按下后未发生移动即抬起）
- 3. 如果控制面板当前为隐藏状态，则显示音乐控制面板和样式控制面板
- 4. 如果控制面板当前为显示状态，则隐藏音乐控制面板和样式控制面板

## 场景六

### 场景概述

桌面歌词已开启，用户通过控制面板锁定桌面歌词

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示，用户点击歌词文本展开控制面板
- 2. 用户点击音乐控制面板中的"锁定"按钮
- 3. 桌面歌词进入锁定状态，触摸事件穿透歌词悬浮窗传递到下层应用
- 4. 控制面板自动隐藏
- 5. 弹出toast提示"桌面歌词已锁定"
- 6. 锁定状态被持久化保存

## 场景七

### 场景概述

桌面歌词已锁定，用户通过设置页面解锁桌面歌词

### 场景逻辑步骤

- 1. 桌面歌词处于锁定状态，用户无法通过触摸歌词悬浮窗进行交互
- 2. 用户进入"设置-歌词"页面，找到"锁定桌面歌词"开关（当前为开启状态）
- 3. 用户关闭"锁定桌面歌词"开关
- 4. 桌面歌词恢复为可触摸状态，用户可以再次拖动歌词和点击展开控制面板
- 5. 弹出toast提示"桌面歌词已解锁"

## 场景八

### 场景概述

桌面歌词已开启，播放歌曲切换时歌词内容实时更新

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示在设备桌面上
- 2. 当前播放歌曲的歌词逐行更新时，桌面歌词悬浮窗中的歌词文本同步刷新为当前歌词行
- 3. 当切换到下一首歌曲时，桌面歌词悬浮窗中的歌词文本同步刷新为新歌曲的当前歌词行
- 4. 当当前歌曲无歌词时，桌面歌词悬浮窗显示歌曲副标题；若副标题也为空则显示空白

## 场景九

### 场景概述

应用启动时，根据持久化的开关状态自动恢复桌面歌词显示

### 场景逻辑步骤

- 1. 应用启动，音乐服务初始化
- 2. 读取持久化保存的桌面歌词开关状态
- 3. 如果开关状态为开启，则自动显示桌面歌词悬浮窗，恢复之前保存的垂直位置
- 4. 如果开关状态为关闭，则不显示桌面歌词悬浮窗

## 场景十

### 场景概述

桌面歌词已开启且未锁定，用户通过控制面板调整歌词文字大小

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示，用户点击歌词文本展开控制面板
- 2. 样式控制面板中包含文字大小减小按钮和文字大小增大按钮
- 3. 用户点击减小按钮，歌词文字大小减小一档（每次减小4dp），最小不低于12dp
- 4. 用户点击增大按钮，歌词文字大小增大一档（每次增大4dp），最大不超过36dp
- 5. 文字大小变化实时生效并持久化保存

## 场景十一

### 场景概述

桌面歌词已开启且未锁定，用户通过控制面板更换歌词文字颜色

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示，用户点击歌词文本展开控制面板
- 2. 样式控制面板中包含多个颜色选项（高亮色、绿色、红色、白色）
- 3. 用户点击某个颜色选项，歌词文字颜色立即变为对应颜色
- 4. 颜色变化实时生效并持久化保存

## 场景十二

### 场景概述

桌面歌词已开启且未锁定，用户通过音乐控制面板控制播放

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗已显示，用户点击歌词文本展开控制面板
- 2. 音乐控制面板中包含上一首、播放/暂停、下一首、锁定按钮
- 3. 用户点击"上一首"按钮，切换到上一首歌曲
- 4. 用户点击"播放/暂停"按钮，切换当前歌曲的播放/暂停状态
- 5. 用户点击"下一首"按钮，切换到下一首歌曲

## 场景十三

### 场景概述

桌面歌词已锁定，用户尝试触摸歌词悬浮窗

### 场景逻辑步骤

- 1. 桌面歌词处于锁定状态
- 2. 用户触摸歌词悬浮窗所在的屏幕区域
- 3. 触摸事件穿透歌词悬浮窗，传递到下层的应用或桌面
- 4. 歌词悬浮窗不响应任何触摸操作，无法拖动也无法展开控制面板
