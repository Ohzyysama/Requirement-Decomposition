# 沉浸模式SPEC

设置-用户界面-沉浸模式，开关默认关闭，打开后隐藏状态栏（但保留状态栏的区域边距），并且仅保留播放页面的歌词、封面以及基本的控制按钮，带来沉浸体验。

## 场景一

### 场景概述

用户开启沉浸模式，播放页面隐藏非核心元素并隐藏状态栏

### 场景逻辑步骤

- 1. 用户通过以下任一方式开启沉浸模式：
  - 1.1 在"设置-用户界面"页面中打开"沉浸模式"开关
  - 1.2 在播放页面长按播放暂停按钮
  - 1.3 在播放页面的控制面板中点击"沉浸模式"切换按钮
- 2. 系统隐藏状态栏
- 3. 播放页面以动画方式隐藏以下元素：
  - 3.1 顶部歌曲标题和艺术家名称栏
  - 3.2 底部功能图标栏（播放模式、播放队列、音效等图标）
  - 3.3 进度条及时间显示
  - 3.4 封面上的Hi-Res标识
  - 3.5 封面上的迷你歌词和音频信息文字
  - 3.6 歌词页面底部的歌词设置栏（翻译切换等）
- 4. 播放页面保留以下元素：
  - 4.1 专辑封面（封面居中显示，上方留出更多空间）
  - 4.2 歌词内容
  - 4.3 基本控制按钮（播放/暂停、上一首、下一首）
- 5. 沉浸模式状态持久化保存，下次打开应用时保持该状态

## 场景二

### 场景概述

用户关闭沉浸模式，且"隐藏状态栏"设置为关闭状态，播放页面恢复所有元素

### 场景逻辑步骤

- 1. 用户当前处于沉浸模式开启状态，且"设置-用户界面-隐藏状态栏"开关为关闭状态
- 2. 用户通过以下任一方式关闭沉浸模式：
  - 2.1 在"设置-用户界面"页面中关闭"沉浸模式"开关
  - 2.2 在播放页面长按播放暂停按钮
  - 2.3 在播放页面的控制面板中点击"沉浸模式"切换按钮
- 3. 系统恢复显示状态栏
- 4. 播放页面以动画方式恢复显示所有被隐藏的元素：
  - 4.1 顶部歌曲标题和艺术家名称栏
  - 4.2 底部功能图标栏
  - 4.3 进度条及时间显示
  - 4.4 封面上的Hi-Res标识
  - 4.5 封面上的迷你歌词和音频信息文字
  - 4.6 歌词页面底部的歌词设置栏
- 5. 封面恢复为正常布局位置
- 6. 沉浸模式关闭状态持久化保存

## 场景三

### 场景概述

用户关闭沉浸模式，但"隐藏状态栏"设置为开启状态，状态栏保持隐藏

### 场景逻辑步骤

- 1. 用户当前处于沉浸模式开启状态，且"设置-用户界面-隐藏状态栏"开关为开启状态
- 2. 用户通过以下任一方式关闭沉浸模式：
  - 2.1 在"设置-用户界面"页面中关闭"沉浸模式"开关
  - 2.2 在播放页面长按播放暂停按钮
  - 2.3 在播放页面的控制面板中点击"沉浸模式"切换按钮
- 3. 由于"隐藏状态栏"设置为开启状态，状态栏保持隐藏不恢复显示
- 4. 播放页面以动画方式恢复显示所有被隐藏的元素（与场景二步骤4相同）
- 5. 封面恢复为正常布局位置
- 6. 沉浸模式关闭状态持久化保存

## 场景四

### 场景概述

用户通过长按播放暂停按钮切换沉浸模式时，显示提示信息

### 场景逻辑步骤

- 1. 用户在播放页面长按播放暂停按钮
- 2. 沉浸模式状态切换（若当前为关闭则开启，若当前为开启则关闭）
- 3. 弹出toast提示用户当前沉浸模式的状态变化
- 4. 播放页面按照对应的开启或关闭逻辑执行UI变化

## 场景五

### 场景概述

应用首次安装或重置后，沉浸模式默认为关闭状态

### 场景逻辑步骤

- 1. 用户首次安装应用或清除应用数据后打开应用
- 2. "设置-用户界面"页面中"沉浸模式"开关默认为关闭状态
- 3. 播放页面显示所有UI元素，状态栏正常显示
- 4. 控制面板中的沉浸模式切换按钮显示为未激活状态
