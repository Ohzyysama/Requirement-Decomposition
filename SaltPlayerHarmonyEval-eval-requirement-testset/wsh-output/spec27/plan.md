# 锁定桌面歌词SPEC

设置-歌词-锁定桌面歌词，开关默认关闭，依赖桌面歌词开关，打开后固定当前设备桌面歌词位置。

## 场景一

### 场景概述

桌面歌词开关关闭时，锁定桌面歌词开关不可操作

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面
- 2. 当前"桌面歌词"开关处于关闭状态
- 3. "锁定桌面歌词"开关显示为禁用状态，用户无法点击或切换该开关

## 场景二

### 场景概述

桌面歌词开关打开时，用户开启锁定桌面歌词

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面
- 2. 当前"桌面歌词"开关处于打开状态，桌面歌词悬浮窗已显示
- 3. 用户点击"锁定桌面歌词"开关，将其从关闭切换为打开
- 4. 桌面歌词悬浮窗固定在当前位置，不再响应触摸拖动操作，触摸事件穿透到下层应用
- 5. 桌面歌词悬浮窗上的控制面板隐藏
- 6. 弹出提示"桌面歌词已锁定"
- 7. 通知栏中桌面歌词图标更新为锁定状态图标

## 场景三

### 场景概述

桌面歌词已锁定时，用户通过设置页解锁桌面歌词

### 场景逻辑步骤

- 1. 用户进入"设置-歌词"页面
- 2. 当前"桌面歌词"开关处于打开状态，"锁定桌面歌词"开关处于打开状态
- 3. 用户点击"锁定桌面歌词"开关，将其从打开切换为关闭
- 4. 桌面歌词悬浮窗恢复可触摸交互状态，用户可以拖动调整位置
- 5. 弹出提示"桌面歌词已解锁"
- 6. 通知栏中桌面歌词图标更新为未锁定状态图标

## 场景四

### 场景概述

用户通过桌面歌词悬浮窗上的锁定按钮锁定桌面歌词

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗当前处于未锁定状态，用户触摸悬浮窗显示控制面板
- 2. 用户点击控制面板中的"锁定"按钮
- 3. 桌面歌词悬浮窗固定在当前位置，不再响应触摸拖动操作，触摸事件穿透到下层应用
- 4. 控制面板隐藏
- 5. 弹出提示"桌面歌词已锁定"
- 6. "设置-歌词"页面中"锁定桌面歌词"开关状态同步更新为打开
- 7. 通知栏中桌面歌词图标更新为锁定状态图标

## 场景五

### 场景概述

桌面歌词已锁定时，用户通过通知栏解锁桌面歌词

### 场景逻辑步骤

- 1. 桌面歌词悬浮窗当前处于锁定状态
- 2. 用户点击通知栏中的桌面歌词按钮
- 3. 桌面歌词悬浮窗恢复可触摸交互状态，用户可以拖动调整位置
- 4. 弹出提示"桌面歌词已解锁"
- 5. "设置-歌词"页面中"锁定桌面歌词"开关状态同步更新为关闭
- 6. 通知栏中桌面歌词图标更新为未锁定状态图标

## 场景六

### 场景概述

锁定状态下关闭桌面歌词开关，再次打开桌面歌词时保持之前的锁定状态

### 场景逻辑步骤

- 1. 用户已开启"锁定桌面歌词"，桌面歌词处于锁定状态
- 2. 用户关闭"桌面歌词"开关，桌面歌词悬浮窗消失
- 3. 用户再次打开"桌面歌词"开关，桌面歌词悬浮窗重新显示
- 4. 由于锁定状态已持久化存储，悬浮窗以锁定状态显示，固定在之前保存的位置，不响应触摸拖动操作

## 场景七

### 场景概述

锁定桌面歌词后，歌词位置在当前设备上持久保存

### 场景逻辑步骤

- 1. 用户拖动桌面歌词悬浮窗到目标位置
- 2. 用户开启"锁定桌面歌词"
- 3. 悬浮窗位置被固定并持久化存储到当前设备
- 4. 应用重启后，桌面歌词悬浮窗仍然显示在之前锁定时的位置
