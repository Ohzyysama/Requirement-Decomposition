# 在列表中显示添加下一首播放SPEC

设置-用户界面中提供"在列表中显示添加下一首播放"开关，控制歌曲列表中每首歌曲是否显示"添加到下一首播放"按钮。

## 场景一

### 场景概述

开关默认为打开状态，歌曲列表中每首歌曲显示"添加到下一首播放"按钮

### 场景逻辑步骤

- 1. 用户首次使用应用或未修改过该设置
- 2. "在列表中显示添加下一首播放"开关默认为打开状态
- 3. 在歌曲列表中，每首歌曲的右侧显示"添加到下一首播放"按钮
- 4. 歌曲标题和艺术家文本区域自动缩短，为按钮留出显示空间

## 场景二

### 场景概述

用户关闭开关后，歌曲列表中每首歌曲不再显示"添加到下一首播放"按钮

### 场景逻辑步骤

- 1. 用户进入"设置"页面，进入"用户界面"设置项，找到"歌曲列表"分区下的"在列表中显示添加下一首播放"开关
- 2. 用户将开关从打开切换为关闭
- 3. 设置立即生效并持久化保存
- 4. 歌曲列表中所有歌曲的"添加到下一首播放"按钮立即隐藏
- 5. 歌曲标题和艺术家文本区域自动扩展，占用原按钮所在的空间

## 场景三

### 场景概述

用户重新打开开关后，歌曲列表中每首歌曲恢复显示"添加到下一首播放"按钮

### 场景逻辑步骤

- 1. 用户进入"设置"页面，进入"用户界面"设置项，找到"歌曲列表"分区下的"在列表中显示添加下一首播放"开关
- 2. 用户将开关从关闭切换为打开
- 3. 设置立即生效并持久化保存
- 4. 歌曲列表中所有歌曲的"添加到下一首播放"按钮立即恢复显示
- 5. 歌曲标题和艺术家文本区域自动缩短，为按钮留出显示空间

## 场景四

### 场景概述

开关打开时，用户点击歌曲列表中某首歌曲的"添加到下一首播放"按钮，该歌曲被添加到下一首播放

### 场景逻辑步骤

- 1. "在列表中显示添加下一首播放"开关为打开状态
- 2. 用户在歌曲列表中点击某首歌曲的"添加到下一首播放"按钮
- 3. 该歌曲被添加到当前播放队列的下一首位置
- 4. 弹出toast提示"成功添加到下一首播放"

## 场景五

### 场景概述

开关打开时，用户进入多选模式，"添加到下一首播放"按钮自动隐藏

### 场景逻辑步骤

- 1. "在列表中显示添加下一首播放"开关为打开状态
- 2. 用户在歌曲列表中进入多选模式
- 3. 歌曲列表中所有歌曲的"添加到下一首播放"按钮自动隐藏
- 4. 歌曲标题和艺术家文本区域自动扩展，占用原按钮所在的空间

## 场景六

### 场景概述

开关打开时，用户退出多选模式，"添加到下一首播放"按钮自动恢复显示

### 场景逻辑步骤

- 1. "在列表中显示添加下一首播放"开关为打开状态
- 2. 用户在歌曲列表的多选模式下退出多选模式
- 3. 歌曲列表中所有歌曲的"添加到下一首播放"按钮自动恢复显示
- 4. 歌曲标题和艺术家文本区域自动缩短，为按钮留出显示空间
