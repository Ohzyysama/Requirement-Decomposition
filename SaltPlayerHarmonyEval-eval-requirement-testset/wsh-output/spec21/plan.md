# 在列表中显示歌曲文件名SPEC

设置-用户界面中提供一个开关，默认关闭，打开后在歌曲列表中显示歌曲文件名（包含扩展名）而不是歌曲标题（文件内嵌的元数据标题）。

## 场景一

### 场景概述

开关默认关闭，歌曲列表中显示歌曲标题

### 场景逻辑步骤

- 1. 用户首次使用应用或未曾修改过该设置
- 2. "在列表中显示歌曲文件名"开关处于关闭状态
- 3. 应用中所有歌曲列表（包括歌曲列表页、歌单详情页、专辑详情页、艺术家详情页、文件夹详情页等）的歌曲主文本显示为歌曲的元数据标题

## 场景二

### 场景概述

用户打开开关后，歌曲列表实时切换为显示歌曲文件名

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，找到"歌曲列表"分类下的"在列表中显示歌曲文件名"开关
- 2. 用户打开该开关
- 3. 应用中所有歌曲列表的歌曲主文本立即切换为显示歌曲的文件名称（包含文件扩展名），而非元数据标题
- 4. 该设置持久化存储，应用重启后开关仍保持打开状态，歌曲列表继续显示文件名

## 场景三

### 场景概述

用户关闭开关后，歌曲列表实时切换为显示歌曲标题

### 场景逻辑步骤

- 1. 用户进入"设置-用户界面"页面，找到"歌曲列表"分类下的"在列表中显示歌曲文件名"开关
- 2. 用户关闭该开关
- 3. 应用中所有歌曲列表的歌曲主文本立即切换为显示歌曲的元数据标题，而非文件名称
- 4. 该设置持久化存储，应用重启后开关仍保持关闭状态，歌曲列表继续显示元数据标题

## 场景四

### 场景概述

开关打开时，文件名的显示规则为文件路径中的最后一段（含扩展名）

### 场景逻辑步骤

- 1. 用户打开"在列表中显示歌曲文件名"开关
- 2. 歌曲列表中每首歌曲显示的文件名取自该歌曲实际存储路径的最后一段，即文件名加扩展名（例如路径为"/storage/music/周杰伦 - 晴天.flac"，则显示"周杰伦 - 晴天.flac"）
- 3. 文件名完整显示，不去除扩展名部分
