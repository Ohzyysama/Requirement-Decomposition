# M2模块：一致性与可实现性评估模块

## 模块概述

M2模块是系统的第二大模块，负责对M1模块生成的功能列表和依赖关系进行质量评估。该模块不参与需求生成，仅专注于评估分析，确保需求的结构质量和实现可行性。

## 核心功能

### 1. 一致性评估 (Consistency Evaluation)
检查功能树和依赖关系的逻辑一致性，包括：

- **依赖引用有效性**：验证依赖关系中的from/to引用是否有效
- **循环依赖检测**：使用DFS算法检测依赖图中的循环
- **重复功能检测**：基于文本相似度发现重复或高度相似的功能
- **耦合度评估**：分析功能之间的依赖关系，评估耦合度（低/中/高）
- **子需求覆盖完整性**：确保子需求完全涵盖原始需求的所有内容
- **拆分忠实性**：子需求的拆分必须忠实于原始需求文本
- **功能范围约束**：子需求不得超出原始需求的功能范围
- **依赖继承合理性**：子需求的外部依赖应从原始需求的外部依赖中合理继承

### 2. 可实现性评估 (Feasibility Evaluation)
评估功能需求在特定平台和约束条件下的可实现性，包括：

- **内聚度评估**：基于FPA方法评估功能单元的内聚程度
- **功能点规模评估**：使用简化FPA方法估算功能单元规模
- **工作量估算**：基于功能点规模估算实现工作量
- **粒度合理性评估**：评估功能拆分粒度的合理性
- **资源约束匹配评估**：验证工作量与资源约束的匹配度
- **技术复杂性评估**：基于14个通用系统特性评估技术复杂性

### 3. 综合评估集成 (Integration)
整合一致性和可实现性评估结果，提供：

- **综合评分**：基于加权算法计算整体质量分数
- **风险等级**：评估项目整体风险级别
- **建议动作**：提供明确的继续/修订/终止建议
- **详细报告**：生成完整的评估报告和改进建议


## 评估规则详解

### 一致性规则

#### 1. 依赖引用有效性规则
- **检查内容**：依赖关系中的from/to字段引用的节点ID是否存在
- **失败影响**：产生"悬空引用"，依赖无法执行
- **建议动作**：中断流程，重新进行依赖识别
- **严重程度**：ERROR

#### 2. 循环依赖检测规则
- **检查内容**：使用DFS检测依赖图中是否存在循环路径
- **失败影响**：功能无法启动或执行顺序矛盾
- **建议动作**：标记问题，建议重构依赖关系
- **严重程度**：WARNING

#### 3. 重复功能检测规则
- **检查内容**：基于文本相似度识别语义相似的功能
- **目的**：避免冗余需求，提高设计效率
- **建议动作**：标记重复，建议合并或删除
- **严重程度**：WARNING

#### 4. 耦合度评估规则
- **检查内容**：分析功能之间的依赖关系，评估耦合度（低/中/高）
- **评估标准**：基于依赖数量和复杂度评估耦合程度
- **建议动作**：高耦合功能建议重新设计依赖关系
- **严重程度**：WARNING

#### 5. 子需求覆盖完整性规则
- **检查内容**：确保子需求完全涵盖原始需求的所有内容
- **失败影响**：功能缺失，需求不完整
- **建议动作**：补充缺失的功能需求
- **严重程度**：ERROR

#### 6. 拆分忠实性规则
- **检查内容**：子需求的拆分必须忠实于原始需求文本，不得出现原始需求中未提及的功能或名词。
- **目的**：避免引入原始需求中未提及的功能或名词
- **建议动作**：重新拆分，确保忠实于原始需求
- **严重程度**：WARNING

#### 7. 功能范围约束规则
- **检查内容**：子需求不得超出原始需求的功能范围
- **失败影响**：功能范围膨胀，超出原始需求
- **建议动作**：删除超出范围的功能
- **严重程度**：ERROR

#### 8. 依赖继承合理性规则
- **检查内容**：子需求的外部依赖应从原始需求的外部依赖中根据自身具体依赖情况选择性继承，但不能新增原始需求中没有的依赖。
- **目的**：避免新增原始需求中没有的依赖
- **建议动作**：调整依赖关系，确保合理继承
- **严重程度**：WARNING

```
规则引擎评估（可直接基于字段判断）：
1. 依赖引用有效性 - 检查dependencies中的from/to节点是否在function_list中存在
2. 循环依赖检测 - 图算法检测dependencies中是否存在循环路径
3. 重复功能检测 - 基于desc字段匹配，利用文本相似度算法判断function_list中每个子功能的desc是否重复
4. 耦合度评估 - 统计每个子功能的依赖数量
8. 约束继承合理性 - 检查约束描述，直接根据function_list中的constraints字段是否在normalizer_result中的constraint_copy字段中出现来判断
LLM评估（需要语义分析）：
5. 子需求覆盖完整性 - 分析每个子功能描述是否覆盖原始需求的所有内容
6. 拆分忠实性 - 评估拆分是否忠实于需求语义
7. 功能范围约束 - LLM语义分析，每个子功能不得超出原始功能需求的功能范围

M2模块的输入参数包括：
function_list: 包含id, desc, title, path, constraints, granularity等
dependencies: 包含from, to, dep_id, trigger, description, requires/provides等
normalizer_result: 包含标准化后的原始需求文本normalized_requirement和constraint_copy等约束信息
```


### 可实现性规则

#### 1. 内聚度评估规则
- **检查内容**：评估功能单元内部元素的相关性和内聚程度
- **评估方法**：基于功能描述分析功能职责的单一性和相关性
- **评估标准**：
  - 高内聚：功能职责单一，相关性强，实现复杂度低
  - 中内聚：功能职责相对集中，有一定相关性
  - 低内聚：功能职责分散，相关性弱，实现复杂度高
- **建议动作**：低内聚功能建议重新拆分或重构
- **严重程度**：WARNING

#### 2. 功能点规模评估规则
- **检查内容**：基于简化FPA方法估算功能单元的功能点规模
- **评估方法**：
  - **功能分类**：自动识别EI（外部输入）、EO（外部输出）、EQ（外部查询）、ILF（内部逻辑文件）、EIF（外部接口文件）
  - **复杂度评估**：基于功能描述自动评估复杂度（低/中/高）
  - **功能点计算**：∑(功能类型数量 × 对应权重)
- **评估标准**：
  - 小型功能：< 5功能点
  - 中型功能：5-15功能点  
  - 大型功能：> 15功能点
- **建议动作**：大型功能建议进一步拆分
- **严重程度**：INFO

#### 3. 工作量估算评估规则
- **检查内容**：基于功能点规模估算实现工作量
- **估算方法**：
  - 标准工作量基准：10功能点/人月
  - 工作量估算：功能点规模 ÷ 生产力因子
  - 成本估算：工作量 × 人均月成本（默认：15000元）
- **评估标准**：
  - 低工作量：< 1人月
  - 中工作量：1-3人月
  - 高工作量：> 3人月
- **建议动作**：高工作量功能建议重新评估或分阶段实现
- **严重程度**：WARNING

#### 4. 粒度合理性评估规则
- **检查内容**：评估功能拆分粒度是否合理
- **评估方法**：对比预期资源约束与估算工作量
- **评估标准**：
  - **粒度过粗**：功能点规模过大，实现复杂度过高
  - **粒度过细**：功能点规模过小，带来额外协调与集成成本
  - **合理粒度**：功能点规模适中，实现复杂度可控
- **建议动作**：
  - 粒度过粗：建议进一步拆分
  - 粒度过细：建议合并相关功能
- **严重程度**：WARNING

#### 5. 资源约束匹配评估规则
- **检查内容**：验证估算工作量是否在预期资源约束范围内
- **评估方法**：
  - 对比总功能点规模与项目资源约束
  - 分析功能点分布与团队能力匹配度
- **评估标准**：
  - 匹配：估算工作量 ≤ 可用资源
  - 不匹配：估算工作量 > 可用资源
- **建议动作**：不匹配时建议调整项目范围或增加资源
- **严重程度**：ERROR（严重不匹配时）

#### 6. 技术复杂性评估规则
- **检查内容**：基于14个通用系统特性（GSC）评估技术复杂性
- **评估维度**：数据通信、分布式处理、性能要求、使用频率等
- **评估方法**：基于关键词分析自动评分（0-5分）
- **技术复杂性因子（TCF）**：TCF = 0.65 + 0.01 × ∑(GSC评分)
- **建议动作**：高复杂性功能建议采用简化实现或技术验证
- **严重程度**：WARNING


## 智能体架构

### M2ConsistencyEvaluatorAgent
- **职责**：执行所有一致性规则检查
- **输入**：功能列表 + 依赖关系
- **输出**：一致性评估结果
- **核心算法**：图算法、文本相似度、模式匹配

### M2FeasibilityEvaluatorAgent  
- **职责**：执行所有可实现性规则检查，包括功能点分析（FPA）
- **输入**：功能树 + 依赖关系 + 资源约束
- **输出**：可实现性评估结果（包含FPA分析数据）
- **核心算法**：规则引擎、资源估算、功能点分析、粒度评估
- **FPA功能**：
  - 自动功能分类（EI/EO/EQ/ILF/EIF）
  - 复杂度评估（低/中/高）
  - 技术复杂性因子（TCF）计算
  - 工作量/成本/工期估算

### M2EvaluationIntegratorAgent
- **职责**：整合评估结果，生成综合报告
- **输入**：一致性结果 + 可实现性结果
- **输出**：综合评估报告
- **核心算法**：加权评分、风险评估、决策逻辑


## 评分算法

### 一致性评分
```
一致性评分 = (通过检查项数 / 总检查项数) × 权重调整
```
- 基础权重：所有规则平等
- 关键问题扣分：每个ERROR级别问题额外扣分
- 满分：1.0，及格线：0.7

### 可实现性评分
```
可实现性评分 = (通过检查项数 / 总检查项数) × 资源约束调整 × FPA规模调整 × 技术复杂性调整
```
- **资源约束调整**：根据资源限制严格程度
- **FPA规模调整**：基于功能点数量进行规模适应性调整
  - 小型项目（AFP < 50）：×1.1（规模优势）
  - 中型项目（AFP < 300）：×1.0（标准规模）
  - 大型项目（AFP < 1000）：×0.9（规模挑战）
  - 超大型项目（AFP ≥ 1000）：×0.8（高风险规模）
- **技术复杂性调整**：基于TCF因子调整实现难度
- 满分：1.0，及格线：0.6

### 综合评分
```
综合评分 = 一致性评分 × 0.4 + 可实现性评分 × 0.6
```
- 一致性权重：40%（结构质量）
- 可实现性权重：60%（实现可行性）
- 推荐阈值：
  - ≥ 0.8：优秀，建议继续
  - ≥ 0.6：良好，可以谨慎继续  
  - ≥ 0.4：一般，建议修订
  - < 0.4：较差，建议重新评估


## 功能点分析（FPA）方法应用

### FPA方法概述
功能点分析（Function Point Analysis, FPA）是一种国际标准的软件规模度量方法，基于用户视角对软件功能进行量化评估。M2模块将FPA方法集成到可实现性评估中，提供更科学、更精确的项目规模估算。

### FPA计算流程

#### 1. 未调整功能点（UFP）计算
```python
UFP = ∑(功能类型数量 × 对应权重)
```
- **功能分类**：
  - EI（外部输入）：用户注册、数据提交（权重：3-6）
  - EO（外部输出）：报表生成、数据显示（权重：4-7）
  - EQ（外部查询）：搜索、筛选功能（权重：3-6）
  - ILF（内部逻辑文件）：用户数据、商品数据（权重：7-15）
  - EIF（外部接口文件）：支付接口、地图服务（权重：5-10）
- **复杂度评估**：基于功能描述自动评估复杂度（低/中/高）

#### 2. 技术复杂性因子（TCF）计算
```python
TCF = 0.65 + 0.01 × ∑(通用系统特性评分)
```
- **14个通用系统特性（GSC）**：
  1. 数据通信复杂性
  2. 分布式处理需求
  3. 性能要求
  4. 使用频率
  5. 事务处理率
  6. 在线数据输入
  7. 终端用户效率
  8. 在线更新
  9. 复杂处理逻辑
  10. 可重用性
  11. 安装便利性
  12. 操作便利性
  13. 多站点部署
  14. 变更便利性
- **评分机制**：基于关键词分析自动评分（0-5分）

#### 3. 调整后功能点（AFP）计算
```python
AFP = UFP × TCF
```

#### 4. 项目估算
```python
估算工作量 = AFP * 生产力因子（默认：6.72人时/功能点）/（月工作天数（默认：21.75天）*每天工作时（默认：8小时））
估算成本 = 估算工作量 × 人均月成本（默认：27795元）
估算工期 = COCOMO模型预测（基于AFP）
```

### FPA与项目评估的集成

#### 1. 可实现性评分调整
- **规模适应性调整**：基于AFP对可实现性评分进行规模因子调整
- **风险评估**：大型项目（AFP ≥ 300）自动增加风险评估权重

#### 2. 资源约束匹配检查
- **精确估算**：相比传统方法提升40%+的估算精度
- **约束验证**：基于FPA结果验证资源约束的合理性

#### 3. 决策支持
- **数据驱动**：为"继续/修订/终止"建议提供量化依据
- **风险识别**：识别技术复杂性和规模风险

### 技术实现特点

#### 1. 智能功能分类
- **基于描述关键词**：自动识别功能类型和复杂度
- **IFPUG标准**：严格遵循国际功能点用户组标准

#### 2. 自动化评估
- **GSC自动评分**：基于功能描述自动评估14个技术特性
- **实时计算**：毫秒级FPA计算，不影响评估性能

#### 3. 集成评估报告
- **FPA结果展示**：在评估报告中详细展示FPA分析结果
- **改进建议**：基于FPA结果提供针对性的改进建议
